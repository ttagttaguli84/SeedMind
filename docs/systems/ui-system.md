# UI/UX 시스템 설계 (UI/UX System Design)

> 작성: Claude Code (Opus) | 2026-04-07  
> 문서 ID: DES-011

## Context

이 문서는 SeedMind의 전체 UI/UX 시스템을 상세히 기술한다. HUD, 인벤토리, 퀘스트 로그, 상점/NPC 대화, 시설 건설, 업적, 알림, 메인 메뉴, 접근성에 이르는 모든 화면 구성과 상호작용 규칙을 정의한다.

마스터 디자인 문서(`docs/design.md`) 섹션 6의 UI 개요를 기반으로, 각 시스템 문서에서 개별적으로 기술된 UI 요구사항을 통합하고 일관된 시각 언어와 상호작용 패턴을 확립한다.

**설계 목표**:
- 핵심 루프(심기 -> 관리 -> 수확 -> 판매 -> 확장)를 방해하지 않는 가벼운 HUD
- 모든 패널 UI의 일관된 레이아웃과 조작 패턴
- 정보의 계층적 전달: HUD(상시) -> 토스트(이벤트) -> 패널(요청 시)
- 접근성을 기본 설계에 포함 (후첨이 아닌 내재적 고려)

### 본 문서가 canonical인 데이터

- HUD 영역 배치 및 표시 요소 정의
- 전체 키 바인딩 통합 맵

- 메인 메뉴 / 씬 전환 UI 구조

- 공통 UI 패턴 (팝업, 확인 대화상자, 툴팁 규격)

## 1. UI 설계 원칙

### 1.1 핵심 원칙

| 원칙 | 설명 | 적용 |
|------|------|------|
| **비침투적 HUD** | 게임 화면을 가리는 UI를 최소화한다 | HUD 요소는 화면 가장자리에 배치, 반투명 배경 |
| **일관된 조작** | 모든 패널은 동일한 열기/닫기/탐색 패턴을 따른다 | ESC로 닫기, 탭으로 전환, 드래그앤드롭 통일 |
| **정보 계층화** | 긴급도에 따라 정보 전달 방식을 구분한다 | 상시(HUD) > 이벤트(토스트) > 요청(패널) |
| **맥락적 표시** | 필요한 순간에만 UI를 표시한다 | 상호작용 범위 진입 시 [E] 프롬프트, 비어 있을 때 슬롯 숨김 |
| **데이터 드리븐** | 모든 UI 수치(크기, 색상, 지속 시간)는 ScriptableObject로 외부화 | UIConfig SO에서 글로벌 UI 파라미터 관리 |

### 1.2 시각 스타일

| 항목 | 사양 |
|------|------|
| 전체 톤 | 로우폴리/파스텔 톤과 조화하는 따뜻한 베이지-브라운 계열 UI 프레임 |
| 폰트 | 한국어 본문: NotoSansKR-Regular, 제목: NotoSansKR-Bold |
| 배경 | 반투명 패널 (알파 80~90%), 둥근 모서리 (8px radius) |
| 아이콘 | 64x64px 기본, 32x32px 축소 (슬롯 내), 외곽선 스타일 |
| 색상 팔레트 | 아래 섹션 9.1 참조 |

## 2. HUD 구조

### 2.1 HUD 영역 배치

게임 플레이 중 항상 표시되는 상시 UI 요소의 배치이다.

```
┌──────────────────────────────────────────────────────────────┐
│ [A] 날짜/시간      [B] 계절/날씨           [C] 골드         │
│                                                              │
│                                         ┌──────────────┐    │
│                                         │[D] 퀘스트추적 │    │
│                  (게임 화면)             │              │    │
│                                         └──────────────┘    │
│                                                              │
│ [E] 에너지 바                                                │
│                                                              │
│   ┌──┬──┬──┬──┬──┬──┬──┬──┐                                 │
│   │ 1│ 2│ 3│ 4│ 5│ 6│ 7│ 8│  [F] 툴바                      │
│   └──┴──┴──┴──┴──┴──┴──┴──┘                                 │
│ [G] 레벨/경험치 바                                           │
└──────────────────────────────────────────────────────────────┘
```

### 2.2 영역별 상세

#### [A] 날짜/시간 영역 (좌측 상단)

| 요소 | 표시 형식 | 갱신 주기 |
|------|----------|----------|
| 연도 | "1년차" | 연 변경 시 |
| 계절 내 일차 | "봄 15일" | 일 변경 시 |
| 요일 | "(월)" | 일 변경 시 |
| 시간 | "14:30" | 매 게임 내 분 (-> see `docs/systems/time-season.md` 섹션 1) |

시간 표시는 24시간제를 사용한다. 18:00 이후 시간 텍스트가 주황색으로 변하여 하루가 저물고 있음을 시각적으로 전달한다. 24:00(자정)에 가까워지면 빨간색 깜빡임으로 변경되어 수면이 필요함을 경고한다.

#### [B] 계절/날씨 영역 (상단 중앙)

| 요소 | 표시 방식 |
|------|----------|
| 계절 아이콘 | 봄(꽃), 여름(해), 가을(단풍), 겨울(눈결정) 아이콘 |
| 날씨 아이콘 | 맑음/흐림/비/폭풍/눈/폭설 아이콘 (-> see `docs/systems/time-season.md` 섹션 3) |
| 날씨 변경 예보 | 날씨 아이콘에 마우스 호버 시 "내일: 비" 툴팁 표시 |

#### [C] 골드 영역 (우측 상단)

| 요소 | 표시 형식 |
|------|----------|
| 골드 아이콘 | 동전 아이콘 + 수치 |
| 골드 수치 | "2,580G" (천 단위 쉼표 구분) |
| 골드 변동 | 획득 시 녹색 "+50G" 플로팅 텍스트 (1.5초), 지출 시 빨간색 "-30G" |

#### [D] 퀘스트 추적 영역 (우측 중단)

퀘스트 추적기 표시 규칙: (-> see `docs/systems/quest-system.md` 섹션 8.3, canonical)

| 파라미터 | 값 |
|----------|-----|
| 최대 동시 추적 수 | 2개 |
| 기본 추적 | 메인 퀘스트 1개 자동 추적 |
| 접기/펼치기 | 추적 영역 상단 삼각형 아이콘 클릭으로 토글 |
| 투명도 | 평소 60% 투명, 마우스 호버 시 100% |

#### [E] 에너지 바 (좌측 하단)

| 요소 | 표시 방식 |
|------|----------|
| 에너지 바 | 녹색 수평 게이지 바 (-> see `docs/systems/farming-system.md` 섹션 3.1 for 에너지 시스템) |
| 수치 표시 | 바 위에 "85/100" 형태 |
| 저에너지 경고 | 20% 이하 시 바가 빨간색으로 변경, 펄스 애니메이션 |
| 에너지 소모 피드백 | 도구 사용 시 바에서 소모량만큼 즉시 감소 + 소모 수치 플로팅 |

#### [F] 툴바 (하단 중앙)

툴바 슬롯 수, 단축키, 기본 배치: (-> see `docs/systems/inventory-system.md` 섹션 2.2, canonical)

| 표시 요소 | 위치 | 설명 |
|-----------|------|------|
| 슬롯 번호 | 슬롯 좌측 상단 | 1~8 숫자 |
| 아이템 아이콘 | 슬롯 중앙 | 64x64px → 48x48px 축소 |
| 수량 | 슬롯 우측 하단 | 스택 2 이상일 때만 표시 |
| 선택 표시 | 슬롯 테두리 | 황금색 테두리 + 약간의 확대(110%) |
| 빈 슬롯 | - | 어두운 회색 배경, 테두리만 표시 |

#### [G] 레벨/경험치 바 (하단 좌측)

| 요소 | 표시 방식 |
|------|----------|
| 레벨 숫자 | "Lv. 3" (레벨 아이콘 옆) |
| 경험치 바 | 파란색 수평 게이지 바 (0.0~1.0 진행률) |
| XP 수치 | 바 위에 "450/800" 형태 (-> see `docs/balance/progression-curve.md` 섹션 2.4.1 for XP 테이블) |
| 레벨업 연출 | 레벨업 시 바 전체가 황금빛으로 번쩍 + "Level Up!" 텍스트 팝업 (2초) + 해금 목록 표시 |

## 3. 인벤토리 UI 레이아웃

인벤토리 UI의 세부 구조(슬롯 수, 배치 규칙, 드래그앤드롭, 정렬, 필터, 툴팁): (-> see `docs/systems/inventory-system.md` 섹션 2, 3, 4, canonical)

본 섹션에서는 inventory-system.md에 기술되지 않은 **통합 패널 프레임** 및 **시스템 간 연동** 관점의 UI 규칙을 보충한다.

### 3.1 통합 패널 구조

인벤토리, 퀘스트 로그, 업적은 하나의 **통합 패널 프레임**을 공유하며, 상단 탭으로 전환한다.

```
┌─────────────────────────────────────────────────────┐
│  [인벤토리]  [퀘스트 로그]  [농장 도전]  [업적]      │
├─────────────────────────────────────────────────────┤
│                                                     │
│            (탭에 따라 내용 변경)                     │
│                                                     │
│                                                     │
└─────────────────────────────────────────────────────┘
```

| 파라미터 | 값 |
|----------|-----|
| 열기 키 | `I` 또는 `Tab` (인벤토리 탭 활성), `J` (퀘스트 탭 활성), `Y` (업적 탭 활성) |
| 닫기 키 | `ESC` 또는 동일 열기 키 재입력 |
| 게임 일시정지 | 패널 열림 시 게임 시간 **일시정지하지 않음** (실시간 진행) |
| 마우스 커서 | 패널 열림 시 커서 자동 표시, 닫힘 시 커서 숨김 |

[OPEN] 통합 패널 열림 시 게임 시간 일시정지 여부. 현재 "정지하지 않음"으로 설정했으나, 인벤토리 정리 중 시간이 흘러 밤이 되는 것이 스트레스가 될 수 있다. 플레이테스트 후 결정.

### 3.2 인벤토리 탭 레이아웃

인벤토리 탭의 세부 배치: (-> see `docs/systems/inventory-system.md` 섹션 4.1~4.5, canonical)

주요 요소:
- **배낭 그리드**: 5열 x N행 (슬롯 수에 따라 가변, -> see `docs/systems/inventory-system.md` 섹션 2.1)
- **툴바 미러**: 패널 하단에 툴바 8칸 표시 (배낭과 툴바 간 아이템 이동 가능)
- **하단 버튼**: [정렬] [카테고리 필터] 버튼
- **슬롯 표시**: 아이콘, 수량, 품질 별, 도구 등급 (-> see `docs/systems/inventory-system.md` 섹션 4.2)
- **툴팁**: 0.3초 호버 후 표시 (-> see `docs/systems/inventory-system.md` 섹션 4.3)

### 3.3 드래그앤드롭 시각 피드백

| 상태 | 시각 피드백 |
|------|------------|
| 드래그 시작 | 원본 슬롯 50% 투명화, 커서에 아이템 아이콘 부착 (80% 크기) |
| 유효 대상 위 | 대상 슬롯 테두리 녹색 하이라이트 |
| 무효 대상 위 | 커서에 금지 아이콘 표시 (빨간 원+사선) |
| 드롭 성공 | 대상 슬롯에 아이콘 "착지" 애니메이션 (바운스 0.2초) |
| 드롭 실패 | 아이콘이 원본 슬롯으로 스냅백 (0.15초) |
| 스택 분할 | 우클릭 드래그 시 수량 분할 슬라이더 표시 |

## 4. 퀘스트 로그 UI

퀘스트 로그의 전체 구조, 상태 아이콘, 추적기, 진행도 표시, 완료 연출, 알림: (-> see `docs/systems/quest-system.md` 섹션 8, canonical)

본 섹션에서는 quest-system.md에 기술되지 않은 보충 규칙을 정의한다.

### 4.1 퀘스트 로그 탭 레이아웃

통합 패널의 "퀘스트 로그" 탭을 선택하면 아래 구조가 표시된다.

```
┌─────────────────────────────────────────────────────┐
│  [인벤토리]  [*퀘스트 로그*]  [농장 도전]  [업적]    │
├──────────────────────┬──────────────────────────────┤
│                      │                              │
│  -- 메인 퀘스트 --   │  [선택한 퀘스트 상세]        │
│  [!] 봄의 첫 수확    │                              │
│  [ ] 농장 확장의시작 │  제목: 봄의 첫 수확          │
│                      │  부여: 시스템                │
│  -- NPC 의뢰 --      │                              │
│  [하나] 감자 필요    │  목표:                       │
│                      │  [V] 감자 5개 수확           │
│  -- 일일 목표 --     │  [ ] 당근 5개 수확 (2/5)     │
│  [*] 부지런한 농부   │                              │
│  [*] 오늘의 수확     │  보상: 50G + 5 XP            │
│                      │                              │
│                      │  [추적 설정] [포기]          │
│                      │                              │
├──────────────────────┴──────────────────────────────┤
│  좌측: 퀘스트 목록 (스크롤)  우측: 선택 퀘스트 상세  │
└─────────────────────────────────────────────────────┘
```

### 4.2 퀘스트 목록 정렬 규칙

| 우선순위 | 기준 |
|----------|------|
| 1 | 상태: 완료(보상 수령 가능) > 활성 > 잠김 |
| 2 | 카테고리: 메인 퀘스트 > NPC 의뢰 > 일일 목표 |
| 3 | 기한 임박순 (NPC 의뢰) |

### 4.3 퀘스트 상세 패널

| 요소 | 내용 |
|------|------|
| 제목 | 퀘스트 한국어 제목 (Bold) |
| 부여자 | 시스템 / NPC 이름 + 초상화 |
| 설명 | 퀘스트 설명 텍스트 |
| 목표 목록 | 각 목표별 체크박스 + 진행도 바 (-> see `docs/systems/quest-system.md` 섹션 8.4) |
| 보상 목록 | 골드/XP/아이템 아이콘 + 수량 |

## 5. 상점/NPC 대화 UI

### 5.1 NPC 상호작용 흐름

```
플레이어가 NPC 2타일 이내 진입
    ↓
[E] 프롬프트 표시 (NPC 머리 위 말풍선 아이콘)

    └── 여행 상인(바람이) → 대화 박스 → 특수 상점 UI
```

NPC 이름, 역할, 캐릭터 설정: (-> see `docs/content/npcs.md`, canonical)

### 5.2 대화 박스 구조

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│   ┌──────┐  "안녕! 오늘도 좋은 아침이야.                 │
│   │ NPC  │   새로운 씨앗이 들어왔는데 볼래?"             │
│   │초상화│                                               │
│   │64x64 │                          [상점 열기]          │
│   └──────┘                          [대화 계속]          │

│                                                          │
└──────────────────────────────────────────────────────────┘
```

| 파라미터 | 값 |
|----------|-----|
| 위치 | 화면 하단 중앙 (게임 화면의 하단 25% 영역) |
| 크기 | 화면 너비의 70%, 높이 약 150px |
| NPC 초상화 | 64x64px, 좌측 배치 |
| NPC 이름 | 초상화 아래, Bold 텍스트 |
| 대화 텍스트 | 초상화 우측, 타이핑 애니메이션 (30자/초) |
| 선택지 | 우측 하단에 최대 3개, 세로 배치 |
| 넘기기 | 좌클릭 또는 스페이스바로 타이핑 완료/다음 대화 |
| 닫기 | ESC 또는 "돌아가기" 선택 |

**타이핑 연출**: 대화 텍스트가 한 글자씩 표시된다. 표시 중 좌클릭하면 즉시 전문 표시. 전문 표시 상태에서 다시 좌클릭하면 다음 대화로 진행.

### 5.4 구매 플로우

```
1. 상점 인벤토리에서 아이템 좌클릭 → 선택 (하이라이트)
2. 하단 수량 조절: +/- 버튼 또는 마우스 휠 (1, 5, 10, 전체 단위)
3. 합계 가격 표시 (골드 부족 시 빨간색)
4. [구매하기] 클릭 → 골드 차감 + 배낭에 추가
5. 배낭 가득 참 시 → [구매하기] 비활성 + "배낭이 가득 찼습니다" 메시지
```

상점 UI 규칙 상세(미해금 표시, 골드/재료 부족 피드백, 수량 선택, 툴팁): (-> see `docs/systems/economy-system.md` 섹션 3.4, canonical)

### 5.5 판매 플로우

```
1. [판매 탭] 클릭 → 좌측이 판매 가능 아이템 목록으로 전환
2. 배낭에서 판매할 아이템 좌클릭 → 선택
3. 수량 조절 → 합계 판매가 표시
4. [판매하기] 클릭 → 아이템 소모 + 골드 획득
5. 도구/건설 재료 → 판매 불가 (회색 처리)
```

## 8. 알림 시스템 (Notification System)

### 8.1 알림 유형 및 우선순위

게임 내 모든 알림을 통합 관리하는 우선순위 큐 시스템.

| 우선순위 | 유형 | 표시 방식 | 지속 시간 | 예시 |
|----------|------|----------|----------|------|
| 1 (최고) | 시스템 경고 | 화면 중앙 모달 팝업 | 플레이어 확인까지 | "저장에 실패했습니다" |
| 2 | 레벨업/해금 | 화면 중앙 배너 | 3초 (자동 닫힘) | "Level Up! Lv.3 달성!" |
| 3 | 퀘스트 완료 | 화면 중앙 배너 | 3초 | "퀘스트 완료! [봄의 첫 수확]" |
| 4 | 업적 달성 | 상단 토스트 | 4~6초 | "업적 달성! [첫 수확의 기쁨]" |
| 5 | 새 퀘스트/의뢰 | 상단 토스트 | 3초 | "새 퀘스트: [농장 확장의 시작]" |
| 6 | 일일 목표 갱신 | 아침 팝업 | 플레이어 확인까지 | "오늘의 목표 2개가 설정되었습니다" |
| 7 | 진행도 갱신 | 우측 플로팅 텍스트 | 2초 | "+1 감자 수확 (7/10)" |
| 8 | 골드 변동 | 골드 영역 플로팅 | 1.5초 | "+50G" / "-30G" |
| 9 | 경고/힌트 | 상단 바 | 2~3초 | "배낭이 가득 찼습니다!", "기한이 내일까지입니다!" |
| 10 (최저) | 컨텍스트 힌트 | 하단 프롬프트 | 상황 유지 동안 | "[E] 상호작용", "[1] 호미 선택" |

### 8.2 큐 관리 규칙

| 규칙 | 설명 |
|------|------|
| 동시 표시 제한 | 토스트 알림은 최대 3개까지 동시 표시 (초과 시 큐에 대기, -> see `docs/systems/ui-architecture.md` 섹션 3.2 for `_maxVisibleToasts`) |
| 우선순위 선점 | 높은 우선순위 알림이 들어오면 낮은 우선순위 알림을 즉시 숨기고 교체 |
| 큐 최대 크기 | 10개 (초과 시 가장 낮은 우선순위 알림 폐기) |
| 같은 유형 병합 | 동일 유형의 알림이 연속 발생하면 병합 (예: 수확 진행도 "7/10" → "8/10" 갱신) |
| 패널 열림 시 | 통합 패널이 열려 있으면 토스트 알림을 큐에 보류, 패널 닫힌 후 순차 표시 |

## 12. UI 계층 구조 (Canvas Sorting Order)

Unity Canvas의 Sorting Order를 정의하여 UI 요소 간 겹침 순서를 관리한다.

> **구현 canonical 참조**: Canvas 오브젝트별 실제 Sort Order 값(Canvas_HUD=0, Canvas_Screen=10, Canvas_Popup=20, Canvas_Notification=30, Canvas_Tutorial=40)은 `docs/systems/ui-architecture.md` 섹션 2.2가 canonical이다. 아래 표는 **논리적 우선순위 개념 모델**로, 각 UI 요소가 어느 Canvas에 속하는지를 보여준다.

| 논리적 우선순위 | UI 요소 | 소속 Canvas | 설명 |
|----------------|---------|------------|------|
| 최하 (HUD) | HUD (영역 A~G) | Canvas_HUD (Sort Order 0) | 상시 표시 |
| 낮음 | 월드 오버레이 (건설 진행, 작물 상태) | Canvas_WorldSpace | 게임 오브젝트 위 표시 |
| 낮음 | 컨텍스트 프롬프트 ([E] 상호작용 등) | Canvas_WorldSpace | 필요 시 표시 |
| 중간 | 통합 패널 (인벤토리/퀘스트/업적) | Canvas_Screen (Sort Order 10) | 요청 시 표시 |
| 중간 | 상점 UI / 대화 박스 | Canvas_Screen (Sort Order 10) | NPC 상호작용 시 |
| 중간 | 건설 배치 모드 UI | Canvas_Screen (Sort Order 10) | 배치 중 표시 |
| 높음 | 알림 토스트 | Canvas_Notification (Sort Order 30) | 이벤트 발생 시 |
| 높음 | 레벨업/퀘스트 완료 배너 | Canvas_Popup (Sort Order 20) | 중앙 배너 |
| 높음 | 확인 팝업 / 모달 대화상자 | Canvas_Popup (Sort Order 20) | 최상위 상호작용 |
| 최고 | 시스템 경고 | Canvas_Popup (Sort Order 20) | 최고 우선순위 |
| 최고 | 씬 전환 오버레이 (페이드/커튼) | Canvas_Tutorial (Sort Order 40) 또는 별도 Canvas | 전체 화면 덮기 |
| 최고 | 로딩 화면 | 별도 Canvas (씬 로드 중) | 씬 로드 중 표시 |

## 13. 공통 UI 패턴

### 13.1 확인 팝업 (Confirmation Dialog)

파괴적 동작(삭제, 폐기, 포기 등)에 사용하는 표준 확인 대화상자.

```
┌────────────────────────────────┐
│                                │
│   정말 이 아이템을 버리시겠습니까?  │
│   "감자 x5"                    │
│                                │
│        [예]      [아니오]      │
│                                │
└────────────────────────────────┘
```

| 파라미터 | 값 |
|----------|-----|
| 배경 | 50% 검정 오버레이 (게임 화면 위) |
| 기본 선택 | [아니오] (실수 방지) |
| 키보드 | Enter = 기본 선택 실행, ESC = 취소(아니오) |

### 13.2 툴팁 (Tooltip)

모든 아이템 및 상호작용 가능한 UI 요소에 적용되는 표준 툴팁.

| 파라미터 | 값 |
|----------|-----|
| 표시 지연 | 0.3초 호버 후 |
| 위치 | 커서 우측 하단 (화면 밖 넘침 시 자동 조정) |
| 최대 너비 | 250px |
| 아이템 툴팁 구조 | 이름(Bold) + 카테고리 + 설명 + 속성 + 가격 (-> see `docs/systems/inventory-system.md` 섹션 4.3) |

### 13.3 플로팅 텍스트 (Floating Text)

월드 또는 HUD 영역에 잠시 표시되는 텍스트.

| 파라미터 | 값 |
|----------|-----|
| 애니메이션 | 아래에서 위로 20px 이동 + 페이드 아웃 |
| 지속 시간 | 1.5~2초 |
| 색상 | 맥락에 따라 (골드 획득: 녹색, 손실: 빨간색, XP: 파란색) |
| 폰트 크기 | HUD 기본 크기의 120% (강조) |

## Open Questions [OPEN]

1. **[OPEN]** 통합 패널 열림 시 게임 시간 일시정지 여부. 현재 "정지하지 않음"으로 설정했으나, 인벤토리 정리 중 시간이 흘러 밤이 되는 것이 스트레스가 될 수 있다. 플레이테스트 후 결정. (-> see 섹션 3.1)

2. **[OPEN]** `Y` 키 업적 패널 할당 최종 확정 필요. 전체 키 바인딩 충돌 검증 후 확정. (-> see `docs/systems/achievement-system.md` 섹션 5.1, 본 문서 섹션 11)

3. **[OPEN]** design.md 섹션 5의 도구 선택 키 범위를 "1~5"에서 "1~8"로 업데이트 필요. 툴바 8칸 확정에 따른 후속 조치. (-> see 섹션 11)

4. **[OPEN]** 커스텀 키 리맵핑 지원 여부. 접근성 측면에서 권장되나 구현 복잡도가 높다. MVP 이후 고려 가능. (-> see 섹션 10.3)

5. **[OPEN]** 미니맵 표시 여부. 현재 HUD에 미니맵이 포함되지 않았으나, 농장 확장 시 전체 맵 파악 수단이 필요할 수 있다. 튜토리얼에서는 하나 위치를 "미니맵에 핀"으로 안내하는 언급이 있다 (-> see `docs/systems/tutorial-system.md` 섹션 2.1 S03). 미니맵 도입 시 HUD 좌측 상단 또는 우측 하단에 배치.

## Risks [RISK]

1. **[RISK]** HUD 요소가 로우폴리 게임 화면을 과도하게 가릴 수 있다. 특히 저해상도(1280x720 이하)에서 HUD 면적 비율이 커진다. 최소 해상도를 1280x720으로 제한하고, 해상도별 HUD 스케일링 테스트 필요.

2. **[RISK]** 알림 시스템의 우선순위 큐가 레벨업 + 업적 달성 + 퀘스트 완료가 동시에 발생하는 상황(예: 첫 수확 시)에서 알림이 연쇄 폭주할 수 있다. 동시 발생 알림의 병합/지연 정책을 플레이테스트에서 검증해야 한다.

3. **[RISK]** 통합 패널의 탭 수가 현재 4개(인벤토리/퀘스트/도전/업적)이지만, 추후 시스템 추가(도감, 맵, 관계도 등)로 탭이 늘어나면 수평 탭 바가 화면 너비를 초과할 수 있다. 탭 5개 이상 시 스크롤 또는 아이콘 전환 방식 검토 필요.

4. **[RISK]** 색약 모드의 대체 색상이 실제 색약 사용자에게 충분히 구분되는지는 플레이테스트 없이 검증 불가. 색약 시뮬레이션 도구로 사전 검증 권장.
