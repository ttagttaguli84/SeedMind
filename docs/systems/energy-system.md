# 에너지 시스템 (Energy System) 상세 설계

> 작성: Claude Code (Sonnet 4.6) | 2026-04-08
> 문서 ID: DES-024

## Context

이 문서는 SeedMind 에너지 시스템의 canonical 문서이다. 에너지(Energy)는 플레이어가 하루 동안 취할 수 있는 행동량을 제한하는 핵심 자원으로, 플레이어가 매일 어떤 활동에 집중할지 선택하도록 강제하는 시간 관리 레이어다.

**이전 상태**: 에너지 소모 규칙이 다음 문서들에 분산 기재되어 있었다.
- `docs/systems/farming-system.md` 섹션 3.2 — 최대 에너지 100, 회복 규칙, 경작 도구별 소모
- `docs/systems/fishing-system.md` 섹션 2.3 — 낚시 캐스팅 3 (BAL-024 확정: Lv.8+ 2로 감소)
- `docs/systems/gathering-system.md` 섹션 1.3 — 맨손 0, 채집 낫 1
- `docs/systems/time-season.md` 섹션 1.3, 1.5 — 야간 가중, 수면/기절 회복

**이 문서의 역할**: 위 문서에 분산된 에너지 규칙을 이 문서로 canonical 이전한다. 기존 분산 문서들의 참조 표기는 후속 FIX 작업에서 처리한다.

**이 문서가 canonical인 데이터**:
- 최대 에너지 기본값 및 증가 방식
- 행동별 에너지 소모량 통합 테이블
- 에너지 회복 방식 (수면, 음식, 기타)
- 에너지 고갈 패널티
- 에너지 UI 표시 방식
- 날씨별 에너지 소모 배수
- 에너지 관련 튜닝 파라미터

## 1. 최대 에너지 (Max Energy)

### 1.1 기본값

| 파라미터 | 기본값 | 설명 |
|----------|--------|------|
| `maxEnergy` | 100 | 게임 시작 시 플레이어 최대 에너지 |
| `startOfDayEnergy` | 100 | 정상 수면 후 시작 시 에너지 (maxEnergy와 동일) |

### 1.2 최대 에너지 증가 방식

최대 에너지는 다음 두 가지 경로로 증가할 수 있다.

| 경로 | 증가량 | 조건 |
|------|--------|------|
| 숙련도 레벨업 보너스 | +5 per 레벨 (레벨 5, 10, 15, 20) | 해당 레벨 도달 시 영구 증가 |
| 특수 음식 섭취 | 해당 날 한정 임시 증가 (+20 상한) | 음식 아이템 효과 (섹션 3.2 참조) |

**레벨업 보너스 적용 레벨 및 총 최대 에너지**:

| 플레이어 레벨 | 최대 에너지 누적 |
|--------------|----------------|
| 1~4 | 100 |
| 5 | 105 |
| 10 | 110 |
| 15 | 115 |
| 20 | 120 |

[OPEN] 레벨업 보너스 적용 레벨(5/10/15/20)은 BAL 확정 전 잠정 설계값이다. 진행 곡선 검증(-> see `docs/balance/progression-curve.md`) 이후 조정될 수 있다.

## 3. 날씨별 에너지 소모 배수

날씨에 따라 도구 사용 시 에너지 소모에 배수가 적용된다. 배수는 최종 소모량에 곱해서 올림 처리(ceil)한다.

| 날씨 | 영문 키 | 에너지 배수 | 예시 (호미 기본 2) |
|------|---------|-----------|-----------------|
| 맑음 | `Clear` | x1.0 | 2 |
| 흐림 | `Cloudy` | x1.0 | 2 |
| 비 | `Rain` | x1.0 | 2 |
| 폭우 | `HeavyRain` | x1.1 | 3 (ceil(2.2)) |
| 폭풍 | `Storm` | x1.25 | 3 (ceil(2.5)) |
| 눈 | `Snow` | x1.1 | 3 (ceil(2.2)) |
| 폭설 | `Blizzard` | x1.3 | 3 (ceil(2.6)) |

**파라미터**:

| 파라미터 | 기본값 | 설명 |
|----------|--------|------|
| `heavyRainEnergyMult` | 1.1 | 폭우 시 에너지 소모 배수 |
| `stormEnergyMult` | 1.25 | 폭풍 시 에너지 소모 배수 |
| `snowEnergyMult` | 1.1 | 눈 시 에너지 소모 배수 |
| `blizzardEnergyMult` | 1.3 | 폭설 시 에너지 소모 배수 |

## 5. 에너지 회복 방식

### 5.1 수면 회복 (Sleep Recovery)

수면은 에너지를 회복하는 주요 경로다.

| 수면 방식 | 수면 시각 | 에너지 회복량 | 보너스 |
|-----------|-----------|--------------|--------|
| 침대 수면 (조기) | 20:00 이전 | 100% + 잔여 에너지의 10% 추가 (최대 maxEnergy + 10) | "숙면" 상태: 다음 날 오전(Morning 시간대) 에너지 소모 -10% |
| 침대 수면 (일반) | 20:00~22:00 | 100% (maxEnergy까지) | 없음 |
| 침대 수면 (늦음) | 22:00~24:00 | 100% (maxEnergy까지) | 없음 |
| 기절 (자동 종료) | 24:00 | 50% (maxEnergy × 0.5) | 골드 페널티 적용 (→ 섹션 6.2) |

**"숙면" 상태 상세**:
- 적용 시간대: 다음 날 Morning(08:00~12:00)
- 효과: 해당 시간대 도구 사용 에너지 소모 -10% (소수점 내림 처리)
- 예시: 호미 기본 2 → 1 (floor(2 × 0.9) = 1)
- 지속: Morning 종료(12:00)까지. 12:00 이후 자동 해제

**파라미터**:

| 파라미터 | 기본값 | 설명 |
|----------|--------|------|
| `sleepFullRecovery` | 1.0 | 침대 수면 기본 회복률 |
| `sleepEarlyBonusEnergy` | 10 | 조기 수면 추가 에너지 보너스 상한 |
| `sleepEarlyBonusMultiplier` | 0.1 | 조기 수면 추가 보너스 배율 (잔여 에너지 × 이 값) |
| `sleepEarlyMorningDiscount` | 0.1 | 숙면 상태 오전 에너지 소모 할인율 |
| `passOutEnergyRecovery` | 0.5 | 기절 시 에너지 회복 비율 (50%) |

### 5.2 음식 회복 (Food Recovery)

음식 아이템 섭취로 즉각적인 에너지 회복이 가능하다.

| 음식 등급 | 예시 아이템 | 즉시 회복량 | 특수 효과 | 비고 |
|-----------|------------|------------|-----------|------|
| 기본 음식 | 야생 과일, 버섯 | +10~15 | 없음 | 채집물 원물 섭취 |
| 일반 요리 | 샐러드, 구운 생선 | +20~30 | 없음 | 요리 시설 산출물 |
| 고급 요리 | 특제 스튜, 축제 음식 | +40~50 | 임시 maxEnergy +20 (해당 날) | 희귀 레시피 |
| 최고급 요리 | 왕실 만찬 등 | +60 | 임시 maxEnergy +20 + 속도 보너스 | [OPEN] 요리 시스템 설계 후 확정 |

**음식 섭취 규칙**:
- maxEnergy를 초과하는 회복은 임시 최대 에너지 증가 형태로 처리 (고급 이상 음식에 한해)
- 기본/일반 음식: maxEnergy 상한까지만 회복
- 임시 maxEnergy 증가: 해당 날(다음 수면 전)까지 유효, 최대 +20
- 같은 날 여러 음식 섭취 시 임시 최대치는 누적되지 않고 최대 +20으로 상한 고정

[OPEN] 음식 아이템 구체 목록, 레시피, 각 음식의 정확한 회복량은 요리 시스템 문서(미작성) 및 콘텐츠 문서에서 확정 예정. 본 섹션의 수치는 설계 방향성 및 등급 체계만 확정한 것이다.

### 5.3 기타 회복 경로

| 경로 | 회복량 | 조건 | 비고 |
|------|--------|------|------|
| 집 내부 휴식 | +20 | 침대 외 집 가구 인근 대기 (게임 시간 30분 경과) | [OPEN] 휴식 기능 추가 여부 미확정 |
| 온천 시설 | +30 | 온천 시설 해금 후 이용 (게임 시간 1시간 소비) | [OPEN] 온천 시설 설계 미확정 |
| 이벤트 보상 | 이벤트별 상이 | 축제, 퀘스트 완료 | [OPEN] 이벤트 시스템 설계 후 확정 |

## 6. 에너지 고갈 (Energy Depletion)

### 6.1 에너지 부족 / 고갈 행동 제한

| 에너지 상태 | 임계값 | 게임플레이 영향 |
|------------|--------|--------------|
| 충분 | > 20 | 모든 행동 가능 |
| 경고 | ≤ 20 | UI 경고 표시 (에너지 바 빨간색 펄스), 모든 행동 가능 |
| 고갈 | 0 | 도구 사용 불가, 이동 속도 50% 감소 |

**에너지 0 상태 상세**:
- 도구 사용 시도 시: "에너지가 부족합니다" 피드백 표시, 도구 동작 실행하지 않음 (에너지 소모 없음)
- 이동은 가능하되 속도 50% 감소
- NPC 대화, 상점 이용은 정상 작동
- 에너지 소모 없는 행동(씨앗 심기, 손 상호작용)은 에너지 0에서도 사용 가능

**파라미터**:

| 파라미터 | 기본값 | 설명 |
|----------|--------|------|
| `energyWarningThreshold` | 20 | 경고 UI 표시 임계값 |
| `energyDepletionSpeedPenalty` | 0.5 | 에너지 0 시 이동 속도 배수 |

### 6.2 기절 페널티 (Pass-out Penalty)

24:00까지 침대 수면을 하지 않으면 기절(pass-out) 처리된다.

| 항목 | 내용 |
|------|------|
| 에너지 회복 | maxEnergy × 50% |
| 골드 손실 | 소지 골드의 5% (최소 0G, 최대 500G) |
| 시작 위치 | 다음 날 06:00 집 앞 |
| 연출 | 화면 페이드아웃 (1초) |

**파라미터**:

| 파라미터 | 기본값 | 조정 범위 | 설명 |
|----------|--------|-----------|------|
| `passOutGoldPenaltyRate` | 0.05 | 0.02~0.10 | 기절 골드 손실 비율 |
| `passOutGoldPenaltyCap` | 500 | 100~1000 | 기절 골드 손실 상한 |

## 7. 에너지 UI 표시 방식

에너지 바의 시각적 표현 상세는 `docs/systems/ui-system.md` 섹션 [E]가 canonical이다. 이 섹션은 에너지 시스템 관점의 UI 연동 요건을 정의한다.

### 7.1 HUD 에너지 바

| 요소 | 스펙 |
|------|------|
| 위치 | 좌측 하단 HUD |
| 형태 | 녹색 수평 게이지 바 |
| 경고 색상 | 에너지 ≤ 20 (energyWarningThreshold) 시 빨간색으로 변경 + 펄스 애니메이션 |
| 소모 피드백 | 도구 사용 시 바에서 소모량만큼 즉시 감소 + 소모 수치 플로팅 텍스트 |
| 숫자 표시 | 현재치 / 최대치 (예: "73 / 100") |

### 7.2 상태 이상 표시

| 상태 | UI 표시 |
|------|---------|
| 에너지 고갈 (0) | 에너지 바 전체 빨간색 + "완전히 지쳤습니다!" 메시지 |
| 숙면 보너스 활성 | 아침 시간대 에너지 바 위에 별빛 아이콘 |
| 임시 최대 에너지 증가 | 에너지 바가 maxEnergy 기준선을 초과하여 연장 표시 (다른 색상, 예: 황금색) |

## 8. 설계 의도 요약

### 8.1 활동별 에너지 포지셔닝

| 활동 | 에너지 소모/회 | 포지셔닝 |
|------|--------------|---------|
| 경작 (호미, Basic) | 2 | 초반 핵심 행동, 비용 부담 있음 |
| 물주기 (Basic) | 1 | 매일 필수, 낮은 비용으로 지속 가능 |
| 수확 (Basic) | 1 | 보상 행위, 낮은 비용 |
| 낚시 캐스팅 (Lv.1~7) | 3~4 | "집중 보상" 활동, 비용 높음 |
| 낚시 캐스팅 (Lv.8+) | 2~3 | 숙련자 효율 보상 |
| 채집 (맨손) | 0 | "빈 시간 활용" 활동, 에너지 무관 |
| 채집 (도구) | 1 | 도구 투자의 소폭 비용 대가 |

**하루 에너지 100 기준 활동 시뮬레이션 예시** (기본 도구, 농장 20타일):
- 경작(신규 타일 5개): 2×5 = 10
- 물주기(20타일): 1×20 = 20
- 수확(10타일): 1×10 = 10
- 낚시(10회 시도): 3×10 = 30
- 남은 에너지: 30 (채집 30회 혹은 추가 활동 가능)

### 8.2 에너지 시스템의 설계 목적

1. **일일 시간 관리 압박**: 에너지 100은 모든 활동을 다 하기에 항상 부족하게 설계 → 매일 우선순위 선택 강제
2. **다양한 활동 균형**: 낚시는 에너지 비용이 높아 채집/경작과 상호 배타적 경쟁 관계 형성
3. **숙련 보상**: 도구 업그레이드와 낚시 레벨업이 에너지 효율을 높여 성장감 제공
4. **채집의 차별화**: 에너지 0 소모로 "언제든 가능한 보조 활동" 포지션 확보
5. **리듬 강화**: 수면 시간 선택(조기/일반/기절)이 다음 날 효율에 영향 → 하루 마무리 결정도 의미 있게

## Risks

1. [RISK] **에너지 고갈로 인한 플레이 차단**: 에너지 0 도달 시 도구 사용 불가 + 이동 속도 50% 감소는 경험 많은 플레이어에게 강한 페널티. 초보 플레이어가 에너지 관리 실패 시 "아무것도 못 하는 시간"이 발생해 지루함 유발 가능. 완화: 에너지 경고(≤20)를 명확히 표시하고, 튜토리얼에서 수면/회복 메커니즘을 조기 교육.
2. [RISK] **낚시의 에너지 대비 효율 우위**: 낚시(에너지 3~4/회, 수익 ~39G)는 물주기(에너지 1/회, 간접 수익)보다 에너지당 직접 수익이 높아 보임. 그러나 물주기는 미래 수확을 위한 투자이므로 단순 비교는 부적절. 장기 시뮬레이션으로 활동 간 균형 검증 필요.
3. [RISK] **날씨 에너지 배수의 체감**: 폭설 x1.3 배수가 수치상으론 크지 않지만 체감 차이가 적을 수 있음. 악천후 조건에서 활동 억제 의도가 불명확해질 위험. 날씨별 에너지 소모 배수를 더 강하게 설정하거나, 이동 속도 감소 등 복합 페널티로 보완 검토.
4. [RISK] **숙면 보너스 복잡도**: 조기 수면 → 숙면 상태 → 오전 에너지 할인 10%의 연쇄 효과가 플레이어에게 명확히 전달되지 않을 경우 시스템이 "있는지도 모르는 기능"이 될 수 있음. UI에서 숙면 상태를 명확히 표시할 것(에너지 바 별빛 아이콘 등).
