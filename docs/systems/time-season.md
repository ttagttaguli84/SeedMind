## Context

이 문서는 SeedMind의 시간 흐름, 계절 순환, 날씨 시스템, 연간 이벤트를 상세히 기술한다. 마스터 디자인 문서(`docs/design.md`) 섹션 4.3에 정의된 시간/계절 개요, `docs/architecture.md` 섹션 4.3의 TimeManager 구조, `docs/systems/crop-growth.md`의 계절 성장 보정, `docs/systems/farming-system.md`의 날씨 연동을 기반으로 확장한다.

**설계 목표**: 시간과 계절이 플레이어에게 자연스러운 리듬과 긴장감을 동시에 부여한다. 하루 안에서는 우선순위 선택의 압박을, 계절 단위에서는 작물 계획의 전략을, 연 단위에서는 장기 목표 달성의 보람을 느끼게 한다.

## 1. 하루 흐름 (Day Cycle)

### 1.1 시간 기본 규칙

(-> see `docs/design.md` 섹션 4.3, `docs/architecture.md` 섹션 4.3 for canonical values)

| 파라미터 | 값 |
|----------|-----|
| 하루 시작 | 06:00 |
| 하루 종료 | 24:00 |
| 게임 내 1시간 | 실시간 약 33초 |
| 게임 내 1일 | 실시간 약 10분 |
| 활동 가능 시간 | 18시간 (06:00~24:00) |

### 1.2 시간대 구분

하루를 5개 시간대로 나누며, 각 시간대는 조명, NPC 행동, 게임플레이에 영향을 준다.

| 시간대 | 영문 키 | 시간 범위 | 실시간 길이 | 조명/색감 |
|--------|---------|-----------|------------|-----------|
| 새벽 | `Dawn` | 06:00~08:00 | ~66초 | 부드러운 주황-분홍, 긴 그림자, 안개 효과 |
| 아침 | `Morning` | 08:00~12:00 | ~132초 | 밝은 황금빛, 따뜻한 톤 |
| 낮 | `Afternoon` | 12:00~17:00 | ~165초 | 밝은 백색광, 가장 선명한 색감 |
| 저녁 | `Evening` | 17:00~20:00 | ~99초 | 주황-빨강 석양, 그림자 길어짐 |
| 밤 | `Night` | 20:00~24:00 | ~132초 | 짙은 파란색, 별빛, 시야 범위 축소 |

### 1.3 시간대별 게임플레이 영향

| 시간대 | 영향 |
|--------|------|
| 새벽 (Dawn) | 매일 06:00 배치 처리 실행: 물 증발, 성장 계산, 날씨 전환. NPC 대부분 미등장. 상점 미오픈 |
| 아침 (Morning) | NPC 등장 시작, 상점 오픈(08:00). 물주기/농작업 최적 시간대 |
| 낮 (Afternoon) | 모든 NPC 활동, 모든 상점 운영. 에너지 소모 기본값 |
| 저녁 (Evening) | 일부 상점 마감(18:00). TV/게시판에서 내일 날씨 예보 확인 가능(18:00~) |
| 밤 (Night) | 대부분 상점 마감. 에너지 소모 +20% 가중. 시야 범위 축소(화면 가장자리 어둡게). 20:00 이후 피로 경고 |

### 1.4 시간 경과 메커니즘

| 기능 | 설명 |
|------|------|
| 기본 흐름 | `timeScale = 1.0`에서 자동 진행 |
| 시간 가속 | 특정 상황에서 자동 가속 (잠자기, 가공 대기 등) |
| 일시정지 | 메뉴(Esc), 인벤토리(Tab), 상점 UI 열람 시 시간 정지 |
| 수동 가속 없음 | 플레이어가 임의로 시간을 빠르게 넘길 수 없음 (잠자기만 가능) |

**설계 의도**: 수동 가속을 금지하여 "시간이 부족한 긴장감"을 유지한다. 모든 활동에 시간 비용이 있어야 우선순위 선택이 의미를 가진다.

### 1.5 하루 종료 프로세스

24:00 도달 시 또는 침대에서 잠자기 선택 시 하루가 종료된다.

**24:00 자동 종료 (기절)**:
1. 플레이어가 24:00까지 침대에 가지 않으면 현재 위치에서 기절
2. 화면 페이드아웃 (1초)
3. 에너지 50%만 회복 (수면 회복의 절반)
4. 기절 페널티: 소지 골드의 5% 손실 (최대 500G, 최소 0G)
5. 다음 날 06:00에 집 앞에서 시작

**침대 수면 (정상 종료)**:
1. 플레이어가 침대와 상호작용 → "잠자기" 선택
2. 시간 가속 (현재 시각 → 24:00 → 다음 날 06:00)
3. 화면 페이드아웃 + 날짜 전환 연출
4. 에너지 100% 회복
5. 다음 날 06:00에 집 내부에서 시작
6. 수면 시각에 따른 보너스:

| 수면 시각 | 에너지 회복 | 보너스 |
|-----------|------------|--------|
| 20:00 이전 | 100% + 잔여 에너지의 10% 추가 (최대 110) | "숙면" 상태: 다음 날 오전 에너지 소모 -10% |
| 20:00~22:00 | 100% | 없음 (기본) |
| 22:00~24:00 | 100% | 없음 |
| 24:00 (기절) | 50% | 골드 페널티 |

### 1.6 날짜 전환 배치 처리 순서

매일 06:00(하루 시작)에 다음 순서로 시스템이 처리된다:

```
1. 날짜 카운터 증가 (currentDay + 1)
2. 계절 전환 체크 (Day 28 → Day 1, Season 전환)
3. 날씨 결정 (오늘의 날씨 확정)
4. 비 효과 적용 (Rain/HeavyRain → 모든 야외 경작 타일 자동 Watered)
5. 작물 성장 계산 (Watered 타일 성장일수 누적)
6. 물 증발 처리 (모든 Watered → Dry)
7. 고사 판정 (연속 Dry 3일 → Withered)
8. 경작지 복귀 체크 (씨앗 없이 Tilled 3일 → Empty)
9. NPC 스케줄 로드
10. 이벤트 체크 (축제, 특수 이벤트 날짜)
11. OnDayChanged 이벤트 발행
```

### 1.7 NPC/상점 스케줄

| 시설/NPC | 운영 시간 | 휴무 |
|----------|-----------|------|
| 잡화 상점 (씨앗/비료) | 08:00~18:00 | 매주 수요일 (Day 3, 10, 17, 24) |
| 대장장이 (도구 업그레이드) | 10:00~16:00 | 매주 금요일 (Day 5, 12, 19, 26) |
| 목공소 (시설 건설) | 09:00~17:00 | 매주 화요일 (Day 2, 9, 16, 23) |
| 출하함 (작물 판매) | 24시간 | 없음 (언제든 투입 가능, 정산은 다음 날 06:00) |

**요일 시스템**: 28일 = 4주. 1주 = 7일. Day 1 = 월요일, Day 7 = 일요일 순환.

| 요일 | 영문 키 | Day 번호 |
|------|---------|---------|
| 월요일 | `Monday` | 1, 8, 15, 22 |
| 화요일 | `Tuesday` | 2, 9, 16, 23 |
| 수요일 | `Wednesday` | 3, 10, 17, 24 |
| 목요일 | `Thursday` | 4, 11, 18, 25 |
| 금요일 | `Friday` | 5, 12, 19, 26 |
| 토요일 | `Saturday` | 6, 13, 20, 27 |
| 일요일 | `Sunday` | 7, 14, 21, 28 |

[OPEN] NPC/상점 목록이 확정되지 않았다 (-> see `docs/content/npcs.md`, 작성 예정 CON-003). 상점 종류와 운영 시간은 NPC 콘텐츠 확정 후 조정 필요.

## 2. 계절 시스템 상세 (Season System)

### 2.1 기본 규칙

(-> see `docs/design.md` 섹션 4.3 for canonical values)

| 파라미터 | 값 |
|----------|-----|
| 1계절 | 28일 |
| 1년 | 4계절 (봄 → 여름 → 가을 → 겨울) |
| 1년 총 일수 | 112일 |
| 1년 실시간 | 약 18.7시간 |

### 2.2 계절별 특성 표

| 항목 | 봄 (Spring) | 여름 (Summer) | 가을 (Autumn) | 겨울 (Winter) |
|------|-------------|---------------|---------------|---------------|
| **테마** | 시작과 성장 | 풍요와 도전 | 수확과 준비 | 휴식과 계획 |
| **재배 가능 작물** | (-> see `docs/systems/crop-growth.md` 섹션 3.1) | (-> see 동일) | (-> see 동일) | 야외 불가, 온실만 가능 |
| **성장 보너스** | 없음 (기본) | 전 작물 x1.05 더위 보너스 (-> see `docs/systems/crop-growth.md` 섹션 2.4) | 없음 (기본) | 야외 성장 불가 |
| **주요 날씨** | 맑음 50%, 비 25% | 맑음 60%, 폭풍 10% | 흐림 25%, 비 25% | 눈 35%, 맑음 30% |
| **환경 색감** | 연두+분홍, 벚꽃 파티클 | 짙은 녹색+선명, 매미 사운드 | 주황+갈색, 낙엽 파티클 | 흰색+파란, 눈 파티클 |
| **특수 이벤트** | 봄 씨앗 축제 (Day 13) | 여름 불꽃 축제 (Day 21) | 수확 축제 (Day 21) | 겨울 별빛 축제 (Day 25) |
| **고유 메커닉** | 없음 | 폭풍 빈도 증가 (10%) | 수확물 가격 +10% (수확철) | 야외 경작 불가, 채집 불가 (낚시는 얼음 낚시로 가능 → see docs/systems/fishing-system.md 섹션 10) |
| **NPC 행동** | 기본 패턴 | 더위 대사 추가 | 축제 준비 행동 | 실내 활동 위주 |

### 2.3 계절별 환경 변화

각 계절은 시각적/청각적으로 뚜렷하게 구분되어야 한다.

#### 조명/색감

| 계절 | Directional Light 색온도 | Ambient Color | Post-Processing |
|------|------------------------|---------------|-----------------|
| 봄 | 따뜻한 백색 (5500K) | 연한 연두 (#E8F5E9) | Bloom 약하게 |
| 여름 | 뜨거운 백색 (6000K) | 선명한 하늘색 (#E3F2FD) | Bloom 강하게, Color Temperature +0.1 |
| 가을 | 따뜻한 주황 (4500K) | 짙은 주황 (#FFF3E0) | Vignette 약하게 |
| 겨울 | 차가운 백색 (7500K) | 연한 파란 (#E8EAF6) | Color Temperature -0.15, Bloom 약하게 |

#### 환경 오브젝트 변화

| 오브젝트 | 봄 | 여름 | 가을 | 겨울 |
|----------|-----|------|------|------|
| 나무 | 연두색 잎 + 벚꽃 | 짙은 녹색 잎 | 주황/빨강 잎 | 잎 없음 (앙상한 가지) |
| 풀/잔디 | 연두색, 꽃 스폰 | 짙은 녹색 | 갈색으로 변화 | 눈 덮임 |
| 땅 텍스처 | 습한 갈색 | 건조한 갈색 | 짙은 갈색 | 눈 덮인 흰색 |
| 물(연못/강) | 기본 | 기본 | 기본 + 낙엽 부유 | 표면 얼음 효과 |
| 파티클 | 벚꽃잎 날림 | 없음 (아지랑이 옵션) | 낙엽 날림 | 눈 내림 |

#### 환경 사운드

| 계절 | 앰비언트 사운드 |
|------|----------------|
| 봄 | 새소리 (활발), 가벼운 바람 |
| 여름 | 매미 소리, 강한 바람 |
| 가을 | 귀뚜라미, 바스락 낙엽 |
| 겨울 | 바람 소리 (쓸쓸), 눈 밟는 소리 (이동 시) |

### 2.4 계절 전환 메커니즘

계절 전환은 Day 28 24:00에 발생한다. 처리 순서는 다음과 같다:

```
계절 전환 순서 (Day 28 24:00 → 새 계절 Day 1 06:00):

1. 수면/기절 처리 (일반 하루 종료와 동일)
2. currentSeason 변경 (Spring → Summer → Autumn → Winter → Spring)
3. 겨울 → 봄일 경우: currentYear + 1, OnYearChanged 발행
4. 계절 불일치 작물 즉시 Withered 전환 (야외만, 온실 제외)
   - Withered 전환 시 화면에 "계절이 바뀌었습니다. [N]개의 작물이 고사했습니다" 알림
5. 환경 오브젝트 교체 (나무, 풀, 땅 텍스처, 파티클)
6. 조명/Post-Processing 프리셋 전환
7. 앰비언트 사운드 교체
8. 상점 인벤토리 갱신 (새 계절 씨앗 추가, 이전 계절 씨앗 제거)
9. OnSeasonChanged 이벤트 발행
10. 일반 날짜 전환 배치 처리 (섹션 1.6 참조, Day 1 06:00)
```

### 2.5 계절 전환 사전 경고 시스템

플레이어가 계절 전환에 대비할 수 있도록 단계적 경고를 제공한다.

| 시점 | 경고 방식 | 내용 |
|------|-----------|------|
| Day 21 (7일 전) | TV/게시판 메시지 | "다음 계절까지 7일 남았습니다" |
| Day 25 (3일 전) | 화면 상단 배너 (노란색) | "[계절명]이 3일 후 시작됩니다. 현재 계절 작물을 수확하세요!" |
| Day 25~28 | 미수확 작물 타일 아이콘 | 해당 타일에 노란 삼각형 경고 아이콘 표시 |
| Day 27 (1일 전) | 화면 상단 배너 (빨간색) | "내일 [계절명]이 시작됩니다! 미수확 작물: [N]개" |
| Day 28 잠자기 시 | 확인 팝업 | "오늘이 [현재 계절]의 마지막 날입니다. 미수확 작물 [N]개가 고사합니다. 잠자겠습니까?" |

(-> see `docs/systems/crop-growth.md` 섹션 3.2의 사전 경고 기본 규칙)

## 3. 날씨 시스템 상세 (Weather System)

### 3.1 날씨 종류

| 날씨 | 영문 키 | 아이콘 | 설명 |
|------|---------|--------|------|
| 맑음 | `Clear` | 태양 | 기본 날씨, 특별한 효과 없음 |
| 흐림 | `Cloudy` | 구름 | 시각적 변화만 (약간 어두움), 게임플레이 영향 없음 |
| 비 | `Rain` | 물방울 | 야외 경작 타일 자동 물주기 |
| 폭우 | `HeavyRain` | 물방울 2개 | 자동 물주기 + 이동속도 감소 |
| 폭풍 | `Storm` | 번개 | 자동 물주기 + 작물 피해 + 이동속도 감소 + 에너지 소모 증가 |
| 눈 | `Snow` | 눈결정 | 야외 작물 성장 정지, 이동속도 감소 |
| 폭설 | `Blizzard` | 눈결정 2개 | 성장 정지 + 이동속도 크게 감소 + 시야 축소 |

### 3.2 계절별 날씨 확률 테이블

날씨는 하루 단위로 결정된다. 매일 06:00에 그날의 날씨가 확정된다.

| 날씨 | 봄 (%) | 여름 (%) | 가을 (%) | 겨울 (%) |
|------|--------|----------|----------|----------|
| Clear | 40 | 45 | 30 | 15 |
| Cloudy | 20 | 15 | 20 | 15 |
| Rain | 25 | 20 | 25 | 0 |
| HeavyRain | 10 | 5 | 15 | 0 |
| Storm | 5 | 15 | 10 | 0 |
| Snow | 0 | 0 | 0 | 35 |
| Blizzard | 0 | 0 | 0 | 35 |

**합계**: 각 계절 100%.

**참고**: `docs/systems/farming-system.md` 섹션 8.1에 5종 날씨(Sunny/Cloudy/Rainy/Storm/Snow) 확률이 기재되어 있으나, 이 문서에서 7종으로 확장했다. farming-system.md의 확률 테이블은 초기 개요이며, **이 문서(time-season.md) 섹션 3.2가 날씨 확률의 canonical source**이다.

### 3.3 날씨 결정 알고리즘

```
매일 06:00 WeatherSystem.DetermineWeather():
  1. 현재 계절의 확률 테이블 로드
  2. 전날 날씨 참조 (연속 날씨 제한)
  3. 가중 랜덤 선택
  4. 연속 규칙 적용:
     - 같은 날씨 최대 3일 연속 (4일째에는 강제로 다른 날씨)
     - Storm/Blizzard는 최대 2일 연속
  5. 결과를 currentWeather에 저장
  6. 내일 날씨 미리 결정 (예보용): tomorrowWeather에 저장
```

### 3.4 날씨의 게임플레이 영향 상세

#### 농작 영향

| 날씨 | 자동 물주기 | 성장 영향 | 작물 피해 |

| Snow | X | 야외 성장 정지 | 없음 |

#### 이동속도 영향

| Rain | x1.0 |
| HeavyRain | x0.85 |
| Storm | x0.75 |

| Blizzard | x0.7 |

#### 에너지 소모 영향

#### NPC 행동 영향

### 3.5 날씨 예보 메커니즘

플레이어는 내일 날씨를 사전에 확인할 수 있다.

| 수단 | 이용 시간 | 정확도 | 비용 | 정보 범위 |
|------|-----------|--------|------|-----------|
| TV (집 내부) | 18:00~24:00 | 100% | 무료 | 내일 날씨 |
| 마을 게시판 | 06:00~24:00 | 100% | 무료 | 내일 날씨 |
| 기상 달력 (시설) | 언제든 | 100% | 2000G (설치비) | 앞으로 3일 날씨 |

**TV 예보 텍스트 예시**:
- "내일은 맑은 하루가 될 것 같습니다. 농작업 하기 좋은 날이네요!"
- "내일은 비 소식이 있습니다. 물뿌리개를 쉴 수 있겠군요."
- "내일은 폭풍이 예상됩니다. 외출을 삼가세요!"

### 3.6 날씨 지속 시간

- **기본 단위**: 하루 단위. 06:00에 결정된 날씨가 다음 06:00까지 유지된다.
- **시간대 내 변동 없음**: 한 날 안에서 맑음→비로 바뀌지 않는다.
- **설계 의도**: 하루 계획을 세우는 데 날씨가 명확한 입력값이 되어야 한다. 시간 내 변동은 예측 불가능성만 높이고 전략적 깊이를 더하지 않는다.

### 3.7 날씨 시각/청각 효과

| 날씨 | 시각 효과 | 청각 효과 |
|------|----------|----------|
| Clear | 밝은 조명, 렌즈 플레어 | 새소리, 바람 |
| Cloudy | 밝기 -15%, 그림자 소프트 | 약한 바람 |
| Rain | 빗줄기 파티클, 웅덩이 생성 | 빗소리 (지속) |
| HeavyRain | 굵은 빗줄기, 시야 약간 흐림 | 강한 빗소리 |
| Storm | 굵은 빗줄기 + 번개 섬광 (랜덤 간격) | 강한 빗소리 + 천둥 |
| Snow | 눈 파티클, 지면 점진적 백색화 | 고요함, 약한 바람 |
| Blizzard | 강한 눈 파티클, 시야 50% 축소 | 강한 바람 소리 |

## 4. 연간 이벤트/축제 (Annual Events)

### 4.1 축제 개요

각 계절에 1개의 축제가 열린다. 축제일에는 특별한 활동과 보상이 제공되며, 일반 농작업도 가능하다 (선택).

### 4.2 축제 상세

#### 봄 씨앗 축제 (Spring Seed Festival)
| 항목 | 내용 |
|------|------|
| 날짜 | 봄 Day 13 (토요일) |
| 장소 | 마을 광장 |
| 시간 | 10:00~18:00 (참여 선택) |
| 활동 | 씨앗 교환: 플레이어가 보유한 작물/씨앗을 다른 NPC의 희귀 씨앗과 교환 |
| 보상 | 일반 상점에서 팔지 않는 특수 씨앗 1종 획득 기회 |
| 조건 | 1년차부터 참여 가능, 교환에 작물 최소 5개 필요 |
| 불참 시 | 일반 하루로 진행 (페널티 없음) |

#### 여름 불꽃 축제 (Summer Fireworks Festival)
| 항목 | 내용 |
|------|------|
| 날짜 | 여름 Day 21 (일요일) |
| 장소 | 해변/호숫가 |
| 시간 | 18:00~24:00 (야간 이벤트) |
| 활동 | 불꽃놀이 감상 + 미니게임 (불꽃 맞추기 타이밍 게임) |
| 보상 | 미니게임 성적에 따라: 1등 500G, 2등 300G, 3등 100G |
| 조건 | 1년차부터 참여 가능 |
| 불참 시 | 일반 하루로 진행 |

#### 가을 수확 축제 (Autumn Harvest Festival)
| 항목 | 내용 |
|------|------|
| 날짜 | 가을 Day 21 (일요일) |
| 장소 | 마을 광장 |
| 시간 | 10:00~18:00 |
| 활동 | 작물 품평회: 플레이어가 출품한 작물의 품질에 따라 순위 결정 |
| 보상 | 1등: 2000G + 트로피(장식), 2등: 1000G, 3등: 500G. Iridium 품질 출품 시 특별상 1000G 추가 |
| 조건 | 1년차부터 참여 가능, 출품 작물 1개 필요 |
| 불참 시 | 일반 하루로 진행 |

#### 겨울 별빛 축제 (Winter Starlight Festival)
| 항목 | 내용 |
|------|------|
| 날짜 | 겨울 Day 25 (목요일) |
| 장소 | 마을 광장 (야간) |
| 시간 | 18:00~24:00 |
| 활동 | NPC에게 선물 주기 + 별빛 아래 대화 |
| 보상 | 참여만으로 다음 해 봄 특수 씨앗 1개 우편 수령. NPC 호감도 보너스 적용 |
| 조건 | 1년차부터 참여 가능 |
| 불참 시 | 일반 하루로 진행, 특수 씨앗 미수령 |

### 4.3 축제일 특수 규칙

- 축제 시간 동안 마을 광장/축제 장소에 특수 NPC와 장식이 배치됨
- 상점은 축제일에 정상 운영하되, 축제 장소 근처에 임시 노점 추가 가능
- 축제 참여는 선택 사항: 참여하지 않으면 일반 하루로 진행 (농작업 가능)
- 축제 참여 중에도 시간은 흐른다 (축제 종료 후 잔여 시간 활용 가능)

[OPEN] 축제 미니게임의 구체적 설계가 필요하다. 특히 불꽃 축제의 타이밍 게임과 수확 축제의 품평회 판정 로직. 이는 별도 콘텐츠 문서(CON 계열)에서 상세화할 수 있다.

## 6. 플레이어 경험 흐름 (Player Experience Flow)

### 6.1 전형적인 하루 타임라인

```
06:00 [새벽]  - 기상. 날씨 확인 (창문/문 밖 비주얼)
06:00~06:30   - TV로 내일 날씨 예보 확인 (전날 놓쳤다면)
              - 오늘 할 일 계획

07:00~10:00 [아침]  - 물주기 (비 오는 날이면 스킵)
                    - 비료 주기, 씨앗 심기
                    - 수확 가능 작물 수확

10:00~12:00          - 상점 방문 (씨앗 구매, 도구 업그레이드 의뢰)
                     - NPC 대화

12:00~17:00 [낮]    - 추가 농작업 또는 탐험
                    - 시설 건설/관리
                    - 출하함에 작물 투입

17:00~18:00 [저녁]  - 마감 전 상점 막바지 방문
18:00~20:00         - TV/게시판에서 내일 날씨 확인
                    - 남은 에너지로 소규모 작업

20:00~22:00 [밤]    - 집으로 귀환
                    - 침대에서 수면 → 하루 종료
```

**시간 압박 설계**: 10분(실시간)의 하루 중 물주기와 수확에만 3~4분이 소요된다 (경작지 규모에 비례). 나머지 6~7분 동안 상점, 건설, 탐험 중 선택해야 한다. 이 시간 부족이 우선순위 결정을 강제한다.

### 6.2 계절별 리듬

| 시기 | 플레이어 행동 패턴 |
|------|-------------------|
| Day 1~7 (초반) | 새 계절 시작. 씨앗 구매, 밭 정리, 새 작물 심기. 탐색/실험 기간 |
| Day 8~14 (중반 전기) | 작물 관리 루틴 확립. 성장 대기 중 다른 활동 (건설, NPC 교류). 봄 축제(Day 13) |
| Day 15~21 (중반 후기) | 첫 수확 시작 (짧은 작물). 재투자 결정. 여름/가을 축제(Day 21) |
| Day 22~28 (후반) | 긴 작물 수확. 다음 계절 준비. 계절 경고 대응. 겨울 축제(Day 25). 미수확 작물 정리 |

### 6.3 1년 진행 리듬

| 계절 | 경제적 리듬 | 감정적 리듬 |
|------|------------|------------|
| 봄 | 투자 기간. 씨앗 대량 구매, 골드 감소 | 희망, 기대감, 계획하는 재미 |
| 여름 | 수확 시작. 고가 작물(수박 등)로 수입 증가 | 바쁨, 성취감, 폭풍 긴장 |
| 가을 | 최대 수입 기간. 수확 축제 보상 + 가격 보너스 | 풍요, 보람, 겨울 대비 긴장 |
| 겨울 | 수입 격감 (온실만 운영). 시설 투자, 계획 수립 | 차분함, 다음 해 전략 구상, 휴식 |

### 6.4 다년 진행 리듬

| 년차 | 초점 | 변화 |
|------|------|------|
| 1년차 | 기본 학습. 봄 작물 위주. 경제 기반 마련 | 모든 것이 새롭다. 실수 허용 구간 |
| 2년차 | 시설 투자 (물탱크, 온실). 효율 최적화 시작 | 패턴 인식, 계절별 전략 형성 |
| 3년차+ | 가공소 운영. 고급 작물 중심. 농장 완성 단계 | 마스터리 달성, 완벽한 1년 도전 |

## Risks

1. [RISK] **겨울 콘텐츠 공백** (부분 완화): 야외 경작이 불가능한 겨울 28일이 플레이어에게 지루하게 느껴질 수 있다. 온실(16타일)만으로는 활동량이 크게 부족하다. **얼음 낚시 허용(→ see docs/systems/fishing-system.md 섹션 10)으로 야외 활동 1종이 추가되어 부분 완화되었으나**, 광산/요리/NPC 이벤트 등 추가 겨울 콘텐츠 검토는 여전히 필요하다.
2. [RISK] **날씨 7종 확장 리스크**: farming-system.md에서 5종이었던 날씨를 7종으로 확장했다. HeavyRain과 Blizzard 추가로 시각/청각 에셋 제작 부담이 증가한다. MCP로 충분한 품질의 날씨 효과를 구현할 수 있는지 검증 필요.
3. [RISK] **기절 페널티 밸런스**: 골드 5% 손실(최대 500G)이 초보에게 가혹할 수 있으나, 너무 낮으면 수면 관리 동기가 사라진다. 플레이테스트 필요.
4. [RISK] **축제 참여의 기회비용**: 축제에 반나절을 소비하면 농작업 시간이 줄어든다. 축제 보상이 농작업 수익보다 낮으면 참여 동기가 부족하고, 너무 높으면 필수가 되어 선택의 의미가 사라진다.
