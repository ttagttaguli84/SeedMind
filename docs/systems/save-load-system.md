# 세이브/로드 UX 설계 (Save/Load UX Design)

> 세이브 전략, 자동저장 트리거, 수동 저장, 멀티슬롯 구조, 게임 종료/시작 흐름, 저장 데이터 범위, 로드 UX  
> 작성: Claude Code (Opus) | 2026-04-06  
> 문서 ID: DES-008

---

## Context

이 문서는 SeedMind의 세이브/로드 시스템을 **게임 디자이너 관점**에서 설계한다. 플레이어가 언제, 어떻게 게임을 저장하고 불러오는지, 그 과정에서 어떤 UI와 피드백을 경험하는지를 정의한다.

기술적 구현(JSON 직렬화, 파일 I/O, 데이터 파이프라인)은 아키텍처 문서에서 다룬다 (-> see `docs/pipeline/data-pipeline.md` 섹션 3). 본 문서는 UX 흐름과 디자인 의도에 집중한다.

**설계 목표**:
- 농장 시뮬레이션 장르의 관습에 부합하는 자연스러운 세이브 경험
- 데이터 손실 위험을 최소화하면서도, 세이브 남용(save-scumming)을 적절히 억제
- 초보 플레이어도 직관적으로 이해할 수 있는 세이브/로드 UI
- 비정상 종료에도 진행 상실을 최소화하는 안전망

## 1. 세이브 철학

### 1.1 전략 선택: 하이브리드 세이브

SeedMind는 **자동저장 + 제한적 수동 저장**의 하이브리드 방식을 채택한다.

| 방식 | 장점 | 단점 | SeedMind 적합성 |
|------|------|------|----------------|
| 자동저장 전용 (Animal Crossing 방식) | 선택의 무게감 유지, save-scumming 차단 | 실수 복구 불가, 플레이어 불만 | 농장 게임에서는 과도한 긴장감 유발 |
| 자유 수동 저장 (일반 PC 게임) | 플레이어 자유도 극대화 | 결과의 무의미화, 도전의 약화 | save-scumming으로 경제 밸런스 붕괴 가능 |
| **하이브리드 (Stardew Valley 방식)** | 안전망 + 적절한 긴장감 | 구현 복잡도 다소 증가 | **채택** |

### 1.2 핵심 원칙

| 원칙 | 설명 |
|------|------|
| **하루 단위 저장** | 가장 주요한 저장 시점은 하루가 끝나는 수면 시점이다. 하루 동안의 행동은 "의미 있는 단위"로 묶인다 |
| **자동저장으로 안전망** | 주기적 자동저장이 비정상 종료에 대비한다. 플레이어가 저장을 잊어도 최대 손실은 제한된다 |
| **수동 저장은 일시정지 메뉴** | 수동 저장을 허용하되, 메뉴 진입이라는 의식적 행동을 요구하여 남용을 자연스럽게 억제한다 |
| **투명한 상태 표시** | 마지막 저장 시점, 자동저장 발생 여부를 플레이어에게 명확히 알린다 |

### 1.3 Save-Scumming 대응

농장 시뮬레이션은 고위험 도박 요소가 없으므로, save-scumming의 영향이 제한적이다. 다만 다음 요소에서 남용이 발생할 수 있다:

| 남용 가능 영역 | 대응 전략 |
|---------------|----------|
| 날씨 예보 확인 후 리로드 | 내일 날씨는 하루 시작 시 결정, 저장 시점에 이미 확정됨 (-> see `docs/systems/time-season.md` 날씨 시스템) |
| 랜덤 품질 재굴림 | 작물 품질은 파종 시 seed가 결정됨 (-> see `docs/systems/crop-growth.md` 품질 결정 시점) |
| 가격 변동 리로드 | 가격은 계절/수급 기반으로 점진 변동, 단일 저장/로드로 유의미한 차이 없음 (-> see `docs/systems/economy-system.md`) |

결론: 별도의 anti-scumming 메커니즘은 불필요하다. 게임 시스템 자체가 랜덤 결과를 저장 시점에 확정하는 방식으로 자연 억제한다.

## 2. 자동저장 트리거

### 2.1 자동저장 시점

| 트리거 | 조건 | 우선순위 | 비고 |
|--------|------|---------|------|
| **하루 시작** | 매일 06:00, 하루 시작 배치 처리 완료 직후 | 필수 | 가장 주요한 자동저장. 전날의 모든 결과(출하 정산, 작물 성장, 날씨 전환)가 반영된 상태 |
| **수면 실행** | 침대 상호작용으로 수면 진입 시 | 필수 | 수면 연출 시작 전에 저장 완료. 하루 종료의 "마침표" 역할 |
| **주기적 백업** | 실시간 10분마다 (게임 내 1일 주기와 동일) | 권장 | 비정상 종료 대비 안전망. 쿨다운 적용 |
| **중요 이벤트 완료** | 시설 건설 완료, 레벨업, 튜토리얼 시퀀스 완료 | 권장 | 돌이킬 수 없는 진행 보호 |

### 2.2 자동저장 쿨다운

연속적인 이벤트(예: 여러 작물 동시 수확 → 레벨업)로 인한 과도한 저장을 방지한다.

| 파라미터 | 값 | 비고 |
|----------|-----|------|
| 최소 자동저장 간격 | 60초 | 이벤트 트리거 기준 쿨다운 (-> see `docs/systems/save-load-architecture.md` 섹션 3.3) |
| 주기적 백업 간격 | 600초 (실시간 10분) | 게임 내 약 1일 주기 |
| 수면 트리거 | 쿨다운 무시 | 항상 즉시 저장 |
| 하루 시작 트리거 | 쿨다운 무시 | 항상 즉시 저장 |

### 2.3 자동저장 UI 인디케이터

자동저장이 발생하면 플레이어에게 비침습적으로 알린다.

| 요소 | 설명 |
|------|------|
| **아이콘** | 화면 우하단에 작은 "씨앗" 또는 "새싹" 아이콘이 회전하며 표시 |
| **표시 시간** | 저장 시작 시 페이드인, 완료 후 2초간 유지 후 페이드아웃 |
| **텍스트** | 없음 (아이콘만으로 충분). 다만 첫 자동저장 시에만 "자동 저장 중..." 텍스트를 1회 표시 |
| **게임플레이 방해** | 없음. 자동저장은 백그라운드에서 진행되며 입력을 차단하지 않음 |
| **실패 시** | 아이콘이 빨간색으로 변하고 "저장 실패" 텍스트 2초 표시. 다음 트리거에서 재시도 |

## 3. 수동 저장

### 3.1 수동 저장 접근 방식

| 접근 경로 | 방법 | 비고 |
|-----------|------|------|
| **일시정지 메뉴** | Esc → 일시정지 메뉴 → "저장" 버튼 | 주요 경로 |
| **단축키** | 없음 | 의도적 생략. 메뉴 진입이라는 의식적 행위를 요구 |
| **침대 수면** | 침대 상호작용 시 자동으로 저장 | 자동저장과 동일 시점이지만 플레이어가 의도적으로 실행 |

### 3.2 수동 저장 허용 조건

농장 시뮬레이션은 실시간 전투가 없으므로, 거의 모든 상황에서 수동 저장을 허용한다.

| 상황 | 저장 허용 | 이유 |
|------|----------|------|
| 일반 농장 활동 (경작, 이동, 대화) | O | 제한 없음 |
| 상점 UI 열려 있을 때 | O | 거래 중에도 저장 가능 (미확정 거래는 저장 안 됨) |
| 시설 건설 모드 (배치 미확정) | O | 배치 미확정 상태는 저장에 포함되지 않음. 확정된 시설만 저장 |
| 튜토리얼 시퀀스 진행 중 | O | 현재 시퀀스/단계가 저장됨 (-> see `docs/systems/tutorial-architecture.md` 섹션 7, TutorialSaveData) |
| 씬 전환 중 | X | 로딩 중에는 저장 불가 |
| 자동저장 진행 중 | X | 저장 충돌 방지. 자동저장 완료 후 허용 |
| 이벤트 컷씬 재생 중 | X | 컷씬 완료 후 저장 가능 |

### 3.3 수동 저장 슬롯 선택

수동 저장 시 슬롯 선택 UI가 표시된다. 현재 게임이 로드된 슬롯이 기본 선택되며, 다른 슬롯으로 변경 가능하다.

```
┌─────────────────────────────────────┐
│         저장할 슬롯 선택              │
│                                     │
│  [슬롯 1] ★ 현재 슬롯               │
│    봄 15일 | 레벨 3 | 1,250G        │
│    플레이 시간: 2시간 30분           │
│                                     │
│  [슬롯 2] 비어 있음                  │
│                                     │
│  [슬롯 3] 비어 있음                  │
│                                     │
│  [취소]                              │
└─────────────────────────────────────┘
```

## 4. 멀티슬롯 구조

### 4.1 슬롯 수

세이브 슬롯 수: **3개** (-> see `docs/pipeline/data-pipeline.md` 섹션 3.1)

| 선택지 | 장점 | 단점 | 판정 |
|--------|------|------|------|
| 1개 (자동저장 전용) | 단순, save-scumming 완전 차단 | 실수 복구 불가, 다중 플레이스루 불가 | 기각 |
| 3개 (Stardew 방식) | 적절한 유연성, 가족/친구 공유 | 약간의 save-scumming 여지 | **채택** |
| 무제한 | 최대 유연성 | UI 복잡, 디스크 관리 부담, save-scumming 조장 | 기각 |

**근거**: 3슬롯은 "메인 농장 1개 + 실험/여가용 1~2개"라는 자연스러운 사용 패턴을 지원한다. 가족 구성원이 각자 슬롯을 사용하는 케이스도 커버한다.

### 4.2 슬롯당 표시 정보

슬롯 선택 화면(타이틀, 저장, 로드 공통)에 표시되는 정보:

| 정보 | 표시 형식 | 출처 |
|------|----------|------|
| 농장 이름 | 텍스트 (예: "해바라기 농장") | 새 게임 시 플레이어 입력 |
| 게임 내 날짜 | "Y1 봄 15일" | TimeSaveData (-> see `docs/pipeline/data-pipeline.md` 섹션 3.3) |
| 플레이어 레벨 | "Lv. 3" | PlayerSaveData |
| 소지 골드 | "1,250G" | PlayerSaveData (-> see `docs/systems/economy-system.md`) |
| 총 플레이 시간 | "2시간 30분" | 세이브 메타데이터 |
| 마지막 저장 시각 | "2026-04-06 15:30" | 세이브 메타데이터 (실제 시계 기준) |
| 썸네일 스크린샷 | 저장 시점의 게임 화면 캡처 (256x144px) | 저장 시 자동 캡처 |

### 4.3 슬롯 관리 UX

#### 덮어쓰기 경고

이미 데이터가 있는 슬롯에 새로 저장하거나 새 게임을 시작할 때:

```
┌─────────────────────────────────┐
│   이 슬롯에는 이미 저장 파일이    │
│   있습니다. 덮어쓰시겠습니까?     │
│                                 │
│   봄 15일 | Lv.3 | 1,250G      │
│   플레이 시간: 2시간 30분        │
│                                 │
│   [덮어쓰기]    [취소]           │
└─────────────────────────────────┘
```

#### 삭제 확인

세이브 파일 삭제 시 2단계 확인:

1. 1차: "이 저장 파일을 삭제하시겠습니까?" → [삭제] / [취소]
2. 2차: "삭제된 파일은 복구할 수 없습니다. 정말 삭제하시겠습니까?" → [삭제 확인] / [취소]

2차 확인에서 [삭제 확인] 버튼은 3초간 비활성화(회색) 상태로 표시되어 실수 클릭을 방지한다.

## 5. 게임 종료 시나리오

### 5.1 정상 종료 흐름

```
Esc → 일시정지 메뉴 → "게임 종료" 선택
    │
    ├── 마지막 저장 후 변경사항 있음?
    │   ├── YES → "저장하지 않은 변경사항이 있습니다."
    │   │         [저장 후 종료] [저장 안 함] [취소]
    │   │
    │   └── NO → "게임을 종료하시겠습니까?"
    │            [종료] [취소]
    │
    └── [저장 후 종료] 선택 시:
        현재 슬롯에 자동 저장 → 완료 후 타이틀 화면으로 이동
```

| 버튼 | 동작 |
|------|------|
| **저장 후 종료** | 현재 슬롯에 저장 → 타이틀 화면 |
| **저장 안 함** | 마지막 저장 시점으로 롤백됨을 인지. 타이틀 화면으로 이동 |
| **취소** | 다이얼로그 닫기, 게임 복귀 |

### 5.2 비정상 종료 대비

| 시나리오 | 대응 |
|----------|------|
| **앱 강제 종료 (Alt+F4, 작업 관리자)** | 마지막 자동저장 시점으로 복구. 주기적 백업(10분 간격)이 최대 손실을 약 1일(게임 내)로 제한 |
| **시스템 크래시 / 정전** | 동일하게 마지막 자동저장 복구 |
| **저장 파일 손상** | 백업 파일로 복구 시도 (섹션 5.3 참조) |

### 5.3 백업 파일 전략

| 파라미터 | 값 | 비고 |
|----------|-----|------|
| 백업 보관 수 | 슬롯당 2개 (현재 + 직전) | `save_{N}.json` + `save_{N}.json.bak` |
| 백업 타이밍 | 새 저장 시작 전, 현재 파일을 .backup으로 복사 | 원자적 쓰기 전 안전망 |
| 손상 감지 | 로드 시 JSON 파싱 실패 또는 version 필드 누락 | 자동으로 backup 파일 로드 시도 |
| 복구 실패 시 | "저장 파일이 손상되었습니다. 백업에서 복구를 시도합니다." UI 표시 |
| 백업도 손상 시 | "저장 파일을 복구할 수 없습니다." → 해당 슬롯 빈 슬롯으로 표시 |

## 6. 새 게임 / 파일 삭제

### 6.1 새 게임 시작 흐름

```
타이틀 화면 → "새 게임" 선택
    │
    ▼
┌─────────────────────────────────┐
│      저장 슬롯을 선택하세요       │
│                                 │
│  [슬롯 1] 봄 15일 | Lv.3       │
│  [슬롯 2] (비어 있음)           │  ← 비어 있는 슬롯이 있으면 자동 선택
│  [슬롯 3] (비어 있음)           │
│                                 │
│  [취소]                         │
└─────────────────────────────────┘
    │
    ├── 빈 슬롯 선택 → 바로 새 게임 설정으로 진행
    │
    └── 기존 데이터 있는 슬롯 선택 → 덮어쓰기 경고 (섹션 4.3)
        → 확인 시 기존 파일 삭제 후 새 게임 설정
```

#### 새 게임 설정 화면

| 입력 항목 | 기본값 | 제한 |
|-----------|--------|------|
| 농장 이름 | "내 농장" | 최대 16자, 특수문자 제한 (영문, 한글, 숫자, 공백만 허용) |
| 플레이어 이름 | "농부" | 최대 8자, 동일 제한 |

[OPEN] 캐릭터 외형 커스터마이징 여부 미정. MVP에서는 고정 외형, 이후 확장 고려.

설정 완료 후 → 게임 시작 (튜토리얼 진입, -> see `docs/systems/tutorial-system.md` 섹션 2.1 메인 튜토리얼 시퀀스)

## 7. 저장 데이터 범위 (게임 디자이너 관점)

### 7.1 저장해야 하는 상태

모든 세이브 데이터의 기술적 스키마는 아키텍처 문서에서 정의한다 (-> see `docs/pipeline/data-pipeline.md` 섹션 3.2~3.3). 본 섹션은 "무엇을 저장하는가"의 디자인 의도를 기술한다.

| 카테고리 | 저장 대상 | 디자인 의도 |
|----------|----------|-------------|
| **플레이어 상태** | 위치, 에너지, 레벨, 경험치, 장착 도구 | 중단한 지점에서 정확히 재개 |
| **인벤토리** | 배낭 슬롯, 툴바, 도구 등급 | 소지품 완전 복원 (-> see `docs/systems/inventory-system.md` 섹션 8) |
| **농장 타일** | 전체 타일 상태, 식재된 작물 (성장 단계, 물/비료 상태, 품질 시드) | 농장의 "현재 모습" 완전 보존 |
| **시간/날씨** | 현재 연도, 계절, 일, 시간, 현재 날씨, 내일 예보, 난수 시드 | 시간 흐름의 연속성 보장 (-> see `docs/systems/time-season.md`) |
| **경제** | 소지 골드, 거래 로그, 수급 보정 누적치, 가격 변동 상태 | 경제 시스템 상태 완전 복원 (-> see `docs/systems/economy-system.md`) |
| **시설** | 건설된 시설 목록, 위치, 가동 상태 | 건설 투자의 보존 |
| **가공** | 진행 중인 가공 작업 (레시피, 잔여 시간) | 가공소에 넣어둔 작물이 사라지면 안 됨 |
| **해금 상태** | 해금된 작물, 도구, 시설, 레시피 목록 | 진행도 보존 |
| **상점 재고** | 일일 재고 잔여량 | 당일 상점 상태 유지 (-> see `docs/content/npcs.md`) |
| **NPC 상태** | 호감도, 대화 이력 플래그 | 관계 진행도 보존 [OPEN] NPC 호감도 시스템 상세 미정. 향후 NPC 시스템 확장 시 구체화 |
| **튜토리얼 진행** | 완료된 시퀀스/단계, 활성 시퀀스/단계, 힌트 쿨다운 | 이탈 후 복귀 시 이어서 안내 (-> see `docs/systems/tutorial-architecture.md` TutorialSaveData) |
| **도구 업그레이드** | 각 도구의 현재 등급, 진행 중인 업그레이드 의뢰 | 대장간에 맡긴 도구 상태 보존 (-> see `docs/systems/tool-upgrade.md`) |
| **농장 이름 / 플레이어 이름** | 새 게임 시 입력한 이름 | 슬롯 식별 및 게임 내 표시 |
| **플레이 시간** | 누적 실시간 플레이 시간 (초) | 슬롯 표시, 통계 |
| **세이브 버전** | 데이터 포맷 버전 | 마이그레이션 지원 |

### 7.2 저장하지 않는 상태 (휘발성)

| 상태 | 이유 |
|------|------|
| UI 열림/닫힘 상태 (인벤토리 창, 상점 창 등) | 로드 시 모든 UI는 닫힌 상태로 시작 |
| 카메라 줌 레벨 / 위치 | 로드 시 플레이어 위치 기준 기본 줌으로 복원 |
| 파티클 이펙트 상태 (물 뿌리기 이펙트 등) | 시각적 연출은 일시적 |
| 사운드/BGM 재생 위치 | 로드 시 현재 시간대/계절에 맞는 BGM으로 새로 시작 |
| 일시정지 상태 | 로드 시 항상 비일시정지 상태 |
| 도구 사용 애니메이션 진행 중 상태 | 로드 시 idle 상태로 복원 |
| 미확정 시설 배치 (건설 모드 미리보기) | 확정 전 상태는 저장 안 됨 |
| 미확정 거래 (상점에서 구매 확인 전) | 확정 전 상태는 저장 안 됨 |

## 8. 로드 UX

### 8.1 게임 로드 접근 경로

| 경로 | 설명 |
|------|------|
| **타이틀 화면 → "이어하기"** | 주요 경로. 슬롯 선택 화면 표시 |
| **타이틀 화면 → 마지막 슬롯 자동 선택** | 슬롯이 1개만 사용 중이면 슬롯 선택 생략 가능 [OPEN] 자동 선택 활성화 여부 설정 옵션으로 제공할지 |
| **게임 내 → 일시정지 메뉴 → "타이틀로"** | 타이틀 화면 복귀 후 다시 로드 |

게임 내에서 직접 "다른 슬롯 로드"는 지원하지 않는다. 반드시 타이틀 화면을 경유한다. 이는 데이터 안전성을 보장하고 UX를 단순하게 유지하기 위함이다.

### 8.2 슬롯 선택 → 로딩 → 게임 재개 흐름

```
타이틀 화면
    │
    ▼
슬롯 선택 화면 (슬롯 정보 + 썸네일 표시)
    │
    ▼ 슬롯 선택
    │
로딩 화면 진입
    │
    ├── 페이드 아웃 (타이틀 → 검정, 0.5초)
    │
    ├── 로딩 화면 표시
    │   ├── 로딩 바 (진행률 0~100%)
    │   ├── 팁 텍스트 (하단, 랜덤 1개)
    │   └── 로딩 중 씨앗/새싹 애니메이션 (장식)
    │
    ├── 데이터 로드 + 씬 생성
    │   ├── 세이브 JSON 파싱
    │   ├── 타일맵 복원
    │   ├── 시설 배치
    │   ├── NPC/상점 초기화
    │   └── 플레이어 상태 복원
    │
    ├── 최소 로딩 시간: 1.5초 (너무 빠르면 깜빡임 느낌 방지)
    │
    ├── 페이드 인 (검정 → 게임 화면, 0.5초)
    │
    └── 게임 재개 (비일시정지 상태)
```

### 8.3 로딩 화면 팁 텍스트

로딩 중 하단에 랜덤으로 1개의 팁을 표시한다. 플레이어 레벨에 따라 표시 가능한 팁 풀이 확장된다.

| 레벨 범위 | 예시 팁 |
|-----------|---------|
| 1~3 | "물을 매일 주면 작물이 더 빨리 자랍니다" |
| 1~3 | "호미로 땅을 갈아야 씨앗을 심을 수 있습니다" |
| 3~5 | "시설을 건설하면 농장 효율이 크게 올라갑니다" |
| 3~5 | "비 오는 날에는 물주기가 필요 없습니다" |
| 5~7 | "가공소에서 작물을 가공하면 더 높은 가격에 팔 수 있습니다" |
| 5~7 | "창고에 작물을 저장했다가 가격이 오를 때 팔아보세요" |
| 7~10 | "온실에서는 계절에 관계없이 어떤 작물이든 재배할 수 있습니다" |

팁 목록은 콘텐츠 데이터로 관리한다. [OPEN] 팁 텍스트의 canonical 위치를 별도 콘텐츠 문서로 분리할지, 본 문서에서 관리할지.

### 8.4 로드 실패 처리

| 실패 유형 | UI 표시 | 동작 |
|-----------|---------|------|
| JSON 파싱 오류 | "저장 파일이 손상되었습니다. 백업에서 복구를 시도합니다." | 자동으로 .backup 파일 로드 시도 |
| 백업도 실패 | "저장 파일을 복구할 수 없습니다." | 슬롯 선택 화면 복귀, 해당 슬롯 "손상됨" 표시 |
| 버전 불일치 | "이전 버전의 저장 파일입니다. 변환 중..." | 마이그레이션 실행 후 로드 |
| 마이그레이션 실패 | "이 저장 파일은 현재 버전과 호환되지 않습니다." | 슬롯 선택 화면 복귀 |

## 9. 타이틀 화면 전체 흐름 정리

```
┌──────────────────────────────┐
│        SeedMind              │
│        타이틀 화면            │
│                              │
│   [이어하기]  ← 세이브 있을 때만 활성
│   [새 게임]                   │
│   [설정]                      │
│   [종료]                      │
└──────────────────────────────┘

"이어하기" → 슬롯 선택(섹션 8.2) → 로딩 → 게임 재개
"새 게임"  → 슬롯 선택(섹션 6.1) → 새 게임 설정 → 튜토리얼 시작
"설정"     → 오디오/그래픽/조작 설정 (별도 문서)
"종료"     → 앱 종료
```

"이어하기" 버튼은 세이브 파일이 하나도 없으면 비활성화(회색) 상태로 표시된다.

## Risks

- [RISK] 주기적 자동저장(10분 간격)이 대규모 농장에서 프레임 드롭을 유발할 수 있다. 비동기 직렬화 + 백그라운드 스레드 쓰기가 필수적이나, Unity의 메인 스레드 제약으로 인해 데이터 수집 단계에서 순간 지연이 발생할 수 있다.
- [RISK] 썸네일 스크린샷(256x144px) 저장이 세이브 파일 크기를 유의미하게 증가시킬 수 있다. PNG 인코딩 기준 슬롯당 약 20~50KB 추가. 별도 파일로 분리 저장하는 것이 바람직하다.
- [RISK] 세이브 버전 마이그레이션은 게임 업데이트마다 누적되는 기술 부채다. 명확한 마이그레이션 체인(v1→v2→v3)을 초기부터 설계해야 한다.
- [RISK] 한글 농장 이름/플레이어 이름의 JSON 인코딩 시 UTF-8 처리가 필요하다. Unity의 JsonUtility는 기본적으로 UTF-8을 지원하나, 파일 시스템 경로에 한글이 포함될 경우 플랫폼별 호환성 테스트 필요.
