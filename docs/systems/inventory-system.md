# 인벤토리/아이템 시스템 (Inventory & Item System) 상세 설계

> 작성: Claude Code (Opus) | 2026-04-06  
> 문서 ID: DES-005

---

## Context

이 문서는 SeedMind의 인벤토리 및 아이템 시스템을 상세히 기술한다. 플레이어가 씨앗, 도구, 수확물, 비료 등 모든 아이템을 획득하고, 보관하고, 사용하고, 거래하는 전체 흐름을 다룬다. 마스터 디자인 문서(`docs/design.md`)의 핵심 게임 루프(씨앗 심기 -> 관리 -> 수확 -> 판매 -> 확장)를 아이템 관점에서 뒷받침하는 시스템이다.

**설계 목표**: 직관적이고 단순한 인벤토리 조작으로 핵심 루프의 흐름을 끊지 않으면서, 배낭 용량 제한과 창고 활용을 통해 "무엇을 들고 다닐 것인가"라는 의미 있는 선택지를 제공한다.

## 1. 아이템 분류 체계 (Item Category System)

### 1.1 카테고리 정의

| 카테고리 | 영문 키 | 설명 | 스택 가능 | 최대 스택 | 드롭 가능 | 비고 |
|----------|---------|------|:---------:|:---------:|:---------:|------|
| 씨앗 | `Seed` | 경작지에 심을 수 있는 씨앗류 | O | 99 | O | 카테고리 내 종류별 구분 |
| 수확물 | `Crop` | 작물 수확 시 획득하는 원재료 | O | 99 | O | 품질(Quality) 속성 보유 |
| 도구 | `Tool` | 경작/수확에 사용하는 도구 | X | 1 | X | 파괴/분실 불가 |
| 비료/소모품 | `Consumable` | 비료, 음식 등 사용 시 소멸 | O | 30 | O | 사용 즉시 효과 적용 |
| 건설 재료 | `Material` | 시설 건설에 필요한 재료 | O | 99 | O | [OPEN] Phase 1에서 건설 재료 시스템 포함 여부 검토 필요 |
| 가공품 | `Processed` | 가공소에서 제작한 아이템 | O | 30 | O | 품질 속성 없음 (-> see `docs/systems/economy-system.md` 섹션 2.5) |

### 1.2 카테고리별 공통 속성

모든 아이템은 다음 공통 속성을 갖는다:

| 속성 | 영문 키 | 타입 | 설명 |
|------|---------|------|------|
| 아이템 ID | `itemId` | string | 고유 식별자 (예: `seed_potato`, `crop_tomato`) |
| 표시명 | `displayName` | string | UI에 표시되는 이름 (예: "감자 씨앗") |
| 카테고리 | `category` | enum | 위 1.1의 카테고리 중 하나 |
| 아이콘 | `icon` | Sprite | 인벤토리 슬롯에 표시할 아이콘 |
| 설명 | `description` | string | 툴팁에 표시할 아이템 설명 |
| 스택 가능 | `isStackable` | bool | 같은 슬롯에 중첩 가능 여부 |
| 최대 스택 | `maxStack` | int | 한 슬롯에 보관 가능한 최대 수량 |
| 판매 가능 | `isSellable` | bool | 상점에서 판매 가능 여부 |
| 기본 판매가 | `baseSellPrice` | int | 판매 시 기본 가격 (-> see `docs/systems/economy-system.md`) |

### 1.3 카테고리별 추가 속성

#### 씨앗 (Seed) 추가 속성

| 속성 | 영문 키 | 설명 |
|------|---------|------|
| 대상 작물 | `cropType` | 이 씨앗이 생산하는 작물 종류 |
| 재배 가능 계절 | `growSeasons` | 심을 수 있는 계절 목록 (-> see `docs/systems/time-season.md`) |

#### 수확물 (Crop) 추가 속성

| 속성 | 영문 키 | 설명 |
|------|---------|------|
| 품질 등급 | `quality` | Normal / Silver / Gold / Iridium (-> see `docs/systems/crop-growth.md`) |
| 작물 분류 | `cropClass` | 과일(Fruit) / 채소(Vegetable) / 특수(Special) — 가공 유형 결정 기준 |

#### 도구 (Tool) 추가 속성

| 속성 | 영문 키 | 설명 |
|------|---------|------|
| 도구 등급 | `toolTier` | Basic / Reinforced / Legendary (→ see `docs/systems/tool-upgrade.md` 섹션 1.1) |
| 에너지 소모 | `energyCost` | 1회 사용 시 소모 에너지 (-> see `docs/systems/farming-system.md` 섹션 3.1) |
| 적용 범위 | `range` | 효과 적용 타일 범위 |

#### 비료/소모품 (Consumable) 추가 속성

| 속성 | 영문 키 | 설명 |
|------|---------|------|
| 효과 타입 | `effectType` | GrowthSpeed / QualityBoost / SoilImprove / EnergyRestore |
| 효과 수치 | `effectValue` | 효과의 강도 (비료별 상이, -> see `docs/systems/farming-system.md` 섹션 5.1) |

## 2. 인벤토리 구조 (Inventory Structure)

### 2.1 배낭 (Backpack)

플레이어가 항상 소지하는 인벤토리 공간.

| 파라미터 | 영문 키 | 기본값 | 설명 |
|----------|---------|--------|------|
| 초기 슬롯 수 | `baseSlotCount` | 15 | 시작 시 사용 가능한 슬롯 |
| 최대 슬롯 수 | `maxSlotCount` | 30 | 업그레이드 완료 시 최대 슬롯 |
| 슬롯 배치 | `layout` | 5열 x N행 | UI 그리드 배치 (행 수는 슬롯 수에 따라 변동) |

#### 배낭 업그레이드 경로

| 단계 | 슬롯 수 | 비용 | 해금 조건 | 설명 |
|------|---------|------|-----------|------|
| 기본 배낭 | 15칸 | - | 게임 시작 | 초기 지급 |
| 중형 배낭 | 20칸 | 1,000G | 레벨 3 | 상점에서 구매 |
| 대형 배낭 | 25칸 | 3,000G | 레벨 5 | 상점에서 구매 |
| 최대 배낭 | 30칸 | 8,000G | 레벨 8 | 상점에서 구매 |

**설계 의도**: 초반 15칸은 도구 4종 + 씨앗 2~3종 + 수확물 여유분으로 충분하되, 작물 종류가 늘어나는 중반부터 공간 부족을 느끼게 한다. 배낭 업그레이드와 창고 건설 사이에서 투자 우선순위 선택이 발생한다.

[RISK] 배낭 업그레이드 비용이 시설 건설 비용(물탱크 500G, 창고 1,000G)과 경쟁한다. 중형 배낭(1,000G)과 창고(1,000G)가 동일 가격대에 같은 레벨 근처에서 해금되므로, 플레이어에게 의미 있는 선택지가 된다.

### 2.2 툴바 (Toolbar / Hotbar)

화면 하단에 항상 표시되는 단축 슬롯.

| 파라미터 | 영문 키 | 값 | 설명 |
|----------|---------|-----|------|
| 슬롯 수 | `toolbarSlotCount` | 8 | 고정 (업그레이드 불가) |
| 단축키 | - | 1~8 숫자키 | 슬롯 선택 |
| 마우스 휠 | - | 스크롤 | 인접 슬롯 순환 선택 |

#### 툴바 기본 배치 (게임 시작 시)

| 슬롯 | 기본 아이템 | 비고 |
|------|-------------|------|
| 1 | 호미 (Hoe) | 고정 아이템 아님, 이동 가능 |
| 2 | 물뿌리개 (Watering Can) | |
| 3 | 낫 (Sickle) | |
| 4 | 도끼 (Axe) | |
| 5~8 | 비어 있음 | 플레이어가 자유 배치 |

**배치 규칙**:
- 모든 카테고리의 아이템을 툴바에 배치할 수 있음
- 도구를 툴바에서 제거해도 배낭에 보관됨 (분실 불가)
- 씨앗이나 비료를 툴바에 배치하면 빠르게 사용 가능
- 툴바 슬롯은 배낭 슬롯과 별도 (총 슬롯 = 배낭 + 툴바 8칸)

### 2.3 창고 (Storage)

시설 건설을 통해 추가되는 외부 저장 공간.

| 파라미터 | 값 | 비고 |
|----------|-----|------|
| 건설 비용 | (-> see `docs/design.md` 섹션 4.6) | 1,000G, 레벨 4 해금 |
| 슬롯 수 | 30칸 | 1동당 |
| 최대 건설 수 | 3동 | 총 90칸 확장 가능 |
| 추가 건설 비용 | 2동째: 2,000G, 3동째: 4,000G | 점진적 비용 증가 |
| 접근 방식 | 창고 건물에 인접하여 E키 상호작용 | 전용 UI 오픈 |

**창고 UI**:
- 배낭 인벤토리와 창고 인벤토리가 나란히 표시
- 드래그 앤 드롭으로 배낭 <-> 창고 간 아이템 이동
- Shift + 좌클릭으로 빠른 이동 (반대쪽으로 자동 이동)
- 도구는 창고에 보관할 수 없음 (항상 배낭/툴바에 유지)

**설계 의도**: 창고는 가격 변동을 대응하여 수확물을 비축하거나(-> see `docs/systems/economy-system.md` 섹션 2.6), 다음 계절 씨앗을 미리 보관하는 전략적 용도로 사용된다. 창고 슬롯이 충분해야 수급 변동 시스템이 의미를 가진다.

[OPEN] 창고 간 아이템 자동 정렬(카테고리별 분류) 기능 필요 여부 검토. 편의성은 높지만 구현 복잡도도 증가한다.

### 2.4 출하함 (Shipping Bin)

수확물을 넣으면 다음 날 자동으로 판매되는 특수 저장소.

| 파라미터 | 값 | 비고 |
|----------|-----|------|
| 설치 | 농장에 기본 제공 (게임 시작 시 존재) | 건설 불필요 |
| 슬롯 수 | 제한 없음 (무한) | 넣은 아이템은 당일 회수 가능, 다음 날 06:00 정산 |
| 판매 가격 | (-> see `docs/systems/economy-system.md` 섹션 2.1~2.6) | 동적 가격 변동 적용 |
| 판매 불가 아이템 | 도구, 건설 재료 | 넣기 자체가 차단됨 |

**출하함 동작**:
1. 출하함에 인접하여 E키 -> 출하함 UI 오픈
2. 배낭에서 아이템을 드래그하여 출하함에 넣기
3. 당일 중에는 출하함에서 아이템 회수 가능
4. 다음 날 06:00에 자동 판매, 판매 대금은 골드에 추가
5. 판매 결과는 다음 날 아침 요약 화면에 표시

## 3. 아이템 상호작용 규칙 (Item Interaction Rules)

### 3.1 기본 조작

| 조작 | 대상 | 동작 | 비고 |
|------|------|------|------|
| 좌클릭 | 툴바 선택 아이템 | 도구 사용 / 아이템 사용 (타일 대상) | 월드에서의 기본 액션 |
| 우클릭 | 인벤토리 슬롯 | 컨텍스트 메뉴 열기 | 사용/장착/버리기/정보 |
| 좌클릭 드래그 | 인벤토리 슬롯 | 아이템 이동 (슬롯 간) | 배낭, 툴바, 창고 모두 동일 |
| Shift + 좌클릭 | 인벤토리 슬롯 | 빠른 이동 | 배낭<->창고, 배낭<->출하함 |
| 우클릭 드래그 | 스택 아이템 | 스택 분할 (절반씩) | 스택 가능 아이템만 |
| 숫자키 1~8 | - | 툴바 슬롯 선택 | 월드 조작 시 |
| 마우스 휠 | - | 툴바 슬롯 순환 | |
| I키 | - | 전체 인벤토리 토글 | Tab키도 동일 기능 (design.md와 호환) |

### 3.2 아이템 사용 규칙 (월드에서)

#### 씨앗 사용
1. 툴바에서 씨앗 선택 (또는 씨앗봉투 도구에서 마우스 휠로 선택)
2. Tilled 타일에 좌클릭
3. 씨앗 1개 소모, 타일이 Planted 상태로 전환
4. 계절 불일치 시 경고 UI 표시 (심기 자체는 가능, -> see `docs/systems/farming-system.md` 섹션 3.3)

#### 비료 사용
1. 툴바에서 비료 선택
2. Tilled / Planted / Dry 타일에 좌클릭
3. 비료 1개 소모, 타일에 비료 효과 적용
4. 이미 비료가 적용된 타일에 사용 시 기존 비료 교체 (확인 팝업)

#### 도구 사용
- 도구별 사용 규칙은 `docs/systems/farming-system.md` 섹션 3에 정의
- 도구 내구도 없음 (파괴/수리 메카닉 없음)
- 도구 업그레이드는 상점에서 골드로 구매 (기존 도구를 상위 등급으로 교체)

#### 음식 사용
1. 인벤토리에서 우클릭 -> "먹기" 선택, 또는 툴바에서 좌클릭
2. 에너지 회복 효과 즉시 적용
3. 아이템 1개 소모

### 3.3 아이템 획득 규칙

| 획득 경로 | 아이템 카테고리 | 동작 |
|-----------|----------------|------|
| 수확 | Crop | 낫 사용 또는 E키 -> 배낭에 자동 추가 |
| 상점 구매 | Seed, Consumable, Tool | 상점 UI에서 구매 -> 배낭에 추가 |
| 아이템 줍기 | 전체 | 드롭된 아이템에 인접 후 E키 또는 자동 줍기 |
| 가공 완료 | Processed | 가공소에서 E키로 회수 -> 배낭에 추가 |

#### 배낭 가득 참 처리
- 배낭 슬롯이 모두 차 있을 때 아이템 획득 시도:
  - "배낭이 가득 찼습니다!" 경고 메시지 표시 (화면 중앙 상단, 2초간)
  - 수확물: 타일에 남아 있음 (수확 동작 취소)
  - 상점 구매: 구매 버튼 비활성화
  - 드롭 아이템: 월드에 그대로 남아 있음 (사라지지 않음)

[RISK] 가공소에서 완성된 가공품을 회수하지 않으면 가공 슬롯이 막힐 수 있다. 가공 완료 아이템은 가공소 내부 버퍼(최대 5개)에 보관하다가, 버퍼도 가득 차면 가공 자체가 일시정지되는 방식으로 처리한다.

### 3.4 아이템 드롭/폐기

| 동작 | 방법 | 결과 |
|------|------|------|
| 드롭 | 인벤토리 밖으로 드래그 | 월드에 아이템 드롭 (60초 후 소멸) |
| 폐기 | 우클릭 메뉴 -> "버리기" | 확인 팝업 후 즉시 삭제 |
| 도구 드롭 | 불가 | 도구는 드롭/폐기 차단 |

**드롭 아이템 동작**:
- 플레이어 위치에서 전방 1타일 거리에 생성
- 60초간 월드에 존재 (반짝임 이펙트)
- 플레이어가 1.5타일 이내로 접근하면 자동 줍기
- 60초 후 소멸 (되돌릴 수 없음)

## 4. 인벤토리 UI 규칙 (Inventory UI Design)

### 4.1 화면 배치

```
┌──────────────────────────────────────────────────┐
│ [시간/날짜]  [계절]  [날씨]           [골드: 0G] │
│                                                  │
│                                                  │
│                  (게임 화면)                      │
│                                                  │
│                                                  │
│  ┌─┬─┬─┬─┬─┬─┬─┬─┐                              │
│  │1│2│3│4│5│6│7│8│  ← 툴바 (항상 표시)           │
│  └─┴─┴─┴─┴─┴─┴─┴─┘                              │
│ [에너지 바]  [레벨/경험치 바]                     │
└──────────────────────────────────────────────────┘
```

#### 전체 인벤토리 (I키 또는 Tab키 토글)

```
┌────────────────────────────────┐
│         인벤토리               │
│  ┌──┬──┬──┬──┬──┐             │
│  │  │  │  │  │  │  ← 행 1     │
```

### 4.2 슬롯 표시 정보

각 인벤토리 슬롯은 다음 정보를 표시한다:

| 요소 | 위치 | 표시 조건 |
|------|------|-----------|
| 아이템 아이콘 | 슬롯 중앙 | 항상 |
| 수량 텍스트 | 슬롯 우측 하단 | 스택 수량 >= 2일 때만 |
| 품질 별 | 슬롯 좌측 하단 | Crop 카테고리, Silver 이상일 때 |
| 도구 등급 표시 | 슬롯 좌측 상단 | Tool 카테고리, Reinforced 이상일 때 (→ see `docs/systems/tool-upgrade.md` 섹션 7.3) |
| 선택 하이라이트 | 슬롯 테두리 | 현재 선택된 툴바 슬롯 |

#### 품질 표시 (수확물)

| 품질 | 아이콘 | 색상 |
|------|--------|------|
| Normal | 없음 | - |
| Silver | 은색 별 1개 | #C0C0C0 |
| Gold | 금색 별 2개 | #FFD700 |
| Iridium | 보라색 별 3개 | #9B59B6 |

### 4.4 정렬 및 필터

#### 정렬 (Sort)
- 인벤토리 UI 하단의 "정렬" 버튼 클릭 시 실행
- 정렬 기준: 카테고리 순서 (Tool -> Seed -> Consumable -> Crop -> Processed -> Material) 내에서 아이템 ID 오름차순
- 빈 슬롯은 뒤쪽으로 밀림
- 같은 아이템의 분산된 스택을 자동 병합

#### 카테고리 필터 (Filter)
- 전체 / 씨앗 / 수확물 / 도구 / 소모품 / 가공품 탭
- 필터 적용 시 해당 카테고리가 아닌 슬롯은 반투명 처리 (숨기지는 않음)
- 필터는 표시용이며, 아이템 이동이나 사용에 영향 없음

## 6. 상점 연동 (Shop Integration)

### 6.1 구매 흐름

1. 상점 NPC 또는 상점 건물에 인접하여 E키
2. 상점 UI 오픈: 좌측에 상점 품목, 우측에 플레이어 배낭
3. 구매할 아이템 클릭 -> 수량 선택 (1/5/10/전부 버튼)
4. 총 비용 표시 -> "구매" 확인
5. 골드 차감, 아이템이 배낭에 추가
6. 배낭 여유 슬롯이 없으면 구매 불가 (이미 보유한 스택 아이템은 스택 여유분만큼 구매 가능)

#### 상점 판매 품목

| 카테고리 | 품목 | 해금 조건 |
|----------|------|-----------|
| 씨앗 | 플레이어 레벨에 따라 해금된 씨앗 | (-> see `docs/design.md` 섹션 4.2) |
| 비료 | 해금 조건 충족한 비료 전종 | (-> see `docs/systems/farming-system.md` 섹션 5.1) |
| 도구 업그레이드 | 상위 등급 도구 | (-> see `docs/systems/farming-system.md` 섹션 7) |
| 배낭 업그레이드 | 중형/대형/최대 배낭 | 본 문서 섹션 2.1 참조 |
| 음식 | [OPEN] 미정 | - |

**계절별 품목 변동**: 상점에서 판매하는 씨앗은 현재 계절에 재배 가능한 씨앗만 진열된다. 다른 계절 씨앗은 목록에서 제외되지만, 이미 구매한 씨앗은 인벤토리에 남아있다.

### 6.2 판매 흐름

플레이어는 두 가지 방법으로 아이템을 판매할 수 있다:

#### 방법 A: 상점 직접 판매
1. 상점 UI에서 배낭의 아이템을 상점 영역으로 드래그
2. 판매가 표시 -> "판매" 확인
3. 즉시 골드 획득
4. 상점 직접 판매는 수급 변동 메커니즘 적용하지 않음 (-> see `docs/systems/economy-system.md` 섹션 2.6.2)

#### 방법 B: 출하함 판매
1. 출하함에 아이템 넣기
2. 다음 날 06:00 자동 판매
3. 수급 변동 메커니즘 적용
4. 대량 판매에는 출하함이 유리하지 않을 수 있음 (가격 하락 위험)

**판매 불가 아이템**: 도구, 건설 재료

**설계 의도**: 상점 직접 판매(즉시 현금화, 수급 무관)와 출하함 판매(대량 가능, 수급 적용) 두 경로를 제공하여, 플레이어가 판매 타이밍과 방법을 전략적으로 선택하게 한다.

[OPEN] 상점 직접 판매에도 수급 변동을 적용할지 검토 필요. 현재 설계는 출하함만 수급 영향을 받으므로, 소량은 상점에서, 대량은 분산 출하하는 전략이 최적이 된다. 이것이 의도된 트레이드오프인지 확인 필요.

## 7. 아이템 ID 명명 규칙 (Item ID Naming Convention)

모든 아이템 ID는 아래 규칙을 따른다:

```
{category}_{name}[_{variant}]
```

| 카테고리 | 형식 | 예시 |
|----------|------|------|
| 씨앗 | `seed_{name}` | `seed_potato`, `seed_tomato` |
| 수확물 | `crop_{name}` | `crop_potato`, `crop_corn` |
| 도구 | `{type}_{tier}` | `hoe_basic`, `sickle_reinforced`, `wateringcan_legendary` |
| 비료 | `fert_{name}` | `fert_basic`, `fert_quality` |
| 가공품 | `proc_{crop}_{type}` | `proc_potato_jam`, `proc_strawberry_juice` |
| 건설 재료 | `mat_{name}` | `mat_wood`, `mat_stone` (Phase 2) |
| 음식 | `food_{name}` | `food_bread` (미정) |

**도구 ID 규칙**: 도구 등급(tier)은 ID suffix에 포함된다. tier 값은 `basic` / `reinforced` / `legendary` 세 단계이다. 같은 도구 종류의 다른 등급은 별개의 ItemData SO 에셋으로 존재한다. (→ see `docs/systems/tool-upgrade.md`, `docs/mcp/tool-upgrade-tasks.md`)

## 8. 세이브/로드 고려사항 (Save/Load Considerations)

인벤토리 데이터는 게임 저장 시 아래 구조로 직렬화된다.

| 데이터 | 저장 대상 | 비고 |
|--------|-----------|------|
| 배낭 슬롯 배열 | 각 슬롯의 (itemId, quantity, quality) | 빈 슬롯도 null로 저장 |
| 툴바 슬롯 배열 | 배낭과 동일 구조 | 8칸 고정 |
| 창고 슬롯 배열 | 창고별 별도 배열 | 건설된 창고 수만큼 — [OPEN] data-pipeline.md 섹션 3.2 InventorySaveData에 창고 슬롯 필드가 아직 없음. BuildingSaveData와의 분리 방식 결정 필요 |
| 출하함 목록 | 당일 넣은 아이템 리스트 | 정산 전 상태 |
| 배낭 업그레이드 상태 | 현재 슬롯 수 | 업그레이드 단계 복원용 |

구체적인 JSON 스키마와 직렬화 방식은 아키텍처 문서에서 다룬다 (-> see `docs/pipeline/data-pipeline.md` 섹션 3.2 for InventorySaveData canonical 스키마).
