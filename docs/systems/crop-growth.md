# 작물 성장 시스템 (Crop Growth System) 상세 설계

> 작성: Claude Code (Opus) | 2026-04-06  
> 문서 ID: DES-002

---

## Context

이 문서는 SeedMind의 작물 성장 시스템을 상세히 기술한다. 경작 시스템(`docs/systems/farming-system.md`)에서 정의한 타일 상태 머신 위에서 작물이 어떻게 성장하고, 수확되며, 실패하는지를 다룬다. 성장 단계별 시각적 변화, 성장 공식, 계절별 재배 제한, 수확 메카닉, 특수 작물, 실패 조건, 시각/청각 피드백까지 모든 세부 사항을 포함한다.

**설계 목표**: 작물 성장이 눈에 보이는 진행감을 제공하고, 비료/토양/계절 조합으로 숙련 플레이어에게 최적화 재미를 준다. 동시에 초보 플레이어는 "물만 주면 자란다"는 단순한 모델로 시작할 수 있어야 한다.

## 1. 성장 단계 상세 (Growth Stages)

모든 작물은 4단계의 성장 과정을 거친다.

### 1.1 단계 정의

| 단계 | 영문 키 | 설명 | 성장일수 비율 |
|------|---------|------|--------------|
| 씨앗 | `Seed` | 땅 속에 심어진 상태 | 0% |
| 새싹 | `Sprout` | 땅 위로 작은 줄기가 올라온 상태 | 1일째 물 준 후 전환 |
| 성장 | `Growing` | 잎과 줄기가 자라는 상태 | 총 성장일수의 약 60% 시점 |
| 수확 가능 | `Harvestable` | 열매/뿌리가 완성된 상태 | 총 성장일수 100% 도달 |

**단계 전환 기준**: 성장일수를 3등분하여 각 구간에 단계를 배정한다.

```
성장 진행률 = (경과_성장일수 / 총_성장일수) * 100%

Seed:         심은 직후 (성장일수 = 0)
Sprout:       0% < 진행률 <= 33%
Growing:      33% < 진행률 <= 99%
Harvestable:  진행률 = 100%
```

**예시 (토마토, 기본 5일)**:
- 0일: Seed (심은 날)
- 1일차 물 줌: 성장일수 1 → 20% → Sprout
- 2일차 물 줌: 성장일수 2 → 40% → Growing
- 3~4일차 물 줌: 성장일수 3~4 → 60~80% → Growing 유지
- 5일차 물 줌: 성장일수 5 → 100% → Harvestable

### 1.2 단계별 시각적 변화 (로우폴리 3D)

각 작물은 단계별로 별도 프리팹(또는 메시 변형)을 갖는다.

| 단계 | 공통 시각 표현 | 로우폴리 특징 |
|------|--------------|--------------|
| Seed | 경작된 흙에 작은 흙더미가 솟아 있음. 씨앗 종류에 따른 색상 태그(작은 깃발/꼬리표) 표시 | 3~4개 폴리곤의 작은 돌기, 높이 0.05m |
| Sprout | 흙 사이로 얇은 줄기 1~2개가 올라옴. 연두색 | 줄기: 단순 실린더 2개, 잎: 삼각형 1~2장, 높이 0.15m |
| Growing | 줄기가 굵어지고 잎이 펼쳐짐. 작물별 고유 형태 시작 | 줄기: 굵은 실린더, 잎: 4~6장, 작물별 색상 분화, 높이 0.3~0.5m |
| Harvestable | 열매/뿌리 등 수확물이 보임. 색이 선명해짐. 미세 흔들림 애니메이션 | 작물별 고유 열매 모델(구/원기둥 기반), 높이 0.4~0.8m |

#### 작물별 Harvestable 시각 예시

| 작물 | Harvestable 시각 | 색상 |
|------|-----------------|------|
| 감자 | 흙 위로 잎이 무성, 흙 사이로 감자가 일부 노출 | 잎: 짙은 녹색, 감자: 갈색 |
| 당근 | 잎이 풍성하게 솟아 있고, 주황색 당근 윗부분이 흙 위로 보임 | 잎: 녹색, 당근: 주황 |
| 토마토 | 지지대에 줄기가 감기고, 빨간 토마토 2~3개가 달림 | 줄기: 녹색, 열매: 빨강 |
| 옥수수 | 키 큰 줄기에 옥수수 이삭 1~2개 | 줄기: 녹색, 이삭: 노랑 |
| 딸기 | 낮은 덤불에 빨간 딸기 여러 개 | 잎: 녹색, 열매: 빨강 |
| 호박 | 넓은 잎 사이에 큰 호박 1개 | 잎: 녹색, 호박: 주황 |
| 해바라기 | 키 큰 줄기 끝에 큰 꽃 1송이 | 줄기: 녹색, 꽃잎: 노랑, 중심: 갈색 |
| 수박 | 넓은 잎이 펼쳐지고, 큰 수박 1개가 땅에 놓임 | 잎: 녹색, 수박: 짙은 녹색+연녹색 줄무늬 |

### 1.3 단계별 플레이어 상호작용

| 단계 | 물주기 | 비료 주기 | 수확 | 제거(낫) | 타일 정보 조회 |
|------|--------|----------|------|---------|--------------|
| Seed | O (필수) | O | X | O (씨앗 소멸, 환불 없음) | O |
| Sprout | O (필수) | O | X | O (작물 소멸) | O |
| Growing | O (필수) | O | X | O (작물 소멸) | O |
| Harvestable | X (불필요) | X | O | O (수확과 동일) | O |
| Withered | X | X | X | O (정리만 가능) | O |

**참고**: 비료는 Seed~Growing 어느 단계에서든 적용 가능하다. 적용 즉시 효과 발생. 이미 적용된 비료가 있으면 새 비료로 대체된다 (-> see `docs/systems/farming-system.md` 섹션 5.2).

### 1.4 단계 전환 트리거

성장 단계 전환은 **매일 아침 06:00의 배치 처리**에서 발생한다 (-> see `docs/architecture.md` 섹션 4.2).

```
매일 06:00 GrowthSystem.ProcessDailyGrowth():
  1. 모든 Watered 타일을 순회
  2. 각 타일의 작물에 성장일수 +1 (비료/토양 보정 적용)
  3. 성장일수에 따라 단계 전환 판정
  4. 단계가 변경되면 프리팹 교체 + 파티클 재생
  5. 모든 Watered 타일을 Dry로 전환 (물 증발)
  6. Dry 연속일수 카운터 업데이트 → 3일 연속 Dry면 Withered 전환
```

## 2. 성장 공식 (Growth Formula)

### 2.1 기본 성장 규칙

```
일일_성장_진행 = 물_제공 여부에 따라 결정

물을 준 날 (Watered 상태):  +1.0 성장일수
물을 안 준 날 (Dry 상태):   +0.0 성장일수 (성장 정지)
```

작물은 **Watered 상태인 날에만** 성장한다. 이것이 가장 기본적인 규칙이며, 아래의 보정은 이 기본값에 곱해진다.

### 2.2 비료 보정 (Fertilizer Multiplier)

비료는 성장 속도에 배수를 적용한다 (-> see `docs/systems/farming-system.md` 섹션 5, canonical).

farming-system.md에서 정의한 4종 비료 중 성장 속도에 영향을 주는 비료:

| 비료 | 영문 키 | 속도 배수 | 해금 | 비고 |
|------|---------|----------|------|------|
| 없음 | `None` | x1.0 | - | 기본 |
| 기본 비료 | `BasicFertilizer` | x1.25 | 시작 | +25% 성장 속도 |
| 속성장 비료 | `SpeedGro` | x1.5 | 레벨 4 | +50% 성장 속도 |
| 고급 비료 | `QualityFertilizer` | x1.0 | 레벨 3 | 성장 속도 영향 없음, 품질 보너스 전용 |
| 유기 비료 | `OrganicFertilizer` | x1.0 | 레벨 6 | 성장 속도 영향 없음, 토양 품질 +1단계 (영구) |

**참고**: 고급 비료와 유기 비료는 성장 속도를 높이지 않는다. 성장 속도 배수는 기본 비료(x1.25)와 속성장 비료(x1.5) 두 종류만 제공된다. 두 종류의 비료는 겹칠 수 없다 (한 타일에 하나의 비료만 적용).

[OPEN] 비료 4종(기본/고급/속성장/유기)으로는 성장 속도 배수가 x1.25와 x1.5 두 단계만 존재한다. 더 높은 배수(x1.75, x2.0 등)를 원한다면 비료 종류를 추가해야 한다. 최종 비료 목록은 CON-001에서 확정 필요.

### 2.3 토양 품질 보정 (Soil Quality Multiplier)

토양 품질에 따른 성장 속도 배수 (-> see `docs/systems/farming-system.md` 섹션 6, canonical).

farming-system.md 섹션 6.1에서 정의한 보너스를 배수로 환산:

| 토양 등급 | 영문 키 | 성장 속도 보너스 | 배수 환산 |
|----------|---------|-----------------|----------|
| 척박 | `Poor` | -10% | x0.9 |
| 보통 | `Normal` | 0% | x1.0 |
| 비옥 | `Fertile` | +10% | x1.1 |
| 풍요 | `Rich` | +20% | x1.2 |

**참고**: 세 번째 등급의 영문 키는 `Fertile`이다 (Good이 아님). 배수는 farming-system.md의 성장 속도 보너스(-10%/0%/+10%/+20%)를 직접 변환한 값(0.9/1.0/1.1/1.2)을 사용한다.

### 2.4 계절 보정 (Season Multiplier)

각 작물은 선호 계절이 있으며, 재배 가능 계절에서 추가 보너스/패널티가 적용된다.

| 조건 | 계절 배수 |
|------|----------|
| 해당 계절이 작물의 **주 계절** (첫 번째 계절) | x1.1 (+10% 보너스) |
| 해당 계절이 작물의 **부 계절** (두 번째 계절, 해당 시) | x1.0 (보정 없음) |
| 여름 (모든 재배 가능 작물 공통) | 추가 x1.05 (더위 보너스) |
| 계절 불일치 | 성장 불가 (즉시 Withered 전환) |
| 온실 내부 — 해당 계절 작물 | x1.0 (계절 보정 무시, 패널티 없음) |
| 온실 내부 — 비계절 작물 | 성장 x0.85 (-> see `docs/content/facilities.md` 섹션 4.3) |
| 온실 내부 — 겨울 전용 작물 | x1.0 (온실이 본래 환경, 패널티 없음) |

### 2.5 종합 성장 공식

```
일일_유효_성장 = 물_제공 * 비료_배수 * 토양_배수 * 계절_배수

실제_성장일수 = ceil(기본_성장일수 / 종합_배수)

종합_배수 = 비료_배수 * 토양_배수 * 계절_배수
```

또는, 누적 성장 방식으로 표현하면:

```
매일 06:00:
  if 타일 == Watered:
    일일_성장량 = 1.0 * 비료_배수 * 토양_배수 * 계절_배수
    누적_성장량 += 일일_성장량
  
  if 누적_성장량 >= 기본_성장일수:
    단계 = Harvestable
```

**설계 결정**: 누적 방식(fractional growth)을 채택한다. 이유:
- 정수 기반 ceil 방식은 배수 효과가 불연속적이어서 비료 투자 효율이 불투명
- 소수점 누적 방식은 모든 보정이 선형적으로 작용하여 직관적

### 2.6 계산 예시

**예시 1**: 토마토 (5일) + 기본 비료(x1.25) + Normal 토양(x1.0) + 주 계절(x1.1)
```
일일 성장량 = 1.0 * 1.25 * 1.0 * 1.1 = 1.375
필요 일수 = ceil(5 / 1.375) = ceil(3.64) = 4일
```

**예시 2**: 수박 (12일) + 속성장 비료(x1.5) + Rich 토양(x1.2) + 여름 보너스(x1.05)
```
일일 성장량 = 1.0 * 1.5 * 1.2 * 1.05 = 1.89
필요 일수 = ceil(12 / 1.89) = ceil(6.35) = 7일
```

**예시 3**: 감자 (3일) + 비료 없음(x1.0) + Poor 토양(x0.9) + 주 계절(x1.1)
```
일일 성장량 = 1.0 * 1.0 * 0.9 * 1.1 = 0.99
필요 일수 = ceil(3 / 0.99) = ceil(3.03) = 4일 (1일 추가 소요)
```

## 3. 계절별 재배 제한 (Seasonal Restrictions)

### 3.1 작물-계절 매핑

(-> see `docs/systems/farming-system.md` 섹션 9.3, canonical)

| 작물 | 봄 | 여름 | 가을 | 겨울 | 비고 |
|------|:--:|:----:|:----:|:----:|------|
| 감자 | O | - | - | - | 봄 전용 |
| 당근 | O | - | O | - | 봄+가을 재배 가능 |
| 토마토 | - | O | - | - | 여름 전용 |
| 옥수수 | - | O | - | - | 여름 전용 |
| 딸기 | O | - | - | - | 봄 전용, 다중 수확 |
| 호박 | - | - | O | - | 가을 전용 |
| 해바라기 | - | O | - | - | 여름 전용 |
| 수박 | - | O | - | - | 여름 전용 |

### 3.2 계절 전환 시 미수확 작물 처리

계절 전환은 매 28일째 24:00에 발생한다 (-> see `docs/design.md` 섹션 4.3).

**기본 규칙**: 계절이 바뀌면, 새 계절에서 재배 불가능한 모든 야외 작물은 **즉시 Withered** 상태로 전환된다.

**유예 기간**: 없음. 즉시 고사가 기본 규칙이다.

**이유**: 유예 기간을 두면 계절 시스템의 긴장감이 약해지고, 플레이어가 계획 없이 플레이할 수 있게 되어 "의미 있는 선택" 원칙에 어긋남.

**완화 장치**:
1. **사전 경고**: 계절 전환 3일 전(25일째)부터 화면 상단에 경고 배너 표시
   - "3일 후 여름이 됩니다. 봄 작물을 수확하세요!"
   - 미수확 봄 작물이 있는 타일에 경고 아이콘(노란 삼각형) 표시
2. **TV/게시판 알림**: 21일째(1주일 전)부터 계절 전환 예고

**다중 계절 작물 처리** (예: 당근 = 봄+가을):
- 봄에 심은 당근이 여름에 미수확 → 즉시 Withered (여름은 당근 재배 불가)
- 가을이 돼도 부활하지 않음
- 가을에 다시 심어야 함

[OPEN] 다중 계절 작물이 중간 계절을 건너뛸 수 있는 메카닉(예: 당근이 여름에 휴면 후 가을에 다시 성장)을 추가할지 검토. 현재는 단순성 우선으로 즉시 고사 채택.

### 3.3 온실 규칙 (Greenhouse Rules)

온실(2000G, 레벨 5 해금 -> see `docs/design.md` 섹션 4.6)은 계절 제한을 무시한다.

| 규칙 | 내용 |
|------|------|
| 계절 무시 | 온실 내부에서는 **모든 계절에 모든 작물** 재배 가능 |
| 날씨 무시 | 온실 내부는 항상 맑음 취급 (비 자동 물주기 효과 없음, 폭풍 피해 없음) |
| 계절 보정 (해당 계절) | x1.0 (보정 없음) |
| 계절 보정 (비계절 작물) | 성장 x0.85, 품질 x0.9 (-> see `docs/content/facilities.md` 섹션 4.3 for canonical) |
| 계절 보정 (겨울 전용 작물) | x1.0 (패널티 없음, 온실이 본래 환경) |
| 판매가 보정 (비계절 작물) | x0.8 (-> see `docs/systems/economy-system.md` 섹션 2.6.5) |
| 판매가 보정 (겨울 전용 작물, 겨울 온실) | 시너지 x1.2 (-> see `docs/systems/economy-system.md` 섹션 2.6.5) |
| 물주기 | 수동으로 물을 줘야 함 (비 효과 없음) |
| 토양 | 온실 내부 토양은 Normal에서 시작, 일반 토양 규칙 동일 적용 |
| 크기 | 온실 내부 경작 가능 면적: 4x4 (16타일) |

**온실의 트레이드오프**:
- 장점: 겨울에도 재배 가능, 폭풍 피해 없음, 비계절 작물도 재배 가능 (단, 소폭 패널티), 겨울 전용 작물은 시너지 판매가 x1.2
- 단점: 비 자동 물주기 없음 (매일 수동 물주기 필요), 면적 제한, 높은 건설 비용, 비계절 작물 성장 15% 감속 + 판매가 x0.8

## 4. 수확 메카닉 (Harvest Mechanics)

### 4.1 단일 수확 작물 (Single Harvest)

대부분의 작물은 한 번 수확하면 작물이 사라지고 타일이 Tilled 상태로 돌아간다.

| 작물 | 유형 | 기본 수확량 |
|------|------|-----------|
| 감자 | 단일 수확 | 1개 |
| 당근 | 단일 수확 | 1개 |
| 토마토 | 단일 수확 | 1개 |
| 옥수수 | 단일 수확 | 1개 |
| 호박 | 단일 수확 | 1개 |
| 해바라기 | 단일 수확 | 1개 (+ 씨앗 1~2개 추가 드롭) |
| 수박 | 단일 수확 | 1개 |

### 4.2 다중 수확 작물 (Multi-Harvest / Regrowth)

일부 작물은 수확 후 작물이 사라지지 않고, Growing 단계로 돌아가 다시 열매를 맺는다.

| 작물 | 재수확 간격 | 최대 수확 횟수 | 비고 |
|------|-----------|--------------|------|
| 딸기 | 3일 | 계절 내 무제한 | 봄 동안 계속 수확 가능 |
| 표고버섯 | 5일 | 계절 내 무제한 | 온실(겨울) 전용, 첫 수확 6일 후 시작 (-> see `docs/content/crops.md` 섹션 3.10) |

**다중 수확 작물 흐름**:
```
[Seed] → [Sprout] → [Growing] → [Harvestable]
                                      │
                                      │ 수확
                                      ▼
                                 [Growing] ← 재성장 (재수확 간격만큼 성장)
                                      │
                                      ▼
                                 [Harvestable] → 반복...
```

- 수확 후 타일 상태는 Planted(Watered 가능)로 유지
- 재수확까지 물주기 필요 (다시 Dry/Watered 사이클)
- 비료 효과는 최초 적용 시 수확까지만 유지, 재수확 시에도 동일 비료 효과 지속
- 계절 전환 시 다중 수확 작물도 동일하게 고사

### 4.3 수확 품질 등급

(-> see `docs/systems/farming-system.md` 섹션 5.4, canonical)

| 품질 | 영문 키 | 가격 배수 | 비주얼 |
|------|---------|----------|--------|
| 일반 | `Normal` | x1.0 | 별 없음 |
| 은별 | `Silver` | x1.25 | 은색 별 1개 |
| 금별 | `Gold` | x1.5 | 금색 별 1개 |
| 이리듐 | `Iridium` | x2.0 | 보라색 별 1개 |

### 4.4 품질 결정 공식

수확 시 품질은 다음 공식으로 결정된다 (-> see `docs/systems/farming-system.md` 섹션 5.4).

```
품질_점수 = 토양_품질_보너스 + 비료_품질_보너스 + random(0, 0.3)
```

**토양 품질 보너스** (-> see `docs/systems/farming-system.md` 섹션 6.1):

| 토양 등급 | 품질 보너스 |
|----------|-----------|
| Poor | +0.0 |
| Normal | +0.1 |
| Fertile | +0.25 |
| Rich | +0.4 |

**비료 품질 보너스**:

| 비료 | 품질 보너스 |
|------|-----------|
| 없음 | +0.0 |
| 고급 비료 (Quality Fertilizer) | +0.3 |

**품질 판정 구간**:

```
품질_점수 < 0.3         → Normal
0.3 <= 품질_점수 < 0.6  → Silver
0.6 <= 품질_점수 < 0.9  → Gold
품질_점수 >= 0.9        → Iridium
```

**예시 1**: Normal 토양(+0.1) + 비료 없음(+0.0) + random(0~0.3)
- 범위: 0.1 ~ 0.4 → Normal 또는 Silver (운에 따라)

**예시 2**: Rich 토양(+0.4) + 고급 비료(+0.3) + random(0~0.3)
- 범위: 0.7 ~ 1.0 → Gold 또는 Iridium

**예시 3**: Poor 토양(+0.0) + 비료 없음(+0.0) + random(0~0.3)
- 범위: 0.0 ~ 0.3 → 거의 항상 Normal

### 4.5 수확 시 획득물

| 항목 | 수량/공식 |
|------|----------|
| 작물 아이템 | 기본 1개 (낫 업그레이드에 따라 확률적 +1) |
| 경험치 | `기본_판매가 * 0.5` (품질과 무관) |
| 씨앗 (해바라기 전용) | 1~2개 추가 드롭 |

(-> see `docs/systems/farming-system.md` 섹션 7.2, 낫 업그레이드 수확량 보너스)

## 5. 특수 작물 메카닉 (Special Crop Mechanics)

### 5.1 거대 작물 (Giant Crops)

특정 조건을 충족하면 인접한 3x3 범위의 같은 작물이 하나의 거대 작물로 변환된다.

**변환 조건**:
1. 3x3 범위(9타일)에 **같은 종류의 작물**이 심어져 있어야 함
2. 9타일 모두 **Harvestable** 단계에 도달해야 함
3. Harvestable 도달 후 **수확하지 않고 1일 경과** 시 변환 판정
4. 변환 확률: **15%** (매일 아침 배치 처리 시 판정)
5. 변환 실패 시 다음 날 다시 판정 (수확하지 않는 한 매일 기회)

**변환 가능 작물** (1차 릴리스 기준):

| 작물 | 거대 작물 가능 | 거대 크기 | 비고 |
|------|:------------:|----------|------|
| 호박 | O | 3x3 | 거대 호박 |
| 수박 | O | 3x3 | 거대 수박 |
| 기타 | X | - | 후속 업데이트에서 추가 검토 |

**거대 작물 수확**:
- 도끼(별도 도구, 현재 미정의)로 수확 → 해당 작물 아이템 **15~21개** 획득
- 품질: 일괄 Gold 이상 (Gold 70%, Iridium 30%)
- 9타일 모두 Tilled로 복귀
- 거대 작물 전용 수확 이펙트 (화면 흔들림 + 대형 파티클)

[OPEN] 거대 작물 수확에 도끼가 필요한데, 현재 도구 목록에 도끼가 없다. 낫으로 대체할지, 도끼를 추가할지 결정 필요.

### 5.2 교잡/돌연변이 (Crossbreeding / Mutation)

후반 콘텐츠로, 인접한 다른 작물 간 교잡이 발생할 수 있다.

**기본 메카닉**:
1. 2종류의 특정 작물이 **인접(상하좌우)** 배치된 상태에서 둘 다 Harvestable 도달
2. 다음 날 아침 배치 처리 시 **5% 확률**로 돌연변이 판정
3. 성공 시, 빈 인접 타일(있으면)에 돌연변이 작물 씨앗이 자동 출현

**교잡 레시피 예시** (후반 콘텐츠, 레벨 7+에서 해금):

| 부모 작물 A | 부모 작물 B | 결과 | 판매가 |
|------------|------------|------|--------|
| 토마토 | 감자 | 토감자 (Tomato-Potato) | 150G |
| 딸기 | 해바라기 | 황금딸기 (Golden Strawberry) | 200G |

**설계 의도**: 교잡은 발견의 재미를 위한 시스템. 효율적 수익원이 아닌, 도감 수집 + 희소성 콘텐츠.

[OPEN] 교잡 작물 목록과 레시피는 후속 콘텐츠 기획(CON-001)에서 확정. 현재는 시스템 프레임워크만 정의.

## 6. 실패 조건 (Failure Conditions)

### 6.1 물 미제공 고사 (Dehydration Wither)

(-> see `docs/systems/farming-system.md` 섹션 9.1)

| 조건 | 결과 |
|------|------|
| **3일 연속** Dry 상태 유지 | Withered 상태로 전환 |

**상세 규칙**:
- Dry 연속일수는 타일별로 추적 (`dryConsecutiveDays` 카운터)
- 물을 주면 카운터 리셋 (0으로)
- 비가 오면 자동 물주기 → 카운터 리셋
- 1일 건조, 2일 건조 시 시각적 경고 표시 (아래 6.4 참조)

### 6.2 계절 불일치 고사 (Season Mismatch Wither)

| 조건 | 결과 |
|------|------|
| 계절 전환 시 새 계절에서 재배 불가능한 작물 | **즉시** Withered 전환 |
| 재배 불가 계절에 씨앗 심기 시도 | 심기 자체는 가능하나, 다음 아침 즉시 Withered |

**참고**: farming-system.md 섹션 3.3에서 "현재 계절에 심을 수 없는 씨앗은 경고 표시 (심기 자체는 가능하지만 성장 불가)"로 정의. 본 문서에서는 이를 "다음 아침 배치 처리에서 계절 불일치 판정 → Withered"로 구체화한다.

### 6.3 폭풍 피해 (Storm Damage)

(-> see `docs/systems/farming-system.md` 섹션 8.2)

| 조건 | 결과 |
|------|------|
| 폭풍(Storm) 날씨일 때 야외 작물 | **10% 확률**로 성장 단계 -1 |

**상세 규칙**:
- Seed 단계에서 피해 시: Seed 유지 (더 낮은 단계 없음), 대신 성장일수 -1 (최소 0)
- Sprout → Seed로 후퇴
- Growing → Sprout로 후퇴
- Harvestable → Growing으로 후퇴 (수확 불가 상태로 복귀)
- Withered 작물은 폭풍 영향 없음 (이미 사망)
- 온실 내부 작물은 폭풍 피해 면역

### 6.4 건조 경고 시각 표현

| 건조 연속일수 | 시각 표현 |
|-------------|----------|
| 0일 (Watered) | 타일 색상 짙은 갈색 (촉촉) |
| 1일 Dry | 타일 색상 밝은 갈색, 작물 색상 약간 흐려짐 |
| 2일 Dry | 타일 색상 연한 회갈색, 작물 잎 끝이 노랗게 변색, 물방울 경고 아이콘(빨간색) |
| 3일 Dry | Withered 전환 (섹션 7.3 참조) |

## 7. 시각/청각 피드백 (Visual & Audio Feedback)

### 7.1 성장 시 피드백

| 이벤트 | 시각 효과 | 청각 효과 |
|--------|---------|----------|
| 단계 전환 (매일 아침) | 작물 주변에 **반짝이 파티클** (초록색 별 입자, 0.5초간 상승) | 부드러운 "띵" 음 (음높이가 단계별로 올라감) |
| Seed → Sprout | 흙이 갈라지며 새싹 솟아오르는 애니메이션 (0.3초) | "뽀록" 흙 갈라지는 소리 |
| Sprout → Growing | 줄기가 위로 뻗는 스케일 애니메이션 (0.4초) | "스르르" 성장 소리 |
| Growing → Harvestable | 열매가 "팝" 하고 나타나는 스케일 바운스 (0.3초) | "퐁" 열매 맺히는 소리 + 반짝이 징글 |

### 7.2 수확 시 피드백

| 이벤트 | 시각 효과 | 청각 효과 |
|--------|---------|----------|
| 단일 수확 | 작물이 위로 살짝 튀어오르며 사라짐 + 아이템 아이콘이 인벤토리 방향으로 날아감 + 품질 별 이펙트 | "뽑" 수확음 + 경쾌한 징글 |
| 다중 수확 (딸기) | 열매만 따지는 모션, 줄기/잎은 유지됨 + 열매 아이콘 날아감 | "따닥" 열매 따는 소리 |
| 거대 작물 수확 | 화면 미세 흔들림(0.2초) + 작물 파편 파티클 사방으로 + 대량 아이템 분출 | "쾅" 묵직한 수확음 + 성취 팡파레 |
| 품질별 추가 이펙트 | Normal: 없음 / Silver: 은색 빛줄기 / Gold: 금색 빛줄기 + 작은 반짝임 / Iridium: 보라색 빛줄기 + 화면 가장자리 보라 플래시 | Silver: 추가 "반짝" / Gold: "차랑" / Iridium: 전용 레어 징글 |

### 7.3 고사 시 피드백

| 이벤트 | 시각 효과 | 청각 효과 |
|--------|---------|----------|
| Withered 전환 | 작물 색상이 0.5초에 걸쳐 **갈색/회색으로 변색** + 시들어 축 늘어지는 스케일 변형 (Y축 0.7배) + 갈색 잎 파티클 낙하 | "시들" 바스락거리는 소리 |
| Withered 타일 지속 상태 | 작물이 축 처진 채로 유지, 주기적 바스락 파티클 (5초마다) | 없음 (지속 소리 없음) |
| Withered 제거 (낫) | 고사체가 부서지며 사라짐 + 갈색 먼지 파티클 | "바스락" 마른 작물 부서지는 소리 |

### 7.4 물주기/비료 시각 피드백

| 이벤트 | 시각 효과 |
|--------|---------|
| 물주기 | 물방울 파티클 + 타일 색상 어두워짐 (-> see farming-system.md 섹션 10.1) |
| 비료 주기 | 비료 종류별 입자 색상 스프레이 (-> see farming-system.md 섹션 5.2) |
| 비 자동 물주기 | 전체 농장에 빗방울 파티클 + 모든 경작 타일 점진적 색상 변화 |

## Open Questions

1. [OPEN] **비료 추가 등급**: farming-system.md canonical 기준 성장 속도를 올리는 비료는 기본(x1.25)과 속성장(x1.5) 두 종류뿐이다. 상위 등급 비료(x1.75, x2.0)를 추가할지 여부는 CON-001에서 확정 필요.

2. [OPEN] **거대 작물 수확 도구**: 거대 작물 수확에 도끼가 필요하나, 현재 도구 목록(호미/물뿌리개/씨앗봉투/낫/손)에 도끼가 없다. 낫으로 대체하거나, 도끼를 도구 목록에 추가해야 한다.

3. [OPEN] **다중 계절 작물의 중간 계절 생존**: 당근(봄+가을)이 여름에 휴면 후 가을에 부활하는 메카닉 추가 여부. 현재는 즉시 고사 채택.

4. [OPEN] **교잡 작물 목록**: 현재 예시 2종만 있음. 최종 목록은 CON-001에서 확정 필요. 교잡이 스코프 내 핵심 기능인지, 후반 보너스 콘텐츠인지 명확히 해야 한다.

5. [OPEN] **다중 수확 작물 추가 후보**: 현재 딸기만 다중 수확. 토마토, 옥수수 등 다른 작물도 다중 수확으로 전환할지 검토 필요 (design.md OPEN과 동일).

## Risks

1. [RISK] **성장 공식 복잡도**: 비료 x 토양 x 계절 3중 배수의 조합이 밸런스 예측을 어렵게 한다. Rich 토양(x1.2) + 속성장 비료(x1.5) + 주 계절(x1.1) = x1.98 배수로, 12일 수박이 7일 만에 자란다. 극단적 케이스의 밸런스 검증 필요 (BAL-001).

2. [RISK] **거대 작물 수익 밸런스**: 9타일 투자 대비 15~21개(Gold+) 수확은 매우 높은 수익. 일반 재배 대비 과도한 이점이 되지 않도록 수치 조정 필요 (BAL-001에서 검증).

3. [RISK] **초보자 정보 과부하**: 성장 단계, 비료 효과, 토양 품질, 계절 보정을 한꺼번에 노출하면 신규 플레이어가 압도당할 수 있음. 레벨별 단계적 해금과 정보 공개가 필수.

4. [RISK] **교잡 시스템 스코프**: 교잡/돌연변이는 구현 복잡도가 높고, 핵심 루프에 필수가 아니다. 스코프 외 판정 시 문서에서 제거하거나 "Phase 2+" 마킹 필요.

5. [RISK] **다중 수확 작물 경제 영향**: 딸기(80G, 3일 재수확)는 초기 투자 후 지속 수익이 발생하여 단일 수확 작물 대비 압도적 효율을 가질 수 있음. BAL-001에서 ROI 비교 분석 필요.
