# 낚시 시스템 (Fishing System) 상세 설계

## Context

이 문서는 SeedMind의 낚시 시스템을 상세히 기술한다. Zone F(연못 구역)의 해금(-> see `docs/systems/farm-expansion.md` 섹션 1.3, 2.1)과 연계되어 플레이어에게 경작 외의 새로운 수입원과 활동을 제공한다.

**설계 목표**: 낚시는 작물 경작의 "반복 노동" 성격과 대비되는 "집중 + 보상" 성격의 활동이다. 짧은 시간에 높은 집중도를 요구하고, 성공하면 즉각적인 보상을 주는 구조로, 경작/축산의 "기다림" 루프와 상호 보완한다. 단, 낚시만으로 경작을 대체할 수 있을 정도의 수익을 내서는 안 되며, "보조 수입 + 가공 재료 + 콘텐츠 다양성" 역할에 머물러야 한다.

## 1. 낚시 도구 (Fishing Rod)

### 1.1 낚싯대 등급

| 등급 | 영문 ID | 획득 방법 | 해금 조건 | 비용 | 효과 |
|------|---------|----------|----------|------|------|
| 기본 낚싯대 | `rod_basic` | Zone F 해금 시 자동 지급 | Zone F 해금 (레벨 5, 3,000G) | 0G (지급) | 기본 미니게임 난이도 |
| 강화 낚싯대 | `rod_reinforced` | 대장장이 업그레이드 | 낚시 숙련도 Lv.3 | 1,500G + 구리 광석 x5 | 게이지 이동 속도 -15%, 물고기 저항 -10% |
| 전설 낚싯대 | `rod_legendary` | 대장장이 업그레이드 | 낚시 숙련도 Lv.7 | 4,000G + 금 광석 x3 | 게이지 이동 속도 -30%, 물고기 저항 -20%, 보물 상자 발견 확률 +10% |

**설계 의도**: 도구 업그레이드 시스템(-> see `docs/systems/tool-upgrade.md`)과 동일한 3단계 구조를 따른다. 기본 낚싯대는 Zone F 해금과 동시에 제공하여 추가 투자 없이 즉시 낚시를 시작할 수 있게 한다.

[OPEN] 강화/전설 낚싯대의 재료(구리 광석, 금 광석)는 광산 시스템 미설계 상태. 광산이 없을 경우 대체 재료(목재 x30, 철 조각 x10 등) 검토 필요. 또는 여행 상인(-> see `docs/content/npcs.md`)에게서 광석 구매 가능하게 할 수 있다.

### 1.2 낚싯대 인벤토리 규칙

- 낚싯대는 도구 슬롯에 장착 (-> see `docs/systems/inventory-system.md`)
- 스택 불가 (도구와 동일)
- 업그레이드 시 기존 등급 소멸, 상위 등급으로 교체

### 2.1 낚시 가능 위치

**Zone F 연못 상세**: 96타일 중 30타일이 물 타일(개간/경작 불가). 물 타일 가장자리에 인접한 육지 타일에서 물을 향해 낚싯대를 사용할 수 있다. 유효 낚시 포인트는 약 20개소 (물리적으로 낚싯대를 던질 수 있는 육지 타일 수. 아키텍처 상 이 20개소는 3개의 FishingPoint MonoBehaviour 오브젝트가 구역별로 관할한다 — -> see `docs/systems/fishing-architecture.md` 섹션 4, 8.1).

### 2.2 낚시 프로세스 플로우

```
[플레이어] 물 타일 인접 육지에서 낚싯대 장착
    |
    v
[마우스 좌클릭] 캐스팅 (찌 던지기)
    |  에너지 소모: 3 (-> see 섹션 2.3)
    v
[대기 단계] 찌가 물 위에서 흔들림 (3~15초 랜덤)
    |  ├─ 찌 미세한 움직임 (페이크 바이트): 1~3회
    |  └─ 진짜 바이트: 찌가 확연히 가라앉음 + "!" 아이콘 + 효과음
    v
[바이트 윈도우] 0.5~1.5초 내 좌클릭 필요
    |
    ├─ [타임아웃] "놓쳤다!" 피드백 → 물고기 도주 → 재캐스팅 가능
    |
    └─ [성공 클릭] → 미니게임 진입
         |
         v
    [미니게임] 타이밍 게이지 (섹션 3 상세)
         |
         ├─ [성공] 물고기 획득 → 인벤토리 추가 + 낚시 XP
         |
         └─ [실패] "도망쳤다..." 피드백 → 에너지 1 추가 소모
```

### 2.3 에너지 소모

낚시 에너지 소모 수치(캐스팅, 미니게임 실패 추가 소모, 숙련도 Lv.8+ 감소)는 canonical 문서를 참조한다.
→ see `docs/systems/energy-system.md` 섹션 2.2

**설계 의도**: 낚시 1회 시도에 에너지 3~4 소모. 하루 최대 에너지 기준(→ see `docs/systems/energy-system.md` 섹션 1.1), 농작업에 에너지 40~50을 쓴다고 가정하면 낚시에 16~20회 시도 가능. 이는 "보조 활동"으로서의 포지셔닝에 적합하다. (BAL-024: 낚시+채집 vs 작물+가공 비율 65% 이하 달성 목적으로 에너지 상향 확정 → see `docs/balance/fishing-gathering-ratio.md`)

### 2.4 낚시 소요 시간

| 단계 | 소요 시간 (게임 내) | 실시간 환산 |
|------|-------------------|------------|
| 캐스팅 애니메이션 | ~2초 | ~2초 |
| 대기 (바이트 전) | 3~15초 | ~3~15초 (게임 시간 정지 아님, 그대로 진행) |
| 바이트 반응 | 0.5~1.5초 | ~0.5~1.5초 |
| 미니게임 | 5~20초 (어종에 따라) | ~5~20초 |

**1회 낚시 평균 소요**: 게임 내 약 15~30초 = 실시간 약 15~30초. 게임 내 시간으로 환산하면 약 7.5~15분(게임 내 1시간 = 실시간 33초 기준). 즉, 낚시 1회에 게임 내 약 8~15분 소요.

[OPEN] 낚시 중 게임 시간이 흐르는 것이 적절한지 검토 필요. 미니게임 진행 중에는 시간 정지(인벤토리/상점과 동일 처리)하고, 대기 단계에서만 시간이 흐르게 하는 방식도 고려 가능.

## 3. 미니게임 메카닉 (ExcitementGauge System)

### 3.1 방식 선정: 흥분 게이지 (ExcitementGauge)

**채택**: ExcitementGauge — 세로 게이지에서 물고기 타깃 존을 따라가는 방식
**불채택**: Oscillating Bar(가로 커서), 버튼 연타, QTE 시퀀스

**선정 근거**:

| 기준 | ExcitementGauge | Oscillating Bar | 버튼 연타 |
|------|----------------|----------------|---------|
| 접근성 | 높음 (홀드/릴리즈) | 높음 (클릭 타이밍) | 중간 |
| 깊이 | 높음 (예측 + 반응 혼합) | 중간 (타이밍만) | 낮음 |
| 스코프 | 작음 (UI 1개) | 작음 | 작음 |
| 반복 피로도 | 낮음 (지속 홀드) | 중간 (반복 클릭) | 높음 |
| 어종별 차별화 | 용이 (`targetZoneWidthMul`, `moveSpeed`) | 제한적 | 어려움 |

**FishData SO 정합성**: FishData SO 필드 `targetZoneWidthMul`(타깃 존 너비 배율)과 `moveSpeed`(타깃 존 이동 속도)가 ExcitementGauge와 직접 대응. 기술 아키텍처 `(→ see docs/systems/fishing-architecture.md 섹션 2)`와 완전 정합.

### 3.2 미니게임 규칙

```
┌───────────────────────────────────┐
│  [ExcitementGauge]                │
│                                   │
│  ┌──┐ ← 세로 게이지               │
│  │  │                             │
│  │▓▓│ ← 타깃 존 (물고기, 상하 이동)│
│  │▓▓│                             │
│  │  │                             │
│  │██│ ← 내 커서 (입력으로 상승)    │
│  │  │                             │
│  └──┘                             │
│                                   │
│  [흥분도] ██████████░░░  (65%)     │
│  홀드: 커서 상승 / 릴리즈: 하강    │
└───────────────────────────────────┘
```

**게이지 구성요소**:

| 요소 | 설명 | 파라미터 |
|------|------|---------|
| 내 커서 | 입력(홀드) 시 상승, 릴리즈 시 하강. 내 위치를 나타냄 | `fillRate`, `decayRate` |
| 타깃 존 | 물고기 아이콘. 게이지 내에서 상하로 이동 | `targetZoneWidthMul`, `moveSpeed` |
| 흥분도 바 (ExcitementGauge) | 커서가 타깃 존 안에 있는 시간만큼 누적. 100% 도달 시 성공 | `successThreshold` |
| 실패 임계값 | 흥분도가 일정 이하로 떨어지면 물고기 도주 | `failThreshold` |

**기본 메카닉**:
1. 세로 게이지에 타깃 존(물고기)이 표시되고 상하로 이동
2. 입력(마우스 홀드 / 모바일 탭 유지) → 내 커서 상승 (`fillRate`)
3. 입력 해제 → 내 커서 하강 (`decayRate`)
4. 커서가 타깃 존 안에 있을 때 → 흥분도 증가
5. 커서가 타깃 존 밖에 있을 때 → 흥분도 감소
6. 흥분도 `successThreshold` 도달 → 낚시 성공
7. 흥분도 `failThreshold` 이하 또는 제한 시간 초과 → 낚시 실패

### 3.3 난이도 파라미터

| 파라미터 | 영문 키 | Common | Uncommon | Rare | Legendary |
|----------|---------|--------|----------|------|-----------|
| 타깃 존 너비 배율 | `targetZoneWidthMul` | 1.2x | 1.0x | 0.75x | 0.5x |
| 타깃 존 이동 속도 | `moveSpeed` | 1.0 | 1.5 | 2.2 | 3.0 |
| 흥분도 증가율 | `fillRate` | 0.20/초 | 0.16/초 | 0.12/초 | 0.08/초 |
| 흥분도 감소율 | `decayRate` | 0.08/초 | 0.12/초 | 0.18/초 | 0.25/초 |
| 성공 임계값 | `successThreshold` | 0.80 | 0.85 | 0.90 | 0.95 |
| 실패 임계값 | `failThreshold` | 0.00 | 0.05 | 0.10 | 0.15 |
| 제한 시간 | `timeLimit` | 15초 | 20초 | 25초 | 30초 |

**이동 패턴**: Common은 단순 상하 진자 이동. Uncommon+는 불규칙 속도 변화. Rare+는 급격한 방향 전환 추가. Legendary는 무작위 순간 이동 포함.

### 3.4 낚싯대 등급 보정

| 파라미터 | 기본 낚싯대 | 강화 낚싯대 | 전설 낚싯대 |
|----------|-----------|-----------|-----------|
| `moveSpeed` 배수 | x1.0 | x0.85 | x0.70 |
| `decayRate` 배수 | x1.0 | x0.90 | x0.80 |
| `targetZoneWidthMul` 추가 | +0 | +0.1 | +0.2 |

**최종 난이도 = 어종 기본 난이도 x 낚싯대 보정**. 예: Legendary 어종(moveSpeed 3.0) + 전설 낚싯대(x0.70) = 실질 moveSpeed 2.1.

## 4. 어종 목록

### 4.1 어종 분류 원칙

- **4단계 희귀도**: Common / Uncommon / Rare / Legendary
- 계절별 3~4종이 주력 등장하되, 일부 어종은 다중 계절에 걸침
- Legendary 어종은 특수 조건(계절+시간대+날씨)이 동시 충족되어야 등장

## 5. 낚시 시간/조건 시스템

### 5.1 시간대별 영향

낚시에서 시간대는 두 가지 방식으로 영향을 미친다: (1) 등장 어종 필터링, (2) 바이트 대기 시간 보정.

| 시간대 | 영문 키 (-> see `docs/systems/time-season.md` 섹션 1.2) | 바이트 대기 시간 배수 | 특징 |
|--------|------|-----|------|
| 새벽 (Dawn) | `Dawn` | x0.7 (빠름) | 물고기 활성도 최고. 희귀종 출현 시간대 |
| 아침 (Morning) | `Morning` | x0.85 | 물고기 활성도 높음 |
| 낮 (Afternoon) | `Afternoon` | x1.0 (기본) | 기본 활성도 |
| 저녁 (Evening) | `Evening` | x1.1 | 활성도 약간 낮음, 야행성 어종 등장 시작 |
| 밤 (Night) | `Night` | x1.3 (느림) | 활성도 낮지만, 야행성 희귀종 전용 시간대 |

**설계 의도**: 새벽(Dawn)에 낚시하면 바이트가 빨리 오므로 시간 효율이 좋다. 이는 "일찍 일어나는 농부"에게 보상을 주면서, 아침 농작업과의 시간 경쟁("새벽에 낚시할까, 농작업부터 할까")을 만든다.

### 5.2 날씨별 영향

| 날씨 | 영문 키 (-> see `docs/systems/time-season.md` 섹션 3.1) | 희귀종 확률 보정 | 특수 효과 |
|------|------|-----|------|
| Clear | `Clear` | x1.0 (기본) | 없음 |
| Cloudy | `Cloudy` | x1.1 | 없음 |
| Rain | `Rain` | x1.3 | 일부 비 전용 어종 활성화 (메기, 뱀장어) |
| HeavyRain | `HeavyRain` | x1.5 | 비 전용 어종 활성화 + 바이트 대기 시간 -20% |
| Storm | `Storm` | x2.0 | Legendary 어종 조건 충족 가능, 에너지 소모 +25% |
| Snow | `Snow` | x1.2 | 겨울 전용 어종 활성화 (얼음 낚시) |
| Blizzard | `Blizzard` | x0.0 (불가) | 낚시 불가 (폭설 시 연못 접근 불가) |

**희귀종 확률 산출 공식**:
```
실제_희귀종_등장확률 = 기본_희귀도_확률 * 날씨_보정 * 숙련도_보정
```

### 5.3 어종별 등장 확률 (기본)

바이트 발생 시, 해당 시점의 조건(계절, 시간대, 날씨)을 충족하는 어종 풀에서 희귀도 기반 가중치 뽑기를 수행한다.

| 희귀도 | 기본 가중치 | 비고 |
|--------|-----------|------|
| Common | 60 | 어종 풀 내 Common 전체에 분배 |
| Uncommon | 25 | 어종 풀 내 Uncommon 전체에 분배 |
| Rare | 12 | 어종 풀 내 Rare 전체에 분배 |
| Legendary | 3 | 어종 풀 내 Legendary 전체에 분배 |

**예시**: 여름 낮(Afternoon) 맑은 날(Clear) 기본 풀 = 붕어(C), 미꾸라지(C), 잉어(C), 블루길(C), 가재(U), 황금 잉어(R), 연꽃 잉어(L).
- Common 4종: 각 60/4 = 15
- Uncommon 1종: 25
- Rare 1종: 12
- Legendary 0종 (연꽃 잉어는 Dawn 전용이므로 Afternoon에 미등장)
- 합계: 60+25+12 = 97 -> 각 어종의 확률 = 가중치/97

## 6. 수확물 활용

### 6.1 직접 판매

모든 생선은 출하함 또는 상점에 판매 가능하다. 가격은 섹션 4.2의 기본 판매가를 따르며, 경제 시스템의 마진 비율(-> see `docs/systems/economy-system.md` 섹션 2.1)은 적용되지 않는다 (생선은 상점에서 구매하는 아이템이 아니므로).

**수급 변동**: 생선은 `SupplyCategory.Fish` 카테고리로 완화된 수급 변동이 적용된다 (saturationThreshold 30, supplyDropRate 0.01, minSupplyMultiplier 0.8). 주당 30마리 이하 판매 시 수급 영향 없음 (-> see `docs/systems/economy-system.md` 섹션 2.6.2.1).

**품질 시스템 적용**: 생선에도 작물과 동일한 4단계 품질을 적용한다(-> see `docs/systems/crop-growth.md` 섹션 4.3 — Normal/Silver/Gold/Iridium). 아키텍처 문서(ARC-026)의 FishingConfig.qualityThresholds도 4단계 기준.

| 품질 | 영문 키 | 판매가 배수 | 결정 조건 |
|------|---------|-----------|----------|
| 일반 (Normal) | `Normal` | x1.0 | 기본 |
| 은별 (Silver) | `Silver` | x1.25 | 낚시 숙련도 Lv.3+, 미니게임 진행도 100% 도달 시 잔여 시간 50%+ |
| 금별 (Gold) | `Gold` | x1.5 | 낚시 숙련도 Lv.6+, 미니게임 진행도 100% 도달 시 잔여 시간 70%+ |
| 이리듐 (Iridium) | `Iridium` | x2.0 | 낚시 숙련도 Lv.9+, 미니게임 진행도 100% 도달 시 잔여 시간 85%+ |

[OPEN] 이리듐 생선 달성 조건(잔여 시간 85%)이 Legendary 어종 난이도와 충돌할 수 있다. 밸런스 시트(BAL) 검증 필요.

**품질 판정 로직**: 미니게임 완료 시 잔여 시간 비율을 기준으로 판정. 빠르게 완료할수록 높은 품질.

### 6.2 가공 연계

생선은 가공소 및 베이커리에서 가공 가능하다. 생선 가공 레시피 5종(구운 생선, 훈제 생선, 생선 초밥, 생선 스튜, 생선 파이)의 상세 사양(재료, 판매가 공식, 가공 시간, 해금 조건)은 (-> see `docs/content/processing-system.md` 섹션 3.5).

생선 가공의 경제 밸런스(ROI, 시간당 부가가치)는 (-> see `docs/balance/fishing-economy.md` 섹션 4).

### 6.3 요리 재료 활용

- 모든 생선은 가공소/베이커리의 재료로 사용 가능
- NPC 선물로 제공 가능 (호감도 시스템 연계 시)
- 일부 퀘스트의 납품 대상 (섹션 8.3 참조)

### 6.4 보물 상자 (Treasure Chest)

낚시 시 극히 낮은 확률로 물고기 대신 보물 상자를 건질 수 있다.

## 7. 낚시 숙련도 시스템

### 7.1 설계 결정: 독립 숙련도

**채택**: 낚시 전용 숙련도 레벨 (메인 레벨과 별도)
**불채택**: 메인 레벨 시스템에 통합

**근거**:
- 메인 레벨(총 10레벨, -> see `docs/design.md` 섹션 4.5)은 작물 경작 중심으로 이미 XP 배분이 완료됨 (-> see `docs/balance/xp-integration.md`)

- 독립 숙련도는 "낚시에 시간을 투자한 플레이어"에게 명확한 성장 피드백 제공
- 다른 보조 활동(채집, 광산 등) 추가 시에도 같은 독립 숙련도 패턴을 적용 가능

### 8.2 어종 도감

- 포획한 어종은 자동으로 **어종 도감**에 등록
- 도감 항목: 어종 이미지, 이름, 희귀도, 등장 조건 힌트, 최대 크기 기록, 포획 횟수
- 도감 완성도는 % 표시 (15종 기준)
- 도감 100% 완성 + 숙련도 Lv.10 = "낚시 마스터" 칭호

| 계절 한정 수집 | "이번 가을에 무지개송어를 잡으세요" | 300G + 희귀 미끼 | 계절 1회 |
| 대물 도전 | "Giant 잉어를 잡으세요" | 500G | 특별 퀘스트 |

[OPEN] 미끼(Bait) 아이템 시스템을 별도로 설계할지 검토 필요. 미끼를 도입하면 "미끼 제작/구매 -> 희귀종 확률 증가"라는 추가 투자-보상 루프가 생기지만, 스코프가 커질 수 있다.

## 9. 밸런스 분석

### 9.1 낚시 수익 포지셔닝

**핵심 원칙**: 낚시는 "보조 수입원"이다. 같은 시간을 투자했을 때 경작보다 낮되, 가공 연계 시 경쟁력이 생기는 구조.

#### 9.1.1 시간당 수익 비교

**낚시 수익 시뮬레이션** (숙련도 Lv.1, 기본 낚싯대 기준):

```
1회 낚시 평균 소요: 실시간 ~20초
1시간(게임 내) = 실시간 33초 -> 약 1.5회/게임시간
1시간 동안 낚시 수익 (평균):
  Common(60%) x 평균25G = 15G
  Uncommon(25%) x 평균53G = 13.3G
  Rare(12%) x 평균163G = 19.5G
  Legendary(3%) x 평균650G = 19.5G
  기대값/회 = 약 67G (가중 평균)

  단, 실전에서 Legendary는 특수 조건이므로 일반 낚시에서는:
  Common(70%) x 25G = 17.5G
  Uncommon(25%) x 53G = 13.3G
  Rare(5%) x 163G = 8.2G
  실질 기대값/회 = 약 39G

1게임시간 수익: 39G x 1.5회 = ~58G/게임시간
```

**작물 수익 비교** (-> see `docs/balance/crop-economy.md` 섹션 1.2):

```
타일 12칸 경작 기준 (중급 플레이어):
  옥수수 12칸: 순이익 50G x 12 = 600G / 7일 = 85.7G/일
  1일 = 18 게임시간 중 농작업 ~4시간 -> 600G/7일/4시간 = ~21.4G/게임시간

  단, 작물은 물주기 후 방치이므로 실제 작업 시간은 하루 10~20분(1~2게임시간)
  농작업 시간당 실질 수익: 600G/7일/1.5시간 = ~57G/게임시간
```

**비교 결론**:
| 활동 | 시간당 수익 (게임시간 기준) | 특성 |
|------|--------------------------|------|
| 낚시 (Lv.1) | ~58G/게임시간 | 능동적 참여 필요, 즉시 수익 |
| 낚시 (Lv.5) | ~75G/게임시간 | 숙련도 보정 적용 |
| 작물 경작 (옥수수) | ~57G/게임시간 (실작업 기준) | 수동 관리 후 자동 성장 |
| 작물 가공 (잼) | ~114G/게임시간 (원재료+가공시간 환산) | 후반 해금, 복합 투자 |

**포지셔닝 판정**: 낚시는 실작업 시간 기준으로 경작과 유사하지만, 경작은 "물주기 후 다른 활동 가능"이라는 이점이 있다. 즉, 낚시는 "전념 시간" 대비 수익이 경작의 병행 수익보다 약간 높게 설정되어 "경작 대기 시간에 낚시를 하면 효율적"이라는 자연스러운 플레이 패턴을 유도한다.

[RISK] 숙련도가 올라갈수록 낚시 시간당 수익이 경작을 크게 초과할 수 있다. Lv.10 + 전설 낚싯대 조합에서 시간당 수익이 100G+에 도달하면 "낚시만 하는 것이 최적"이 될 위험. 정기적으로 밸런스 시트(BAL 문서)에서 검증 필요.

#### 9.1.2 에너지 효율 비교

| 활동 | 에너지 소모/회 | 수익/회 | 골드/에너지 |
|------|-------------|--------|-----------|
| 낚시 (Lv.1) | 2~3 | ~39G | ~15G/에너지 |
| 낚시 (Lv.8+) | 1~2 | ~50G | ~33G/에너지 |
| 물주기 (기본 물뿌리개) | 2 | ~5G (타일당 일일 기여) | ~2.5G/에너지 |
| 수확 | 2 | ~30~100G (작물에 따라) | ~25G/에너지 |

**관찰**: 에너지 효율 측면에서 낚시는 수확과 비슷하고, 물주기보다 훨씬 효율적이다. 그러나 물주기는 미래 수확을 위한 "투자"이므로 단순 비교는 부적절하다.

> (BAL-013 적용 후 일일 총수익은 -> see `docs/balance/fishing-economy.md` 섹션 3 참조)

## 10. 겨울 얼음 낚시 (Ice Fishing)

### 10.1 얼음 낚시 메카닉

겨울에 Zone F 연못이 결빙되면, 일반 물 타일 위에서 **얼음 구멍 뚫기 -> 낚시** 프로세스가 활성화된다.

| 단계 | 행동 | 에너지 | 비고 |
|------|------|--------|------|
| 1. 얼음 구멍 뚫기 | 곡괭이로 물 타일 클릭 (1회) | 3 | 곡괭이 필요 (Basic 이상) |
| 2. 구멍 유지 | 3일간 유효, 이후 재결빙 | - | 눈 내리는 날은 다음 날 재결빙 |
| 3. 낚시 | 일반 낚시와 동일 프로세스 | 2~3 | 미니게임 동일 |

**얼음 낚시 제한**:
- 동시에 유지 가능한 얼음 구멍: 최대 3개
- Blizzard 날에는 낚시 불가 (시야 및 이동 제한)
- 겨울 어종만 등장 (빙어, 얼음 빙어왕)

**설계 의도**: 겨울에 "할 것이 없다"는 문제(time-season.md [OPEN] 항목 5, 현재 [RESOLVED])를 부분적으로 해결한다. 겨울 낚시는 어종이 2종뿐이라 수익성은 낮지만, 겨울에도 야외 활동이 가능하다는 것 자체가 플레이어 만족에 기여한다.

## Risks

1. [RISK] **낚시 수익 밸런스 붕괴**: 숙련도/장비 풀업 시 낚시만으로 경작 대비 과도한 수익을 올릴 위험. 특히 Legendary 어종(500G~800G)의 반복 포획 시. 대응: 숙련도 보정에 상한선 설정, Legendary 출현 조건을 매우 제한적으로 유지.
2. [RISK] **미니게임 반복 피로**: 수백 회 반복 시 타이밍 게이지가 지루해질 수 있음. 대응: 어종별 난이도 차이, 숙련도 성장에 따른 체감 난이도 감소, 추후 미니게임 변형 추가 고려.
3. [RISK] **겨울 콘텐츠 의존**: 겨울 활동을 낚시에 의존하면 낚시 시스템 자체가 지연될 때 겨울이 빈 시즌이 됨. 대응: 온실 경작, 가공소 운영, NPC 교류 등 낚시 외 겨울 활동도 확보.
4. [RISK] **어종 수 확장성**: 15종은 Zone F 연못만을 위한 것. 강/바다 확장 시 어종 목록 대폭 증가 필요. 대응: 어종 데이터를 SO(ScriptableObject)로 외부화하여 콘텐츠 추가 용이하게.
