# 수집 도감 시스템 (Collection System) 상세 설계

## 1. Context

이 문서는 SeedMind의 수집 도감(Collection) 시스템을 통합적으로 설계한다. 기존 어종 도감(fish-catalog.md, CON-011)과 새로 추가되는 채집 도감을 단일 UI 프레임워크 아래 통합하여, 플레이어에게 일관된 "수집의 즐거움"을 제공한다.

**설계 목표**: 수집 도감은 발견/수집/완성이라는 보편적 동기를 활용하여 반복 플레이의 목적성을 부여한다. 어종 도감이 "낚시"에, 채집 도감이 "채집"에 각각 장기 목표를 제공하며, 통합 도감 UI를 통해 전체 수집 진행도를 한눈에 파악할 수 있게 한다.

**이 문서가 해소하는 OPEN 항목**: `docs/systems/gathering-system.md` 섹션 9, OPEN#3 — "채집 아이템 27종의 도감을 별도 탭으로 구현할지, 기존 어종 도감과 통합 '도감' 시스템으로 관리할지 결정 필요."

### 1.1 본 문서가 canonical인 데이터

- 통합 도감 vs 분리 도감 설계 결정 및 근거
- 채집 도감 항목 구조 (어종 도감과의 차이점 포함)
- 채집 아이템 27종 도감 항목 (초회 채집 보상 — 희귀도별 보상 기준 및 아이템별 보상 테이블)
- 채집 도감 완성 마일스톤 보상 (10/20/27종)
- 통합 도감 UI 구조 (탭, 필터, 정렬, 진행률)
- 채집 도감 장식 보상 아이템 정의
- 채집 마스터 칭호 해금 조건 (도감 100% + 숙련도 Lv.10)

## 2. 설계 결정: 통합 도감 채택 근거

### 2.1 선택지 비교

| 관점 | A: 통합 도감 (단일 UI, 탭 분리) | B: 분리 도감 (독립 UI 2개) |
|------|-------------------------------|--------------------------|
| UX 일관성 | 하나의 진입점에서 모든 수집 현황 확인 | 메뉴 항목 2개로 분산, 인지 부하 증가 |
| 전체 수집 진행도 | 통합 진행률 바(42종 중 N종) 표시 가능 | 어종/채집 각각의 진행률만 확인 가능 |
| 아키텍처 영향 | 공통 도감 프레임워크 1개 + 탭별 어댑터 | 독립 매니저 2개 (FishCatalogManager + GatherCatalogManager) |
| 확장성 | 향후 카테고리 추가(요리, 곤충 등) 시 탭만 추가 | 카테고리마다 새 UI/매니저 필요 |
| 구현 공수 | 공통 UI 프레임워크 설계 필요 (초기 비용 약간 높음) | 기존 FishCatalogManager 복제로 빠른 구현 |
| 기존 설계 호환성 | fish-catalog.md의 UI 구조(섹션 5)를 통합 프레임워크로 흡수 | fish-catalog.md 변경 없음 |

### 2.2 결정

**채택: 방안 A — 통합 도감 (단일 UI, 탭 분리)**

**근거**:
1. **UX 우위**: 플레이어가 "내가 얼마나 수집했는가"를 한 화면에서 확인할 수 있다. 수집 게임의 핵심 만족감은 "전체 완성도"에서 오며, 이를 분산시키면 효과가 반감된다.
2. **확장성**: 향후 요리 도감, 광물 도감 등이 추가될 가능성이 높다. 탭 기반 통합 구조는 카테고리 추가 시 UI 변경 최소화.
3. **아키텍처 비용 수용 가능**: 기존 FishCatalogManager를 공통 인터페이스(ICatalogProvider)로 추상화하고, GatherCatalogManager를 동일 인터페이스로 구현하면 된다. 초기 리팩터링 비용은 있으나 장기적으로 유지보수에 유리.

**불채택 근거 (방안 B)**:
- 기존 FishCatalogManager를 그대로 유지할 수 있어 단기 구현은 빠르지만, 채집 도감용 GatherCatalogManager를 별도 구현해야 하므로 코드 중복 발생.
- 향후 카테고리 확장 시마다 독립 UI를 만들어야 하는 기술 부채 누적.

### 2.3 기존 fish-catalog.md와의 관계

fish-catalog.md(CON-011)의 **콘텐츠 데이터**(어종 항목, 힌트 텍스트, 크기 범위, 초회 보상, 마일스톤 보상)는 그대로 유효하다. 본 문서는 fish-catalog.md의 UI 구조(섹션 5)를 통합 도감 프레임워크로 흡수하되, 콘텐츠 canonical은 fish-catalog.md에 유지한다.

**변경 영향**:
- fish-catalog.md 섹션 5.1의 "메뉴 > 도감 > 어종 도감 탭" → "메뉴 > 수집 도감 > 어종 탭"으로 경로 변경 필요
- fish-catalog.md 섹션 5.4의 진행률 표시가 통합 도감의 탭 내부 진행률로 위치 변경

[OPEN] fish-catalog.md 섹션 5의 UI 경로 문구 수정은 별도 FIX 태스크로 처리해야 한다.

## 3. 도감 공통 구조

### 3.1 도감 항목 상태 모델

어종 도감과 채집 도감 모두 동일한 3단계 상태 모델을 공유한다.

| 상태 | 영문 키 | 조건 | 시각적 표현 |
|------|---------|------|-----------|
| 잠금 (Locked) | `Locked` | 해당 아이템을 한 번도 획득한 적 없음 | 실루엣(검은색) + "???" 이름 + 힌트 텍스트만 표시 |
| 발견 (Discovered) | `Discovered` | 1회 이상 획득 | 컬러 이미지 + 실제 이름 + 상세 정보 공개 |
| 기록 갱신 (NewRecord) | `NewRecord` | 어종 전용: 최대 크기 기록 갱신 시 | "NEW!" 뱃지 일시 표시 (3초) |

**차이점**: 채집물은 크기 시스템이 없으므로 `NewRecord` 상태가 적용되지 않는다. 채집 도감 항목은 `Locked` / `Discovered` 2단계만 사용한다.

### 3.2 도감 항목 공통 필드

모든 도감 항목이 공유하는 필드 구조:

| 필드 | 타입 | 어종 | 채집물 | 설명 |
|------|------|:----:|:-----:|------|
| `itemId` | string | O | O | 고유 식별자 (fish_xxx / gather_xxx) |
| `displayName` | string | O | O | 표시 이름 |
| `rarity` | enum | O | O | Common/Uncommon/Rare/Legendary |
| `hintLocked` | string | O | O | 미발견 힌트 텍스트 |
| `descriptionUnlocked` | string | O | O | 발견 후 설명 텍스트 |
| `firstDiscoverGold` | int | O | O | 초회 발견 골드 보상 |
| `firstDiscoverXP` | int | O | O | 초회 발견 XP 보상 |
| `season` | enum[] | O | O | 등장 계절 |
| `sizeMin` / `sizeMax` | float | O | X | 크기 범위 (어종 전용) |
| `maxRecordSize` | float | O | X | 플레이어 최대 크기 기록 (어종 전용, 런타임) |
| `catchCount` / `gatherCount` | int | O | O | 누적 획득 횟수 (런타임) |
| `pointType` | string | X | O | 채집 포인트 유형 (채집물 전용) |
| `itemCategory` | string | X | O | 아이템 유형: 꽃/식물/버섯/열매/수생 식물/광물 (채집물 전용) |

### 3.3 초회 발견 보상 기준

어종 도감과 채집 도감 모두 동일한 희귀도별 초회 보상 기준을 적용한다. 이는 두 시스템 간 보상 일관성을 보장한다.

| 희귀도 | 초회 골드 | 초회 XP | 적용 |
|--------|----------|---------|------|
| Common | 5G | 2 XP | 어종 5종 + 채집 8종 = 13종 |
| Uncommon | 15G | 5 XP | 어종 4종 + 채집 9종 = 13종 |
| Rare | 50G | 15 XP | 어종 4종 + 채집 6종 = 10종 |
| Legendary | 200G | 50 XP | 어종 2종 + 채집 4종 = 6종 |

**설계 의도**: 어종 도감(-> see `docs/content/fish-catalog.md` 섹션 3.1)의 초회 보상 기준을 그대로 채집 도감에 적용한다. 두 시스템의 희귀도 등급이 동일한 의미(Common = 흔함, Legendary = 극히 희소)를 가지므로, 보상 규모를 일치시켜 "같은 희귀도 = 같은 가치"라는 직관을 형성한다.

## 4. 어종 도감 (Fish Catalog) — fish-catalog.md와 연동

어종 도감의 콘텐츠 데이터(15종 어종 항목, 크기 시스템, 힌트 텍스트, 초회 보상, 마일스톤 보상)는 `docs/content/fish-catalog.md`를 canonical로 유지한다.

본 섹션은 통합 도감 프레임워크 내에서의 어종 도감 탭 연동 방식만 기술한다.

### 4.1 어종 도감 탭 요약

| 항목 | 값 | 참조 |
|------|-----|------|
| 탭 이름 | "어종" | 통합 도감 내 첫 번째 탭 |
| 총 항목 수 | 15종 | (-> see `docs/content/fish-catalog.md` 섹션 3.1) |
| 고유 기능 | 크기 시스템 (sizeMin~sizeMax, 3등급, 기록 갱신) | (-> see `docs/content/fish-catalog.md` 섹션 2) |
| 초회 보상 총합 | 625G + 162 XP | (-> see `docs/content/fish-catalog.md` 섹션 3.1) |
| 마일스톤 | 5/10/15종, 보상 총합 900G + 260 XP | (-> see `docs/content/fish-catalog.md` 섹션 4.1) |
| 도감 장식 보상 | 배경 2종, 프레임 1종 | (-> see `docs/content/fish-catalog.md` 섹션 4.3) |

### 4.2 어종 도감 카드 특수 요소

어종 도감 카드에만 표시되는 정보:
- **크기 범위**: sizeMin ~ sizeMax (cm)
- **내 최대 기록**: 플레이어의 해당 어종 최대 크기 + 등급 (Small/Medium/Large)
- **Giant 태그**: Giant 변이 가능 어종에 [Giant] 마크 표시

채집 도감 카드에는 이러한 크기 관련 요소가 없으며, 대신 채집물 고유 요소(섹션 5.1)가 표시된다.

## 5. 채집 도감 (Gathering Catalog)

### 5.1 도감 항목 구조

채집물 도감 카드에 표시되는 정보 구성:

```
+------------------------------------------+
| [희귀도 컬러 바] (Common=회, Uncommon=녹, |
|                  Rare=청, Legendary=금)   |
+------------------------------------------+
|                                          |
|      [채집물 이미지 / 실루엣]              |
|                                          |
+------------------------------------------+
| 이름: 송이버섯                 희귀도: ★★★☆ |
| 유형: 버섯                               |
| 채집 위치: 숲 바닥                         |
| 채집 횟수: 12회                           |
+------------------------------------------+
| [발견 후 설명 텍스트 영역]                  |
+------------------------------------------+
| 등장 조건: 가을 | 숲 바닥 | Rain 다음 날   |
+------------------------------------------+
```

**어종 도감 카드와의 차이점**:

| 요소 | 어종 도감 | 채집 도감 |
|------|----------|----------|
| 크기 범위 | O (sizeMin~sizeMax) | X (크기 시스템 없음) |
| 최대 크기 기록 | O | X |
| 크기 등급 표시 | O (Small/Medium/Large) | X |
| 아이템 유형 | X (모두 "물고기") | O (꽃/식물/버섯/열매/수생 식물/광물) |
| 채집 위치 | X (모두 "연못") | O (숲 바닥/덤불/동굴 입구/연못가/초원) |
| 날씨 특수 조건 | O (비, 눈 등) | O (Rain 다음 날 버섯, Snow 등) |

**설계 의도**: 어종 도감은 "같은 물고기를 또 잡아 크기 기록을 갱신"하는 반복 동기를 크기 시스템으로 제공한다. 채집물은 크기가 무의미하므로, 대신 "어디서 나오는가"(위치)와 "어떤 종류인가"(유형)를 강조하여 탐색과 분류의 즐거움을 부각한다.

**초회 보상 총합 산출**:
- Common 8종: 8 x (5G + 2 XP) = 40G + 16 XP
- Uncommon 9종: 9 x (15G + 5 XP) = 135G + 45 XP
- Rare 6종: 6 x (50G + 15 XP) = 300G + 90 XP
- Legendary 4종: 4 x (200G + 50 XP) = 800G + 200 XP
- **합계: 1,275G + 351 XP**

**어종 도감 초회 보상(625G + 162 XP)과의 비교**: 채집 도감은 27종으로 어종(15종)보다 항목 수가 1.8배 많고, Rare/Legendary 비중도 높다(Legendary 4종 vs 2종). 따라서 초회 보상 총합이 약 2배인 것은 항목 수와 희귀도 분포에 비례한다.

#### 5.2.2 미발견 힌트 텍스트 및 발견 후 설명 텍스트

> 채집 아이템별 힌트 텍스트(`hintLocked`)와 발견 후 설명 텍스트(`descriptionUnlocked`)의 실제 텍스트 값은 `docs/content/gathering-items.md`(CON-012)가 canonical이다. 각 아이템의 "힌트 텍스트" 행을 참조한다.
>
> (-> see `docs/content/gathering-items.md` 섹션 3~7 각 아이템별 "힌트 텍스트" 행)

**힌트 텍스트 설계 원칙** (텍스트 작성 가이드라인):
- 미발견 힌트는 계절, 위치, 특수 조건을 **간접적으로 암시**한다. 구체적인 수치나 조건명은 노출하지 않는다.
- 발견 후 설명은 구체적인 등장 조건이 공개되며, 채집물의 활용처도 포함한다.

### 5.3 완성 마일스톤 보상

채집 도감 등록 종 수에 따라 단계별 마일스톤 보상을 지급한다.

#### 5.3.1 마일스톤 설계 원칙

- 어종 도감의 마일스톤 (5/10/15종, -> see `docs/content/fish-catalog.md` 섹션 4.1)과 비례적으로 설계
- 채집물 27종은 어종 15종의 1.8배이므로, 마일스톤 간격을 10/20/27종으로 설정
- 보상 규모는 어종 마일스톤과 유사하되, 항목 수 차이를 반영하여 약간 상향

#### 5.3.2 마일스톤 테이블

| 마일스톤 | 필요 종 수 | 골드 | XP | 특수 보상 | 비고 |
|----------|:--------:|:----:|:---:|----------|------|
| 초보 채집가 | 10종 | 150G | 40 XP | 채집 낫 수리 키트 x1 | Common/Uncommon 위주로 1~2계절 내 자연스럽게 달성 |
| 숙련 채집가 | 20종 | 400G | 100 XP | 도감 배경 "사계절의 숲" + 고급 비료 x5 | 3~4계절 이상 플레이 필요. 겨울 한정 아이템 포함 |
| 채집 도감 마스터 | 27종 (전종) | 600G | 180 XP | 도감 배경 "전설의 자연" + 도감 프레임 "숲의 정령" | Legendary 4종 전부 필요. 장기 목표 |

**설계 의도**:
- **10종 마일스톤**: Zone D 해금(여름 중반) 후 봄~가을의 Common/Uncommon 채집물과 광물을 모으면 1~2계절 내 달성. 어종 5종 마일스톤(Common 5종으로 달성 가능)과 유사한 즉각 보상 단계.
- **20종 마일스톤**: 3~4계절 이상 플레이하여 겨울 한정 아이템(겨울 나무껍질, 눈꽃 이끼, 동충하초)을 포함해야 한다. 어종 10종 마일스톤(2~3계절)과 유사한 중기 목표.
- **27종(전종) 마일스톤**: Legendary 4종(산삼, 황금 연꽃, 천년 영지, 자수정)의 극히 제한적 등장 조건 충족 필요. 특히 황금 연꽃(Clear + Dawn + Lv.7+)과 천년 영지(Cloudy + Lv.7+)는 특정 날씨를 기다려야 한다. 어종 15종 마일스톤과 마찬가지로 게임 후반 장기 목표.

**마일스톤 보상 총합**: 골드 1,150G + XP 320.

**채집 도감 시스템 자체 보상 총합**: 초회 보상(1,275G + 351 XP) + 마일스톤(1,150G + 320 XP) = **2,425G + 671 XP**

**어종 도감 자체 보상(1,525G + 422 XP)과의 비교**: 채집 도감 보상이 약 1.6배. 항목 수(27 vs 15 = 1.8배)와 대략 비례하며, 마일스톤 단계가 더 멀리 떨어져 있어 시간 투자 대비 적절한 보상.

```
+=====================================================+
|  [수집 도감]                    전체 진행률: 28/42종  |
+=====================================================+
|  [어종]  |  [채집]  |  (향후 확장 예약)               |

|                                                     |

|                                                     |
+=====================================================+
```

- **탭 바**: [어종] [채집] 두 탭. 향후 카테고리 추가 시 탭을 덧붙이는 구조

### 6.5 초회 채집 등록 팝업

채집물을 처음 채집하면 도감 등록 팝업이 표시된다. 어종 초회 등록 팝업(-> see `docs/content/fish-catalog.md` 섹션 5.6)과 동일한 구조를 채집물에 맞게 변형한다.

```
+------------------------------------------+
|      * 새로운 채집물 발견! *                |
+------------------------------------------+
|                                          |
|      [채집물 컬러 이미지]                  |
|                                          |
|        송이버섯 (Rare)                    |
|        유형: 버섯 | 위치: 숲 바닥          |
|                                          |
|  초회 등록 보상: 50G + 15 XP              |

|  [도감 보기]          [닫기]              |
+------------------------------------------+
```

- 팝업은 게임 시간 정지 상태에서 표시
- "도감 보기" 클릭 시 수집 도감 > 채집 탭으로 이동, 해당 아이템 카드에 포커스
- 마일스톤 달성 시 추가 팝업 표시 (섹션 5.3.2의 보상 안내)

## Risks

1. **[RISK]** 채집 도감 초회 보상 총합(1,275G)이 어종 도감(625G)의 2배에 달한다. 이는 항목 수(27 vs 15)에 비례하지만, 채집이 "보조 활동"임을 감안하면 채집 도감에서 얻는 총 골드(도감+마일스톤+업적 = 3,425G)가 과다할 수 있다. 다만 전종 완성은 4~5계절 이상 소요되므로 시간당 보상 밀도는 낮다.

2. **[RISK]** 통합 도감 UI 도입으로 기존 fish-catalog.md의 UI 구조(섹션 5)와 fishing-architecture.md의 FishCatalogManager(ARC-030)에 변경이 필요하다. 아키텍처 리팩터링 범위가 예상보다 클 수 있다.

3. **[RISK]** 도감 장식 보상(배경, 프레임)이 어종 도감(3종)과 채집 도감(3종) 합쳐 6종이 되며, 향후 카테고리 추가 시 더 늘어난다. 도감 장식 관리 UI가 복잡해질 수 있다. 장식 적용 방식(어종 탭/채집 탭 독립 적용 vs 전체 도감 공통 적용)은 아키텍처에서 결정 필요.
