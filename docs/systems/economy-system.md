# 경제 시스템 (Economy System) 상세 설계

> 작성: Claude Code (Opus) | 2026-04-06  
> 문서 ID: DES-004

---

## Context

이 문서는 SeedMind의 경제 시스템을 상세히 기술한다. 마스터 디자인 문서(`docs/design.md`) 섹션 4.4에 정의된 경제 시스템 개요를 기반으로, 화폐 구조, 가격 시스템, 상점 시스템, 수입/지출 밸런스, 거래 메커니즘, 동적 가격 변동을 포함한 모든 경제 관련 세부 사항을 다룬다.

**설계 목표**: 플레이어가 "무엇을 심고, 언제 팔고, 어디에 투자할지"를 끊임없이 고민하게 만든다. 돈이 항상 부족하되, 올바른 선택을 하면 확실한 성장을 느낄 수 있어야 한다. 어떤 하나의 작물이나 전략이 압도적으로 유리하지 않도록, 모든 수익원에 트레이드오프를 부여한다.

## 1. 골드(화폐) 시스템 (Gold System)

### 1.1 화폐 단위

| 항목 | 내용 |
|------|------|
| 화폐 이름 | 골드 (Gold) |
| 단위 표기 | `G` (숫자 뒤에 붙임, 예: `500G`) |
| 최소 단위 | 1G (소수점 없음, 모든 거래는 정수 단위) |
| 최대 소지 가능 | 999,999G |
| UI 표시 | 3자리마다 콤마 구분 (예: `12,500G`) |

### 1.2 초기 지급

| 파라미터 | 값 | 근거 |
|----------|-----|------|
| 게임 시작 시 소지 골드 | 500G | 감자 씨앗(15G) x10 + 물뿌리개 리필 여유 + 기본 비료 시도 가능 |

**500G의 의미**: 플레이어가 첫 계절에 기본 작물 씨앗을 10~15개 구매하고, 기본 비료를 2~3개 시험해볼 수 있는 금액. 모든 것을 다 할 수 없어 선택이 필요하다.

### 1.3 골드 획득 경로

| 경로 | 설명 | 게임 단계 |
|------|------|-----------|
| 작물 직접 판매 | 출하함 또는 상점에서 작물 판매 | 초반~종반 (핵심) |
| 가공품 판매 | 가공소에서 만든 잼/주스 등 판매 | 중반~종반 (-> see 섹션 2.5) |
| 축제 보상 | 축제 참여/미니게임 보상 (-> see `docs/systems/time-season.md` 섹션 4.2) | 전 구간 |
| 채집물 판매 | 야생 채집물(꽃, 열매 등) 판매 | 초반~중반 |
| 거대 작물 수확 | 거대 작물 대량 수확물 판매 (-> see `docs/systems/crop-growth.md` 섹션 5.1) | 후반 |

[OPEN] 채집 시스템의 상세 설계가 미정. 채집물 종류와 판매가는 콘텐츠 문서(CON 계열)에서 확정 필요.

### 1.4 골드 소실 경로

| 경로 | 설명 |
|------|------|
| 기절 페널티 | 24:00 기절 시 소지 골드의 5%, 최대 500G (-> see `docs/systems/time-season.md` 섹션 1.5) |

**설계 의도**: 기절 페널티 외에 골드가 사라지는 메커니즘은 없다. 구매한 물품은 환불 불가이므로, 잘못된 구매가 사실상의 손실이다.

## 2. 가격 시스템 (Pricing System)

### 2.1 기본 가격 구조

모든 아이템에는 **기본 판매가**와 **기본 구매가**가 존재한다.

| 개념 | 정의 |
|------|------|
| 기본 판매가 (baseSellPrice) | 아이템을 상점/출하함에 팔 때 받는 기본 금액 |
| 기본 구매가 (baseBuyPrice) | 아이템을 상점에서 살 때 내는 기본 금액 |
| 마진 비율 (marginRatio) | 구매가 대비 판매가의 비율. 기본 `0.5` (구매가의 50%가 판매가) |

```
baseSellPrice = baseBuyPrice * marginRatio
```

**작물 판매가는 canonical 값을 따른다** (-> see `docs/design.md` 섹션 4.2). 씨앗 구매가는 아래 섹션 2.2에서 정의.

### 2.2 씨앗 구매가

씨앗 구매가는 작물 판매가에 기반하여 산출한다. 플레이어가 투자 대비 수익을 계산할 수 있도록 일관된 비율을 적용한다.

| 파라미터 | 값 |
|----------|-----|
| 씨앗가 비율 (seedPriceRatio) | 작물 기본 판매가의 50% |

```
seedBuyPrice = round(baseSellPrice * seedPriceRatio)
```

| 작물 | 기본 판매가 (-> see `docs/design.md`) | 씨앗 구매가 | 순이익 (일반 품질, 비료 없음) |
|------|------|------|------|
| 감자 | 30G | 15G | 15G |
| 당근 | 35G | 18G | 17G |
| 토마토 | 60G | 30G | 30G |
| 옥수수 | 100G | 50G | 50G |
| 딸기 | 80G | 40G | 40G (+ 재수확 수익) |
| 호박 | 200G | 100G | 100G |
| 해바라기 | 150G | 75G | 75G |
| 수박 | 350G | 175G | 175G |

**일일 수익 효율** (순이익 / 성장일수, 일반 품질 기준):

| 작물 | 순이익 | 성장일수 (-> see `docs/design.md`) | 일일 효율 | 비고 |
|------|--------|-----------|-----------|------|
| 감자 | 15G | 3일 | 5.0G/일 | 초반 안정 수익 |
| 당근 | 17G | 3일 | 5.7G/일 | 감자보다 약간 효율적 |
| 토마토 | 30G | 5일 | 6.0G/일 | 레벨 2 해금, 중간 효율 |
| 옥수수 | 50G | 7일 | 7.1G/일 | 장기 투자형 |
| 딸기 | 40G + (재수확당 80G) | 5일 + 3일/회 | ~13.3G/일* | 재수확 포함, 최고 효율 |
| 호박 | 100G | 10일 | 10.0G/일 | 고리스크 고리턴 |
| 해바라기 | 75G | 8일 | 9.4G/일 | 씨앗 드롭 보너스 별도 |
| 수박 | 175G | 12일 | 14.6G/일 | 최고 단가, 계절 리스크 |

*딸기: 봄 28일 기준, 첫 수확 5일 + 재수확 약 7회(21일/3일) = 총 8회 수확. (40G + 80G*7) / 28일 = 21.4G/일. 단, 씨앗 비용 포함 시 (80G*8 - 40G) / 28일 = 약 21.4G/일.

[RISK] 딸기의 재수확 효율이 다른 작물 대비 과도하게 높을 수 있다. 재수확 간격 조정(3일 -> 4일) 또는 재수확 횟수 제한 검토 필요. 밸런스 시트(BAL-001)에서 정밀 분석 필요.

### 2.5 가공품 가격 산정

가공소(3,000G, 레벨 7 해금)에서 작물을 가공품으로 변환할 수 있다 (-> see `docs/design.md` 섹션 4.4).

#### 가공 공식

```
가공품_판매가 = floor(원재료_기본판매가 * 가공_배수 + 가공_고정보너스)
```

| 가공 유형 | 영문 키 | 가공 배수 | 고정 보너스 | 가공 시간 | 비고 |
|----------|---------|----------|-----------|-----------|------|
| 잼 (Jam) | `Jam` | x2.0 | +50G | 4시간 (게임 내) | 모든 과일/채소 가공 가능 |
| 주스 (Juice) | `Juice` | x2.5 | +30G | 6시간 (게임 내) | 과일류만 가공 가능 |
| 절임 (Pickle) | `Pickle` | x2.0 | +30G | 4시간 (게임 내) | 채소류만 가공 가능 |

**작물 분류** (가공 유형 결정 기준):

| 작물 | 분류 | 가공 가능 유형 |
|------|------|--------------|
| 감자 | 채소 | 잼, 절임 |
| 당근 | 채소 | 잼, 절임 |
| 토마토 | 과일/채소 겸용 | 잼, 주스, 절임 |
| 옥수수 | 채소 | 잼, 절임 |
| 딸기 | 과일 | 잼, 주스 |
| 호박 | 채소 | 잼, 절임 |
| 해바라기 | 특수 (씨앗) | 가공 불가 |
| 수박 | 과일 | 잼, 주스 |

**가공품 가격 예시** (일반 품질 원재료 기준):

| 가공품 | 원재료 판매가 | 가공 유형 | 가공품 판매가 | 순이익 증가분 |
|--------|-------------|----------|-------------|-------------|
| 감자 잼 | 30G | 잼 (x2.0 + 50G) | 110G | +80G |
| 딸기 주스 | 80G | 주스 (x2.5 + 30G) | 230G | +150G |
| 수박 주스 | 350G | 주스 (x2.5 + 30G) | 905G | +555G |
| 호박 절임 | 200G | 절임 (x2.0 + 30G) | 430G | +230G |
| 토마토 주스 | 60G | 주스 (x2.5 + 30G) | 180G | +120G |

**가공품과 품질**: 가공품의 판매가는 **원재료의 품질에 영향받지 않는다**. 가공 과정에서 품질 차이가 사라진다. 따라서 일반(Normal) 품질 작물을 가공하는 것이 가장 효율적이고, 고품질 작물은 직접 판매가 유리하다.

**설계 의도**: 가공 vs 직판의 트레이드오프를 만든다.
- 직판: 즉시 현금화, 고품질일수록 유리
- 가공: 시간 투자 필요, 일반 품질도 높은 수익, 가공소 슬롯 제한

#### 가공소 슬롯

| 파라미터 | 값 |
|----------|-----|
| 초기 슬롯 수 | 1 |
| 최대 슬롯 수 | 3 |
| 슬롯 확장 비용 | 2번째: 1,500G, 3번째: 3,000G |

각 슬롯은 독립적으로 가공을 진행한다. 동시에 최대 3개 가공품 제작 가능.

[OPEN] 가공소 슬롯 확장이 레벨 해금 조건을 추가로 가져야 하는지 검토 필요. 현재는 골드만으로 확장 가능.

### 2.6 동적 가격 변동 (Dynamic Pricing)

게임 내 수급과 환경에 따라 작물 판매가가 변동한다.

#### 2.6.1 가격 변동 요인

| 요인 | 영문 키 | 영향 | 적용 대상 |
|------|---------|------|-----------|
| 계절 보너스 | `seasonalBonus` | 가을 수확물 +10% | 가을에 수확한 모든 작물 (-> see `docs/systems/time-season.md` 섹션 2.2) |
| 수급 변동 | `supplyDemand` | 같은 작물 대량 판매 시 가격 하락 | 출하함 판매 (섹션 5.1 참조) |
| 날씨 영향 | `weatherEffect` | 폭풍/폭설 다음 날 가격 상승 | 전체 작물 |
| 축제 보너스 | `festivalBonus` | 수확 축제 주간 가격 상승 | 전체 작물 |

#### 2.6.2 수급 변동 메커니즘

플레이어가 같은 작물을 대량으로 출하하면, 시장이 포화되어 가격이 하락한다.

```
수급_배수 = max(minSupplyMultiplier, 1.0 - (주간_출하량 / 포화_기준량) * 하락_계수)
```

| 파라미터 | 영문 키 | 기본값 | 설명 |
|----------|---------|--------|------|
| 포화 기준량 | `saturationThreshold` | 20개 | 이 수량까지는 가격 변동 없음 |
| 하락 계수 | `supplyDropRate` | 0.02 | 포화 기준 초과 1개당 2% 하락 |
| 최저 배수 | `minSupplyMultiplier` | 0.7 | 아무리 많이 팔아도 70% 이하로 떨어지지 않음 |
| 회복 속도 | `demandRecoveryRate` | 주간 5개씩 | 매주 초 포화량 5개 감소 (수요 회복) |

**예시**: 감자 30개를 한 주에 출하
- 초과량: 30 - 20 = 10개
- 수급 배수: 1.0 - (10 * 0.02) = 0.8 (20% 할인)
- 실제 판매가: 30G * 0.8 = 24G (10개째부터 적용)

**설계 의도**: 단일 작물 대량 재배(모노컬처) 전략을 약화시키고, 작물 다양화에 인센티브를 부여한다.

#### 2.6.2.1 수급 카테고리 시스템 (Supply Category System) [FIX-044]

모든 판매 가능 아이템은 `SupplyCategory` enum에 따라 수급 변동 민감도가 달라진다. 카테고리별로 독립적인 수급 풀(주간 출하량 카운터)을 운영한다.

**enum SupplyCategory**:

| 값 | 영문 키 | 대상 아이템 | 설계 의도 |
|---|---------|-----------|----------|
| 0 | `Crop` | 작물 (감자, 딸기, 밀 등) | 기본 수급 변동 — 모노컬처 억제 |
| 1 | `AnimalProduct` | 동물 생산물 (달걀, 우유, 염소젖, 양모 등) | 완화된 수급 변동 — 안정적 수입원 역할 보존 |
| 2 | `Fish` | 생선 (붕어, 잉어, 메기 등) | 완화된 수급 변동 — 보조 수입원 역할 보존 |
| 3 | `Processed` | 가공품 (잼, 주스, 치즈, 구운 생선 등) | 수급 변동 면제 — 가공 투자 보상 보호 |

**카테고리별 수급 파라미터**:

| 파라미터 | Crop | AnimalProduct | Fish | Processed |
|----------|------|---------------|------|-----------|
| `saturationThreshold` | 20 | 35 | 30 | N/A (면제) |
| `supplyDropRate` | 0.02 | 0.008 | 0.01 | 0 |
| `minSupplyMultiplier` | 0.7 | 0.85 | 0.8 | 1.0 |
| `demandRecoveryRate` | 5/주 | 7/주 | 6/주 | N/A |

**수급 배수 공식 (카테고리 적용)**:

```
수급_배수(itemId) = max(
    categoryParams[item.supplyCategory].minSupplyMultiplier,
    1.0 - max(0, 주간_출하량[itemId] - categoryParams[item.supplyCategory].saturationThreshold)
        * categoryParams[item.supplyCategory].supplyDropRate
)
```

**설계 근거 (옵션 B 선택 — 별도 카테고리, 완화 적용)**:

1. **완전 면제(옵션 C) 기각 사유**: 동물 생산물이 수급 변동을 완전히 면하면, 닭 8마리 + 소 4마리의 대량 사육이 무위험 고수익 전략이 된다. 작물 다양화 인센티브와 상충하며, 후반부 경제에서 목축이 지배 전략으로 수렴한다.

2. **동일 풀(옵션 A) 기각 사유**: 동물 생산물은 작물과 달리 (a) 초기 투자(시설+구매)가 크고, (b) 일일 사료비라는 고정 지출이 있으며, (c) 생산량을 즉시 늘릴 수 없다(동물 구매 후 성장 대기). 작물과 동일한 supplyDropRate 0.02를 적용하면 Break-even 일수가 과도하게 늘어나 투자 정당성이 사라진다.

3. **별도 카테고리(옵션 B) 채택 사유**: 동물 생산물에 작물 대비 60% 낮은 하락 계수(0.008 vs 0.02)와 75% 높은 포화 기준(35 vs 20)을 적용하면, 적정 규모(닭 4~6마리, 소 2~3마리) 사육 시에는 수급 영향이 거의 없고, 극단적 대량 사육(닭 8+소 4+염소 4 등)에서만 수익이 소폭 감소하는 구조가 된다.

**수급 영향 시뮬레이션 (AnimalProduct 카테고리)**:

| 주간 달걀 출하량 | 초과량 | 수급 배수 | 실질 판매가 (기본 35G) |
|----------------|--------|----------|---------------------|
| 28개 (닭 4마리) | 0 | x1.0 | 35G |
| 42개 (닭 6마리) | 7 | x0.944 | 33G |
| 56개 (닭 8마리) | 21 | x0.832 | 29G |

닭 4~6마리까지는 실질적 영향이 미미(0~2G 감소)하며, 8마리 풀 가동에서도 최대 17% 하락으로 수입의 안정성이 유지된다. BAL-008의 ROI 계산(수급 미적용)과의 괴리가 작아 고정 수입 모델을 크게 훼손하지 않는다.

**낚시(Fish) 카테고리**: 낚시는 활동 시간 대비 획득량이 자연스럽게 제한되므로(시간당 4~8마리), 포화 기준 30개에 도달하려면 주 4~7시간 낚시에 전념해야 한다. 대부분의 플레이어가 이 한도를 넘지 않으므로 사실상 면제에 가깝되, 낚시만 반복하는 극단적 패턴에는 소폭 수급 하락이 발생한다.

**가공품(Processed) 면제**: 가공품은 원재료 구매/재배 → 가공 대기시간 → 판매라는 다단계 투자를 거친다. 원재료 단계에서 이미 수급 변동(작물/동물 생산물 직판 기회비용)이 반영되므로, 가공품에 중복 적용하면 가공소 투자 인센티브가 과도하게 약화된다.

#### 2.6.3 날씨 영향

악천후 다음 날에는 작물 가격이 소폭 상승한다 (희소성 반영).

| 전날 날씨 | 가격 배수 |
|----------|----------|
| Clear, Cloudy, Rain | x1.0 (변동 없음) |
| HeavyRain | x1.02 (+2%) |
| Storm | x1.05 (+5%) |
| Snow | x1.03 (+3%) |
| Blizzard | x1.08 (+8%) |

**지속**: 1일만 적용. 다음 날 날씨가 보통이면 배수 초기화.

#### 2.6.4 축제 보너스

| 기간 | 가격 배수 | 비고 |
|------|----------|------|
| 가을 수확 축제 주간 (Day 18~24) | x1.15 (+15%) | 수확 축제(Day 21) 전후 |

다른 축제는 가격에 영향을 주지 않는다.

#### 2.6.5 온실 판매가 보정 (Greenhouse Price Modifier) [BAL-010]

온실에서 재배한 비계절 작물의 판매가에 추가 보정을 적용한다. 온실의 성장 속도/품질 페널티(-> see `docs/content/facilities.md` 섹션 4.3)만으로는 비계절 작물(특히 고수익 작물)의 경쟁력을 충분히 억제하지 못하는 문제를 해결하기 위한 메커니즘이다.

**적용 조건**: 아래 두 조건을 **모두** 만족하는 경우에만 적용

1. 작물이 온실 타일에서 수확되었음 (`isGreenhouse == true`)
2. 해당 작물의 재배 가능 계절(`allowedSeasons`)이 현재 계절을 포함하지 않음 (`isInSeason == false`)

**[RESOLVED-BAL-010] 확정된 보정 방식** (2026-04-07 확정, -> see `docs/balance/crop-economy.md` 섹션 4.3.10):

| 보정 유형 | 적용 대상 | 배수 | 파라미터 |
|----------|----------|------|---------|
| 비계절 온실 판매가 페널티 | 비주 계절 온실 작물 (전 계절 적용) | **x0.8** | `greenhouseOffSeasonPenalty` |
| 겨울 전용 작물 온실 시너지 보너스 | 겨울 전용 작물 3종 (온실 재배 시) | **x1.2** | `greenhouseSynergyBonus` (CropData) |

두 보정은 동시에 적용되지 않는다: 겨울 전용 작물에는 페널티 대신 시너지 보너스가 적용되며, 해당 계절 온실 작물에는 어떤 보정도 적용되지 않는다.

**적용 제외 대상**:

- 해당 계절 작물을 온실에서 재배 (`isInSeason == true`): 보정 없음
- 겨울 전용 작물을 온실에서 재배 (`allowedSeasons == SeasonFlag.Winter`): 페널티 면제, 시너지 보너스(x1.2) 적용

#### 2.6.6 최종 판매가 공식

```
최종_판매가 = floor(기본_판매가 * 품질_배수 * 계절_배수 * 수급_배수 * 날씨_배수 * 축제_배수 * 온실_배수)
```

- 품질 배수: (-> see `docs/systems/crop-growth.md` 섹션 4.3, canonical)
  - Normal: x1.0, Silver: x1.25, Gold: x1.5, Iridium: x2.0
- 계절 배수: 가을 x1.1, 그 외 x1.0
- 수급 배수: x0.7 ~ x1.0 (섹션 2.6.2)
- 날씨 배수: x1.0 ~ x1.08 (섹션 2.6.3)
- 축제 배수: x1.0 또는 x1.15 (섹션 2.6.4)
- 온실 배수: x1.0 (야외 또는 해당 계절 온실), x0.8 (비주 계절 온실 작물), x1.2 (겨울 전용 작물 온실) (-> see 섹션 2.6.5, [RESOLVED-BAL-010])

**가격 변동 범위 제한**:

| 파라미터 | 영문 키 | 값 | 설명 |
|----------|---------|-----|------|
| 최저 가격 배율 | `minPriceMultiplier` | 0.6 | 모든 배수 곱한 결과가 x0.6 이하가 되지 않음 |
| 최고 가격 배율 | `maxPriceMultiplier` | 1.5 | 모든 배수 곱한 결과가 x1.5 이상이 되지 않음 |

```
종합_배수 = clamp(품질_배수 * 계절_배수 * 수급_배수 * 날씨_배수 * 축제_배수 * 온실_배수, minPriceMultiplier, maxPriceMultiplier)
```

**참고**: 품질 배수(최대 x2.0)는 clamp 범위를 초과할 수 있다. clamp는 품질 배수를 **제외한** 외부 변동 요인에만 적용한다.

```
외부_배수 = clamp(계절_배수 * 수급_배수 * 날씨_배수 * 축제_배수 * 온실_배수, minPriceMultiplier, maxPriceMultiplier)
최종_판매가 = floor(기본_판매가 * 품질_배수 * 외부_배수)
```

### 4.1 판매 채널

#### 출하함 (Shipping Bin)

| 접근성 | 24시간, 농장 내 | 영업시간 한정, 마을 이동 필요 |
| 정산 속도 | 다음 날 (느림) | 즉시 (빠름) |
| 가격 | 동일 | 동일 |
| 취소 | 당일 가능 | 불가 |

**설계 의도**: 두 채널의 가격을 동일하게 유지하여 불필요한 이동 페널티를 방지한다. 출하함은 편의성, 상점 직판은 즉시 현금화의 장점. 주로 출하함을 사용하되, 급하게 골드가 필요할 때 상점을 방문하는 패턴을 유도.

### 4.2 품질에 따른 가격 보정

(-> see `docs/systems/crop-growth.md` 섹션 4.3, canonical)

| 품질 | 배수 | 예시 (감자 30G 기준) |
|------|------|---------------------|
| Normal | x1.0 | 30G |
| Silver | x1.25 | 37G |
| Gold | x1.5 | 45G |
| Iridium | x2.0 | 60G |

### 4.3 대량 거래

#### 대량 판매 (출하함)

출하함에 한 번에 많은 양을 넣을 수 있으며, 수량 제한은 없다. 단, 수급 변동 메커니즘(섹션 2.6.2)이 대량 판매의 효율을 자연스럽게 제한한다.

**대량 판매 보너스/패널티 없음**: 대량 판매 자체에 별도 보너스나 패널티는 없다. 수급 변동이 실질적인 대량 판매 패널티 역할을 한다.

#### 대량 구매 (상점)

| 수량 | 할인 |
|------|------|
| 1~9개 | 없음 (정가) |
| 10~24개 | -5% |
| 25개 이상 | -10% |

```
대량_할인가 = floor(baseBuyPrice * (1.0 - bulkDiscount) * 수량)
```

**설계 의도**: 대량 구매 할인으로 계획적 투자를 장려한다. 씨앗을 한꺼번에 사면 약간 저렴하므로, 자금 여유가 생기면 미리 대량 구매하는 전략이 유효하다.

### 4.4 환불 정책

| 규칙 | 내용 |
|------|------|
| 아이템 환불 | **불가**. 구매한 아이템은 돌려줄 수 없음 |
| 시설 해체 | 시설 해체 시 **건설 비용의 25%** 자재 회수 (골드 환불 아님, 목재/석재 형태) |
| 도구 업그레이드 | **되돌릴 수 없음** |

**설계 의도**: 환불 불가로 구매 결정에 무게감을 부여한다. "이 씨앗을 살까 말까"의 고민이 의미를 가짐.

### 5.3 인플레이션/디플레이션 방지 메커니즘

| 메커니즘 | 유형 | 설명 |
|----------|------|------|
| 수급 변동 | 골드 흡수 | 대량 판매 시 가격 하락 (섹션 2.6.2) |
| 시설 비용 증가 | 골드 흡수 | 후반 시설은 점진적으로 비쌈 |
| 도구 업그레이드 비용 | 골드 흡수 | 단계별 비용 증가 (-> see `docs/systems/tool-upgrade.md` 섹션 2.1) |
| 농장 확장 비용 | 골드 흡수 | Zone B~G 6단계 총 16,000G (→ see `docs/systems/farm-expansion.md` 섹션 2.1) |
| 가공소 슬롯 제한 | 수입 제한 | 동시 가공 3개 제한으로 수입 상한 존재 |
| 기절 페널티 | 골드 소멸 | 최대 500G 소멸 (소량이지만 존재) |
| 환불 불가 | 실질 손실 | 잘못된 구매 = 영구 손실 |

**핵심 원칙**: 플레이어의 수입은 시간에 비례하여 자연스럽게 증가하되, 투자처(시설, 도구, 확장)가 항상 수입보다 빠르게 소비할 수 있도록 설계한다. "돈이 남는" 상태가 오래 지속되지 않아야 한다.

[OPEN] 후반부(레벨 8~10) 플레이어의 골드 과잉 문제를 해결할 추가 골드 싱크가 필요할 수 있다. 장식 아이템, 농장 꾸미기, 고가 수집품 등의 콘텐츠 추가 검토 필요.

## 6. 창고 시스템 (Storage System)

창고(1,000G, 레벨 4 해금)는 작물을 저장하여 가격 변동에 대응할 수 있게 한다 (-> see `docs/design.md` 섹션 4.6).

### 6.1 기본 규칙

| 항목 | 내용 |
|------|------|
| 저장 용량 | 50슬롯 (각 슬롯에 같은 아이템 최대 99개 스택) |
| 보관 기한 | 무제한 (작물이 부패하지 않음) |
| 접근 | 농장 내 창고 건물에서 상호작용(E키) |

**설계 의도**: 창고는 가격이 낮을 때 저장했다가, 축제 보너스 주간이나 날씨 보너스 때 판매하는 전략을 가능하게 한다. 작물 부패가 없는 이유는 부패 시스템이 지나치게 복잡성을 추가하기 때문이다.

[OPEN] 작물 부패 시스템(저장 기한 제한) 도입 여부. 부패가 없으면 저장 후 판매 전략이 지나치게 강력할 수 있다. 부패를 넣으면 복잡성이 증가한다. 현재는 부패 없음으로 확정, 밸런스 테스트 후 재검토.
