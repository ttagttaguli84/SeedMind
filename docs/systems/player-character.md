## 1. 개요

SeedMind의 플레이어 캐릭터는 로우폴리 스타일라이즈드 3D 농장 시뮬레이션에 맞게 설계된다. 기어다니는(leisurely) 플레이 감각을 지향하며, Stardew Valley의 이동감에서 영감을 받은 부드럽고 반응성 높은 조작을 목표로 한다.

| 항목 | 값 |
|------|-----|
| 시점 | 쿼터뷰 3D (Orthographic, X=45°, Y=45°) |
| 이동 방식 | WASD 8방향 탑다운 입력 |
| 이동 속도 | 4 tiles/sec (기본), 6 tiles/sec (달리기, 미래 확장 여지) |
| 인터랙션 반경 | 1.5 tiles (플레이어 중심 기준) |
| 캐릭터 높이 | 타일 1m 대비 0.7m |
| 카메라 추적 | Cinemachine Virtual Camera (SmoothDamp 대비 파라미터 조정 유연성 우수) |

## 2. 이동 시스템

### 2.1 이동 방식

WASD(또는 방향키) 8방향 이동. 화면 기준 탑다운 입력을 쿼터뷰 카메라 방향에 맞게 변환한다.

| 입력 | 화면 방향 | 월드 이동 방향 |
|------|-----------|---------------|
| W | 위 | 북서 (+Z -X) |
| S | 아래 | 남동 (-Z +X) |
| A | 왼쪽 | 남서 (-Z -X) |
| D | 오른쪽 | 북동 (+Z +X) |
| W+D | 위 오른쪽 | 북 (+Z) |
| W+A | 위 왼쪽 | 서 (-X) |
| S+D | 아래 오른쪽 | 동 (+X) |
| S+A | 아래 왼쪽 | 남 (-Z) |

대각선 입력 시 벡터를 normalize하여 이동 속도가 일정하게 유지된다.

### 2.2 이동 속도

| 상태 | 속도 | 설명 |
|------|------|------|
| 기본 이동 | **4 tiles/sec** | 일반 이동. 8x8 그리드 끝에서 끝 2초 소요 |
| 달리기 | 6 tiles/sec | [OPEN] 에너지 소모와 연계 여부 미결정. Phase 2 확장 항목 |
| 인터랙션 중 | 0 tiles/sec | 도구 사용 애니메이션 재생 구간에는 이동 불가 |

**수치 근거**:
- 8x8 그리드를 기본 이동으로 2초에 횡단 — 농장 전체를 탐색하는 데 너무 짧지도 길지도 않은 시간
- Stardew Valley는 타일 크기 대비 약 3~4 tiles/sec 수준. SeedMind는 3D 쿼터뷰이므로 4 tiles/sec로 약간 빠르게 설정해 이동 쾌감 확보
- 낮은 이동 속도(3 이하)는 "느리다"는 불편감 → "기어다니는 플레이"의 의도와도 상충. 4가 최솟값

### 2.3 충돌 처리

| 충돌 대상 | 처리 방식 |
|-----------|-----------|
| 농장 경계 울타리 | 이동 불가. Rigidbody + Collider 충돌 |
| 건물/시설 | 이동 불가. 건물 외곽 Collider |
| 나무/바위 (환경 장식) | 이동 불가. Static Collider |
| 작물 타일 | 통과 가능. 작물 Collider는 인터랙션용(IsTrigger)으로만 사용 |
| 다른 NPC | [OPEN] NPC 충돌 처리 방식 미결정. 기본적으로 통과 허용 (밀리는 현상 방지) |

## 3. 인터랙션 시스템

### 3.1 타일 인터랙션 반경

플레이어 중심에서 **1.5 tiles** 이내의 대상에 인터랙션 가능하다.

```
  [1.5 tiles radius 시각화]

  . . . . .
  . X X X .
  . X P X .   P = 플레이어, X = 인터랙션 가능 범위
  . X X X .
  . . . . .
```

**수치 근거**:
- 1 tile: 바로 옆 타일만 가능 → 정밀하지만 이동 빈도가 너무 높아 피로감 유발
- 2 tiles: 인터랙션이 너무 넓어 실수로 인접 타일을 건드리는 오조작 발생
- 1.5 tiles: 상하좌우 1칸 + 대각선 1칸 커버. 가장 일반적인 농장 시뮬레이션 반경. Stardew Valley와 동일 수준

### 3.2 인터랙션 가능 대상 목록

| 대상 | 인터랙션 유형 | 트리거 방식 |
|------|-------------|------------|
| 빈 타일 (Empty) | 호미로 경작 (Tilled 전환) | 마우스 좌클릭 (도구 장착 시) |
| 경작 타일 (Tilled) | 씨앗 심기 (Planted 전환) | 마우스 좌클릭 (씨앗 장착 시) |
| 심어진 타일 (Planted) | 물주기 (Watered 전환) | 마우스 좌클릭 (물뿌리개 장착 시) |
| 수확 가능 타일 (Harvestable) | 수확 (낫 또는 손) | 마우스 좌클릭 |
| 상점 (Shop) | 구매/판매 UI 오픈 | E 키 |
| 출하함 (ShippingBin) | 작물 투입 | E 키 또는 마우스 좌클릭 |
| NPC | 대화 시작 | E 키 |
| 건물/시설 | 시설 UI 오픈 | E 키 |

### 3.3 인터랙션 우선순위

동일 반경 내에 복수의 인터랙션 대상이 겹칠 경우 다음 우선순위를 적용한다:

| 우선순위 | 대상 | 이유 |
|---------|------|------|
| 1 | 마우스 커서가 직접 가리키는 타일/오브젝트 | 플레이어 의도를 최우선 반영 |
| 2 | NPC / 시설 (E키 인터랙션 대상) | 실수로 수확 또는 경작이 트리거되는 것을 방지 |
| 3 | 수확 가능 작물 | 수확은 되돌릴 수 없으므로 명시적 클릭 요구 |
| 4 | 경작/물주기 대상 타일 | 오조작 시 수정 가능한 행동 |

## 4. 도구 시스템 연동

### 4.1 장비 슬롯 (1~5 숫자키)

도구바는 5개 슬롯으로 구성되며 숫자키 1~5로 선택한다.

| 슬롯 | 기본 숫자키 | 기본 도구 |
|------|-----------|----------|
| 1 | 1 | 호미 (Hoe) |
| 2 | 2 | 물뿌리개 (Watering Can) |
| 3 | 3 | 씨앗 (Seeds) — 감자 x5, 당근 x5 지급 |
| 4 | 4 | 낫 (Scythe) |
| 5 | 5 | 손 (Hand) — 기본 상호작용, 아이템 줍기 |

### 4.2 기본 도구 목록 및 기능

| 도구 ID | 이름 | 기능 | 에너지 소모 |

레벨 업 또는 대장장이 NPC를 통해 추가 도구 슬롯 확장 및 신규 도구 장착이 가능하다.

| 도구 | 해금 조건 | 기능 |

| 강화 호미 / 물뿌리개 / 낫 | 레벨 3+ 업그레이드 | (→ see `docs/systems/tool-upgrade.md`) |

## 5. 애니메이션 상태 머신

### 5.1 상태 목록 및 전환 조건

```
             [입력 없음]
    Idle ←──────────────── Walk
      │                      ↑
      │   [WASD 입력]         │
      └──────────────────────┘
      │
      │ [마우스 좌클릭 / 도구 사용]
      ↓
  Tool_Dig (호미)
  Tool_Water (물뿌리개)
  Tool_Harvest (낫/손)
  Tool_Plant (씨앗)
      │
      │ [애니메이션 완료]
      ↓
    Idle
      │
      │ [E키 / NPC 인터랙션]
      ↓
    Talk
      │
      │ [대화 종료]
      ↓
    Idle
```

| 상태 ID | 이름 | 진입 조건 | 종료 조건 | 루프 |
|---------|------|-----------|-----------|------|
| `Idle` | 대기 | WASD 입력 없음, 도구 미사용 | WASD 입력 또는 도구 사용 | O |
| `Walk` | 걷기 | WASD 입력 | 입력 없음 | O |
| `Tool_Dig` | 호미 사용 | 호미 장착 + 마우스 좌클릭 (인터랙션 범위 내 타일) | 단발 애니메이션 완료 | X |
| `Tool_Water` | 물 주기 | 물뿌리개 장착 + 마우스 좌클릭 | 단발 애니메이션 완료 | X |
| `Tool_Harvest` | 수확 | 낫/손 장착 + 마우스 좌클릭 (Harvestable 타일) | 단발 애니메이션 완료 | X |
| `Tool_Plant` | 씨앗 심기 | 씨앗 장착 + 마우스 좌클릭 (Tilled 타일) | 단발 애니메이션 완료 | X |
| `Talk` | 대화 | E키 + 인터랙션 범위 내 NPC/시설 | 대화 UI 닫기 | X |

**도구 사용 중 이동 제한**: `Tool_*` 및 `Talk` 상태에서는 이동 입력을 무시한다. 단발 애니메이션(약 0.4~0.6초)이 완료된 후 `Idle`로 복귀하며, 이후 입력이 가능하다.

### 5.2 애니메이션 클립 목록

| 클립 ID | 상태 | 길이(초) | 비고 |
|---------|------|----------|------|
| `Anim_Idle` | Idle | ∞ (루프) | 미세한 호흡 모션 포함 |
| `Anim_Walk` | Walk | 0.6 (루프) | 걷기 사이클. 8방향 모두 동일 클립 + 방향 회전 |
| `Anim_Dig` | Tool_Dig | 0.5 | 호미를 땅에 내리꽂는 동작 |
| `Anim_Water` | Tool_Water | 0.6 | 물뿌리개를 앞으로 기울이는 동작 |
| `Anim_Harvest` | Tool_Harvest | 0.5 | 낫을 옆으로 스윙하거나 손으로 뽑는 동작 |
| `Anim_Plant` | Tool_Plant | 0.5 | 쪼그려 앉아 씨앗을 심는 동작 |
| `Anim_Talk` | Talk | ∞ (루프) | 고개를 살짝 끄덕이는 대기 모션 |

## 6. 카메라 시스템

### 6.1 시점 방식 확정: 쿼터뷰 (Orthographic, 45도)

**결정**: 탑다운이 아닌 **쿼터뷰(X=45°, Y=45°) 고정** 방식을 채택한다.

**결정 근거**:

| 비교 항목 | 쿼터뷰 (채택) | 탑다운 |
|----------|-------------|--------|
| 캐릭터 식별성 | 캐릭터 옆면이 보여 존재감 명확 | 머리 위만 보여 캐릭터 개성 표현 어려움 |
| 건물/시설 표현 | 입체감 있는 건물 외벽 표현 가능 | 지붕만 보임 — 로우폴리 건물의 매력 반감 |
| 작물 성장 표현 | 성장 단계별 높이 변화가 시각적으로 드러남 | 높이 변화 불명확 |
| 이동 직관성 | WASD 입력과 시각적 방향의 약간의 어긋남 존재 | WASD가 화면 방향과 1:1 일치 |
| 아트 스타일 적합성 | Islanders, A Short Hike 레퍼런스와 일치 | 덜 스타일라이즈드한 느낌 |

이동 방향 어긋남(쿼터뷰의 유일한 단점)은 섹션 2.1의 입력 변환 매핑으로 직관적으로 처리한다.

> **이 결정으로 `docs/design.md` Open Questions의 "쿼터뷰 vs 탑다운" 항목이 확정되었다.**
> `docs/mcp/scene-setup-tasks.md` 섹션 1.6, 3.1의 [OPEN] 태그도 이 문서 결정을 따른다.

### 6.2 추적 방식 및 파라미터

**추적 방식 결정: Cinemachine Virtual Camera**

| 비교 항목 | Cinemachine (채택) | SmoothDamp (코드 직접 구현) |
|----------|-------------------|--------------------------|
| 파라미터 조정 | Inspector에서 실시간 조정 가능 | 코드 수정 후 재컴파일 |
| Dead Zone 지원 | 기본 제공 (플레이어가 중앙에서 일정 범위 이동해도 카메라 정지) | 직접 구현 필요 |
| 충돌 처리 | 확장 가능 | 미지원 |
| Unity 6 호환성 | 공식 지원 | 기본 내장 |

**Dead Zone 설정 근거**: 카메라가 플레이어를 항상 화면 정중앙에 고정하면, 마우스 커서로 인근 타일을 조작할 때 카메라가 미세하게 흔들려 불쾌한 경험이 발생한다. Dead Zone을 두어 작은 이동에는 카메라가 반응하지 않도록 한다.

### 6.3 카메라 설정값

| 파라미터 | 값 | 설명 |
|---------|-----|------|
| Projection | Orthographic | 원근 왜곡 없음. 로우폴리 스타일에 적합 |
| X Rotation | 45° | 쿼터뷰 수직 각도 |
| Y Rotation | 45° | 쿼터뷰 수평 각도 (북서쪽을 위로) |
| Orthographic Size | 6 (초기값) | 8x8 그리드가 화면에 적절히 담기는 크기 |
| Zoom 범위 | Size 4~10 | 줌인(개별 타일 상세) ~ 줌아웃(농장 전체 조망) |
| 줌 입력 | 마우스 휠 | 부드러운 보간 전환 |
| Background Color | `#2C3E50` | 어두운 청록색. 로우폴리 스카이박스 없을 때 기본 배경 |
| Cinemachine Damping X | 0.5 | 좌우 추적 부드러움 |
| Cinemachine Damping Y | 0.5 | 상하 추적 부드러움 |
| Dead Zone (X/Y) | 0.15 | 플레이어가 화면 중앙 15% 이내 이동 시 카메라 정지 |
| Soft Zone (X/Y) | 0.3 | Dead Zone 외곽~Soft Zone 경계에서 서서히 추적 시작 |
| 카메라 오프셋 (높이) | Y + 8m | 플레이어 위 8m 지점에서 45도로 내려다보는 위치 |

### 7.1 모델 스펙

## 8. 레벨 및 스탯 연동

### 8.1 에너지 시스템 연동

플레이어의 도구 사용은 에너지를 소모한다. 에너지가 0이 되면 캐릭터가 기절하고 다음 날 시작 지점에서 부활한다.

에너지 소모 수치 및 회복 규칙: (→ see `docs/systems/energy-system.md`)

| 연동 지점 | 설명 |
|-----------|------|
| `Tool_Dig` 상태 진입 | 호미 에너지 소모 트리거 |
| `Tool_Water` 상태 진입 | 물뿌리개 에너지 소모 트리거 |
| `Tool_Harvest` 상태 진입 | 낫 에너지 소모 트리거 |
| 에너지 = 0 도달 | 강제 기절 이벤트 → `Idle` 상태 고정 → 씬 전환 |

### 8.2 도구 강화 연동

도구 등급(Basic → Reinforced → Legendary)에 따라 도구 사용 효과 범위와 에너지 효율이 달라진다. 업그레이드 조건 및 효과: (→ see `docs/systems/tool-upgrade.md`)

| 연동 지점 | 설명 |
|-----------|------|
| 호미 등급 | Reinforced: 1x3 범위 경작, Legendary: 3x3 범위 경작 (단발 애니메이션 동일, 타일 일괄 처리) |
| 물뿌리개 등급 | Reinforced: 1x3, Legendary: 3x3 범위 물주기 |
| 낫 등급 | Reinforced: 이동 속도 패널티 없이 수확, Legendary: 1x3 범위 일괄 수확 |

## 11. Risks

- [RISK] Cinemachine Dead Zone 파라미터는 실제 플레이 테스트 없이 확정이 어렵다. 섹션 6.3의 수치(0.15, 0.3)는 1차 추정치이며 Phase 2 테스트 후 조정이 필요하다.
- [RISK] 쿼터뷰 이동 방향 변환(섹션 2.1 매핑 테이블)이 플레이어에게 직관적으로 느껴지지 않을 수 있다. 튜토리얼에서 명시적 안내가 필요하다 (→ see `docs/systems/tutorial-system.md`).
- [RISK] 로우폴리 캐릭터 모델을 MCP만으로 생성하는 데 품질 한계가 존재한다. 외부 에셋 스토어(Unity Asset Store) 활용을 Phase 2 시작 시 검토해야 한다.
- [RISK] 도구 사용 중 이동 불가(0.4~0.6초 잠금) 설계가 고속 플레이어에게 불편감을 줄 수 있다. 애니메이션 길이를 0.3초 이하로 줄이거나 이동 허용 여부를 Phase 2에서 재검토.
