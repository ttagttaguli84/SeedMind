# 농장 장식 시스템 (Decoration System) 상세 설계

> 작성: Claude Code (Sonnet 4.6) | 2026-04-08
> 문서 ID: DES-023

---

## Context

이 문서는 SeedMind의 농장 장식/꾸미기 시스템을 상세히 기술한다. 마스터 디자인 문서(`docs/design.md`) 섹션 2의 핵심 감정("농장 꾸미기의 즐거움") 및 섹션 4.6 시설 목록을 기반으로, 울타리·경로·조명·장식물 배치 메카닉, 카테고리 분류, 경제 연동, 해금 조건을 다룬다.

**설계 목표**: 장식 시스템은 게임 핵심 루프(경작-수확-판매-투자)와 독립적으로 동작하는 **표현적 레이어**다. 농장 운영 효율에 직접적인 영향을 주지 않으며, 플레이어가 번 골드를 자신만의 농장 정체성을 구현하는 데 소모할 수 있는 **골드 소모처**로 기능한다.

**필요 여부 결정**: DES-023 검토 결과 장식 시스템은 포함한다. 근거:

| 근거 | 설명 |
|------|------|
| 핵심 감정 일치 | design.md 섹션 2 "핵심 감정"에 "농장 꾸미기의 즐거움"이 명시됨 |
| 골드 소모처 부재 | 중후반 골드 잉여 구간에서 소모처가 없으면 경제 밸런스가 무의미해짐 |
| 레퍼런스 검증 | Stardew Valley·Harvest Moon 등 장르 표준 시스템 |
| 스코프 제어 가능 | 기능적 메카닉이 없어도 데이터 정의만으로 구현 가능 |

**스코프 원칙**: 장식품은 생산성 보너스를 주지 않는다. 예외적으로 경미한 환경 효과(조명 야간 가시성, 경로 이동속도)는 허용하되, 핵심 수익 루프와 연결되지 않아야 한다.

---

## 1. 시스템 개요

### 1.1 장식 시스템 위상

```
핵심 루프 (경작 → 판매 → 투자)
         │
         ├─ 시설 건설 (생산성 향상)
         ├─ 구역 확장 (farming area 증가)
         └─ 장식 배치 (표현적 레이어, 골드 소모처)
```

장식 시스템은 핵심 루프와 연결되지 않은 독립 레이어다. 장식을 안 해도 게임 진행에 불이익이 없고, 해도 생산 효율에 직접 이득이 없다.

### 1.2 장식 오브젝트 분류

| 카테고리 | 영문 ID | 설명 | 배치 방식 |
|----------|---------|------|----------|
| 울타리 | `Fence` | 구역 경계 표시, 동물 차단 | 엣지(edge) 배치 (타일 경계) |
| 경로 | `Path` | 타일 위에 깔리는 바닥 텍스처 | 타일 오버레이 |
| 조명 | `Light` | 야간 가시 반경 제공 | 타일 점유 (1x1) |
| 장식물 | `Ornament` | 조각상·화분·벤치 등 | 타일 점유 (1x1 또는 2x2) |
| 수경 장식 | `WaterDecor` | 연못·분수 (Zone F 전용) | 타일 점유 (2x2~3x3) |

---

## 2. 카테고리 상세 설계

### 2.1 울타리 (Fence)

#### 개요

울타리는 타일 경계(edge)에 배치되며, 방향에 따라 수평/수직 변형 스프라이트가 자동 적용된다. 코너 연결 스프라이트가 자동 생성된다(auto-tiling).

#### 기능 효과

| 효과 | 설명 |
|------|------|
| 경계 표시 | 시각적 구역 구분. 게임플레이 효과 없음 |
| 동물 이동 차단 | Zone E 목장 동물이 외부 구역으로 탈출하지 못하도록 차단. **[OPEN#1] 동물 이동 메카닉과 연동 여부는 livestock-system.md 확정 후 재검토 필요** |
| 내구도 | 기본 나무 울타리는 계절당 내구도 하락. 수리 비용 소모. 돌·금속 울타리는 영구적 |

#### 울타리 종류

| 이름 | 영문 ID | 구매가 | 내구도 | 해금 |
|------|---------|--------|--------|------|
| 나무 울타리 | `FenceWood` | 5G/단 | 계절당 10% 감소, 10계절 | 시작 |
| 돌 울타리 | `FenceStone` | 15G/단 | 영구 | 레벨 3 |
| 쇠 울타리 | `FenceIron` | 30G/단 | 영구 | 레벨 6 |
| 꽃 울타리 | `FenceFloral` | 20G/단 | 영구 | 레벨 5, 봄 한정 계절 판매 |

**내구도 설계 의도**: 나무 울타리의 수리 비용은 주기적 골드 소모를 유도하되, 규모가 크지 않아 부담스럽지 않아야 한다.

**수리**: 나무 울타리가 내구도 0에 도달하면 자동 철거되지 않고 "부서진 울타리" 상태로 전환된다. 플레이어가 클릭하면 구매가의 20%로 수리 가능.

### 2.2 경로 (Path)

#### 개요

경로는 타일 위에 덧씌워지는 바닥 텍스처다. 경작지 타일에는 배치 불가 (Farmland 타일과 충돌). 잔디·흙 타일에만 배치 가능.

#### 기능 효과

| 효과 | 설명 |
|------|------|
| 이동 속도 보너스 | 경로 위 이동 시 +10% 속도. **의도적으로 작게 설정** — 장식 목적이 주다 |
| 잡초 억제 | 경로가 깔린 타일에는 잡초가 자라지 않음 |
| 비 후 진흙 방지 | 비 다음날 흙 타일이 진흙 상태가 되지 않음 |

#### 경로 종류

| 이름 | 영문 ID | 구매가 | 재질 | 해금 |
|------|---------|--------|------|------|
| 흙 다짐 경로 | `PathDirt` | 2G/타일 | 흙 | 시작 |
| 자갈 경로 | `PathGravel` | 5G/타일 | 자갈 | 레벨 2 |
| 돌판 경로 | `PathStone` | 12G/타일 | 돌 | 레벨 4 |
| 벽돌 경로 | `PathBrick` | 18G/타일 | 벽돌 | 레벨 6 |
| 목판 경로 | `PathWood` | 10G/타일 | 나무 | 레벨 3 |

**설계 제약**: 경로 위 이동속도 보너스는 최대 +10%로 제한. 경로 확장이 게임 효율 극대화 전략으로 변질되지 않도록 수치를 낮게 유지.

### 2.3 조명 (Light)

#### 개요

조명은 1x1 타일을 점유하는 오브젝트다. 야간(20:00~06:00) 시간대에 주변 타일의 가시성을 확보한다. 게임 내 조명이 없는 야간에는 화면이 어두워져 작업 효율이 감소한다. (→ see `docs/systems/time-season.md` 야간 가시성 규칙)

#### 기능 효과

| 효과 | 파라미터 | 설명 |
|------|----------|------|
| 가시 반경 | `lightRadius` | 조명 중심으로 N타일 반경 내 야간 가시성 확보 |
| 미관 효과 | - | 로우폴리 광원 이펙트 (점광원 스타일) |

#### 조명 종류

| 이름 | 영문 ID | 구매가 | lightRadius | 해금 | 특이사항 |
|------|---------|--------|-------------|------|---------|
| 횃불 | `LightTorch` | 30G | 2타일 | 레벨 2 | 비·눈 날씨에 꺼짐 (시각 이벤트) |
| 등롱 | `LightLantern` | 80G | 3타일 | 레벨 4 | 항상 켜짐 |
| 가로등 | `LightStreet` | 200G | 5타일 | 레벨 6 | 항상 켜짐, 2x1 타일 점유 |
| 마법 수정 조명 | `LightCrystal` | 500G | 4타일 | 레벨 8 | 색상 변경 가능 (UI 팔레트) |

**[OPEN#2] 야간 조명 가시성 규칙은 time-season.md에 아직 정의되지 않았다. 본 문서는 야간 가시성 메카닉이 존재한다고 전제하나, 야간이 실제로 어두워지는 구현 여부는 ARC 단계에서 결정되어야 한다.**

### 2.4 장식물 (Ornament)

#### 개요

장식물은 타일을 점유하는 순수 미관 오브젝트다. 생산성 보너스 없음. 플레이어가 농장의 개성을 표현하는 주요 수단.

| 돌 조각상 | `OrnaStatueStone` | 300G | 1x1 | 레벨 5 | 없음 |
| 풍차 (소형) | `OrnaWindmillS` | 400G | 2x2 | 레벨 5 | 없음 |

**농장 표지판**: 텍스트 입력 가능 (최대 20자). 농장 이름 표시 등 활용.

## 3. 배치 메카닉

### 3.1 배치 방식: 그리드 고정 배치

장식품은 **그리드 고정 배치(grid-snapped placement)** 방식을 사용한다. 자유 배치(free placement)는 채택하지 않는다.

**결정 근거**:

| 방식 | 장점 | 단점 |
|------|------|------|
| 그리드 고정 | 구현 단순, 타일 시스템과 일관성, 충돌 검사 명확 | 세밀한 표현 제한 |
| 자유 배치 | 표현 자유도 높음 | 타일 시스템과 충돌, 구현 복잡, 충돌 검사 어려움 |

농장이 타일 기반 그리드로 운영되므로(→ see `docs/systems/farming-system.md` 섹션 1), 장식 배치도 그리드에 맞추는 것이 아키텍처적으로 일관성이 있다.

### 3.2 배치 UI 플로우

```
[플레이어] 인벤토리 또는 장식 메뉴 열기
    │
    ▼
[장식 배치 모드 진입]
    │  - 커서가 타일 하이라이트 모드로 전환
    │  - 오브젝트 프리뷰(반투명) 표시
    ▼
[타일 위에 커서 이동]
    │  ├─ 배치 가능: 초록색 하이라이트
    │  └─ 배치 불가: 빨간색 하이라이트 (타일 점유 충돌, 경작지, 수원 등)
    ▼
[좌클릭 확정] 또는 [우클릭/Esc 취소]
    │
    ▼
[장식물 배치 완료]
    │  - 인벤토리에서 차감 (또는 골드 직접 차감)
    │  - 철거: 장식물 클릭 후 E키 → "철거" 선택 → 구매가 0% 환불 (소모형)
```

**철거 정책**: 장식품을 철거해도 골드 환불이 없다. 장식에 쓴 골드는 회수 불가. 이는 경제적 의사결정에 무게를 부여하기 위함이다.

**[OPEN#3] 철거 시 100% 환불 vs 0% 환불은 플레이테스트 후 재결정 가능. 현재는 0% 환불(소모형)로 설계하나, 플레이어 경험 관점에서 50% 부분 환불도 검토 대상.**

### 3.3 배치 제약

| 제약 | 설명 |
|------|------|
| 경작지 타일 위 장식 금지 | Farmland 타일 위에 장식물·경로 배치 불가. 경작지를 장식으로 막으면 작물 심기 불가 |
| 수원(우물·연못) 위 금지 | 물 관련 오브젝트 위에 장식 배치 불가 |
| 시설 타일 위 금지 | 건물 점유 타일 위에 배치 불가 |
| 구역 미해금 타일 금지 | 아직 해금되지 않은 Zone의 타일에 배치 불가 |
| 울타리: 엣지 배치만 허용 | 울타리는 타일 점유가 아닌 타일 경계(edge)에 배치. 타일 자체를 막지 않음 |

### 3.4 구매 방법

장식품은 두 가지 경로로 구매한다:

| 경로 | 설명 |
|------|------|
| 상점 직구매 | 목공소(Carpenter) 또는 잡화점 NPC에서 직접 구매. 골드 즉시 차감. 인벤토리에 추가 |
| 이벤트/퀘스트 보상 | 축제 보상, 퀘스트 완료 보상으로 획득 (→ see `docs/systems/quest-system.md`) |

**인벤토리 처리**: 장식품은 일반 인벤토리 슬롯을 차지한다. 장식품 전용 창고 슬롯은 없다. (→ see `docs/systems/inventory-system.md`)

## 4. 해금 조건

### 4.1 해금 방식

장식품은 플레이어 레벨 및 구역 해금에 연동된다. 별도의 "장식 스킬 트리"는 없다.

| 조건 유형 | 설명 |
|----------|------|
| 레벨 해금 | 플레이어 레벨이 N 이상이면 상점에 등장 |
| 구역 해금 | 특정 Zone 해금 후 관련 장식 등장 (수경 장식 = Zone F) |
| 계절 한정 | 특정 계절에만 상점 판매. 구매 후 영구 보유 |
| 이벤트 한정 | 축제·퀘스트 보상으로만 입수 가능 |

### 4.2 해금 테이블 요약

| 레벨 | 신규 해금 장식 카테고리 |
|------|----------------------|
| 시작 | 흙 다짐 경로, 나무 울타리, 나무 허수아비, 농장 표지판 |
| 레벨 2 | 자갈 경로, 횃불, 꽃 화분 (소) |
| 레벨 3 | 목판 경로, 나무 벤치, 돌 울타리, 꽃 화분 (대), 눈사람 (겨울 한정) |
| 레벨 4 | 돌판 경로, 등롱, 우물 장식 |
| 레벨 5 | 풍차 (소형), 꽃 울타리 (봄 한정), 돌 조각상, 호박 등불 (가을 한정) |
| 레벨 6 | 벽돌 경로, 쇠 울타리, 가로등 |

## 5. 경제 연동

### 5.1 골드 소모처 역할

장식 시스템의 경제적 역할은 **후반 골드 잉여 구간의 흡수**다. 게임 진행상 시설 건설과 구역 확장이 완료된 이후에도 플레이어는 계속 골드를 수령하므로, 의미 있는 소모처가 필요하다.

경제 수치 참조: (→ see `docs/systems/economy-system.md` 초기 지급 골드, 연간 수익 추정치)

**설계 방향**:
- 소량 구매 (2G 경로 타일 단위) ~ 대량 구매 (2,000G 황금 조각상) 까지 골드 소비 스펙트럼 제공
- 단일 아이템 최대 가격을 2,000G로 상한 설정. 시설 건설(최소 500G~5,000G)과 가격대 중복 회피 불필요 — 장식은 "지금 당장 안 사도 되는" 선택적 소비다
- 계절 한정 아이템의 존재로 "다음 계절에 사야지"라는 저축 동기 유발

### 5.2 장식 지출 추정

완전 장식(전 구역, 모든 경로 포함 가정) 시 예상 누적 지출 규모 예시 (튜닝 파라미터, 확정 수치 아님):

| 범주 | 예상 규모 |
|------|---------|
| 경로 (Zone A 전체 50타일 흙 다짐) | ~100G |
| 경로 (전 구역 돌판 200타일) | ~2,400G |
| 울타리 (농장 외곽 전체 돌 울타리) | ~1,500G |
| 조명 (각 Zone 1개씩) | ~500G |
| 장식물 (중간 수준 선택) | ~1,000~3,000G |
| 수경 장식 (Zone F 완전 장식) | ~2,000~3,000G |
| **총 예상 범위** | **7,500G ~ 12,500G** |

이 규모는 (→ see `docs/systems/economy-system.md` 섹션 1.2~1.3) 경제 수치 기준으로 1년차 후반~2년차 수준의 소비 규모에 해당한다. 필수 지출이 아니므로 게임 진행에 지장을 주지 않는다.

## 6. 장식품 데이터 스키마

### 6.1 DecorationItemData 구조 (예시)

장식품 데이터는 ScriptableObject로 정의한다. 아래는 설계 예시이며 실제 구현 스키마는 data-pipeline.md에서 확정된다 (→ see `docs/pipeline/data-pipeline.md`).

```
// DecorationItemData 필드 예시 (문서용, 실제 코드 아님)
DecorationItemData {
    string  itemId           // 예: "FenceStone"
    string  displayName      // 예: "돌 울타리"
    DecoCategoryType category // Fence / Path / Light / Ornament / WaterDecor
    int     buyPrice         // 구매가 (G)  → see canonical: 섹션 2.1~2.5
    int     tileWidthX       // 점유 타일 X  (울타리는 0, edge 배치)
    int     tileHeightZ      // 점유 타일 Z  (울타리는 0, edge 배치)
    bool    isEdgePlaced     // true = 울타리 (edge), false = tile 점유
    int     unlockLevel      // 해금 레벨 (0 = 시작부터)
    string  unlockZoneId     // 해금 구역 ID, 없으면 ""  → see docs/systems/farm-expansion.md
    Season  limitedSeason    // 계절 한정 판매 (None = 항상 판매)
    float   lightRadius      // 조명만 사용, 나머지는 0
    float   moveSpeedBonus   // 경로만 사용 (0.1 = +10%), 나머지는 0
    int     durabilityMax    // 울타리 내구도 최대값 (0 = 영구)
}
```

**확장 원칙**: DecoCategoryType 열거형에 새 카테고리 추가 시, 해당 카테고리를 처리하는 배치 시스템과 렌더러 코드 예시도 동시에 업데이트해야 한다.

## Risks

- [RISK] 스코프 크리프: 장식 시스템은 콘텐츠 추가가 쉬워 Phase 2에서 지속 확장될 수 있다. MVP에서 카테고리 5종, 아이템 총 30개 이하로 제한하고 추가는 Phase 3+ 이후로 미룬다.
- [RISK] 타일 점유 충돌 복잡도: 2x2 이상 장식물과 경로·울타리·시설이 혼재하는 경우 배치 충돌 검사 로직이 복잡해질 수 있다. 그리드 고정 배치로 구현 단순화를 도모했으나, SO 데이터에서 tileWidthX/tileHeightZ 필드 정합성 관리가 필요하다.
- [RISK] 야간 조명 효과의 플랫폼 의존성: 야간 어두움 효과는 PC 디스플레이 밝기에 따라 효과 차이가 클 수 있다. 접근성 옵션으로 야간 어두움 비활성화 설정 고려 필요.
