## Context

이 문서는 SeedMind의 전체 비주얼 방향성을 정의한다. 아트 스타일, 색상 팔레트, 조명 체계, 캐릭터/오브젝트 스타일, UI 비주얼, 성장 단계 시각화 원칙을 포함한다.

마스터 디자인 문서(`docs/design.md`) 섹션 7의 비주얼 방향 개요, `docs/systems/time-season.md`의 계절별 환경 변화, `docs/systems/ui-system.md`의 UI 시각 스타일, `docs/content/crops.md`의 작물별 시각 묘사를 기반으로 통합 확장한다.

**설계 목표**: 로우폴리 스타일라이즈드 3D 아트로 따뜻하고 아늑한 농장 분위기를 전달한다. 계절과 시간의 변화가 시각적으로 뚜렷하게 느껴지며, 플레이어의 농장이 성장하는 과정이 눈에 보이는 보람으로 이어져야 한다.

## 1. 아트 방향성 (Art Direction)

### 1.1 스타일 정의

SeedMind는 **로우폴리 스타일라이즈드 3D**(Low-poly Stylized 3D)를 채택한다.

| 항목 | 정의 |
|------|------|
| 렌더링 방식 | Unity URP (Universal Render Pipeline) |
| 셰이딩 | Flat Shading 기본, 부분적 Smooth Shading (캐릭터 얼굴, 둥근 과일) |
| 텍스처 | **텍스처리스(Textureless)** 기본. 버텍스 컬러 또는 단색 머티리얼 사용. 필요 시 최소한의 컬러 그라데이션 텍스처 허용 |
| 외곽선 | 외곽선(Outline) 효과 없음. 형태와 색상으로 오브젝트를 구분 |
| 그림자 | 소프트 섀도우. 지나치게 날카로운 그림자는 로우폴리 분위기를 해침 |

### 1.2 폴리곤 수 가이드라인

오브젝트 카테고리별 폴리곤 예산을 정한다. 쿼터뷰 시점에서 한 화면에 수십 개의 오브젝트가 동시에 보일 수 있으므로, 개별 오브젝트의 폴리곤 수를 엄격히 제한한다.

| 카테고리 | 폴리곤 예산 (삼각형 기준) | 비고 |
|----------|-------------------------|------|
| 플레이어 캐릭터 | 800~1,200 tri | 가장 상세한 오브젝트 |
| NPC | 600~1,000 tri | 플레이어보다 약간 단순 |
| 작물 (Harvestable 단계) | 50~200 tri | 성장 단계에 따라 증가 |
| 작물 (Seed/Sprout) | 10~30 tri | 극도로 단순한 형태 |
| 나무 | 300~600 tri | 계절별 변형은 머티리얼 교체로 처리 |
| 건물/시설 | 400~1,500 tri | 크기에 비례. 온실(6x6) 최대 |
| 지형 타일 | 타일당 4~8 tri | 플랫 평면 기본, 가장자리 약간의 높이 변화 |
| 소품 (울타리, 표지판 등) | 20~100 tri | 반복 배치되므로 최소화 |
| 도구 | 30~80 tri | 플레이어 손에 들리는 크기 |

### 1.3 에지 처리 방식

| 방식 | 적용 대상 | 설명 |
|------|----------|------|
| Hard Edge (Flat) | 건물, 울타리, 도구, 지형 | 면이 뚜렷하게 구분되는 로우폴리 특유의 기하학적 느낌 |
| Soft Edge (Smooth) | 캐릭터 몸체, 둥근 과일(토마토, 수박, 호박), 물 | 부드러운 곡면이 필요한 유기적 형태 |
| 혼합 | 나무 (줄기: Hard, 잎 덩어리: Soft) | 자연물의 자연스러운 느낌 |

### 1.4 레퍼런스 톤

(-> see `docs/design.md` 섹션 7 for 레퍼런스 게임 목록)

목표하는 비주얼 느낌을 세 가지 축으로 정의한다.

| 축 | 레퍼런스 | 차용 요소 |
|----|----------|----------|
| **기하학적 단순함** | Islanders, Townscaper | 복잡한 디테일 없이 형태만으로 읽히는 건물과 지형 |
| **따뜻한 색감** | A Short Hike, Ooblets | 파스텔 톤의 부드러운 색상, 그림자에도 색이 있는 느낌 |
| **농장 분위기** | Stardew Valley (2D 참고), Harvest Moon | 작물이 자라는 밭, 소박한 농가, 계절 변화의 정서 |

### 1.5 분위기 키워드

게임 전반에 걸쳐 유지해야 할 감정적 톤을 정의한다.

| 키워드 | 설명 | 비주얼 표현 |
|--------|------|------------|
| **따뜻함 (Warmth)** | 포근하고 안전한 공간 | 난색 위주 팔레트, 부드러운 조명 |
| **아늑함 (Cozy)** | 나만의 농장에 대한 애착 | 작은 스케일의 디테일, 둥근 형태 |
| **계절감 (Seasonality)** | 시간의 흐름이 눈에 보임 | 계절별 색상 대비, 환경 오브젝트 변화 |
| **성장 (Growth)** | 작물과 농장이 함께 자라는 느낌 | 단계적 크기/색상 변화, 시설 추가의 시각적 임팩트 |
| **소박함 (Rustic)** | 시골 농장의 단순하고 정직한 미감 | 나무/돌 소재 느낌, 과도한 장식 배제 |

## 2. 색상 팔레트

모든 색상은 HEX 코드로 표기한다. 각 팔레트는 Unity 머티리얼 또는 ScriptableObject 색상 프리셋으로 관리한다.

### 3.3 계절별 조명 특성

계절별 Directional Light 색온도 및 Post-Processing: (-> see `docs/systems/time-season.md` 섹션 2.3, canonical)

각 계절은 시간대 조명에 계절 보정을 덮어씌운다.

### 3.4 날씨가 비주얼에 미치는 영향

날씨 종류 및 확률: (-> see `docs/systems/time-season.md` 섹션 3.1~3.2, canonical)

| 날씨 | 조명 변화 | 파티클 효과 | 추가 비주얼 |
|------|----------|------------|------------|
| 맑음 (Clear) | 기본 시간대 조명 그대로 | 없음 | 없음 |
| 흐림 (Cloudy) | Directional Light 강도 -20%, 색상 회색 가중 | 없음 | 하늘 텍스처 구름 추가 |
| 비 (Rain) | Directional Light 강도 -30%, Ambient 어두움 | 빗방울 파티클 (중간 밀도) | 물웅덩이 생성, 땅 색상 짙어짐 |
| 폭우 (HeavyRain) | Directional Light 강도 -40% | 빗방울 파티클 (높은 밀도, 큰 입자) | 물웅덩이 확대, 화면 가장자리 물방울 효과 |
| 폭풍 (Storm) | Directional Light 강도 -50%, 간헐적 번개 플래시 | 빗방울 + 번개 파티클 | 번개 시 화면 전체 순간 밝아짐 (`#FFFFFF`, 0.1초), 나무 흔들림 증가 |
| 눈 (Snow) | Directional Light 강도 -10%, 색온도 한층 차가움 | 눈 파티클 (느리게 내림, 중간 밀도) | 지면에 점진적 적설 효과 (흰색 레이어 증가) |
| 폭설 (Blizzard) | Directional Light 강도 -35%, 시야 크게 제한 | 눈 파티클 (빠르게 옆으로 날림, 높은 밀도) | 시야 반경 축소 (비네팅 강화), 흰색 안개 |

**번개 연출 상세**:
- 빈도: 폭풍 중 평균 30~60초 간격
- 연출: 화면 순간 밝아짐(0.1초) + 0.5초 후 천둥 사운드
- 접근성: 화면 번쩍임 끄기 옵션 시, 플래시 대신 부드러운 글로우로 대체 (-> see `docs/systems/ui-system.md` 섹션 10.3)

4단계의 시각적 변화에 대한 **비주얼 디자인 원칙**을 정의한다.

| 단계 | 시각적 원칙 | 색상 | 크기 | 디테일 |
|------|------------|------|------|--------|
| Seed | **거의 보이지 않음**. 흙더미 위 작은 돌기. 씨앗 종류를 구분하는 작은 색상 태그(꼬리표) | 흙색 + 태그 색상 | 0.05m | 최소. 3~4 폴리곤 |
| Sprout | **기대감**. 연한 새싹이 돋아남. 모든 작물이 비슷한 형태 | 연두 `#AED581` | 0.15m | 줄기 1~2개 + 잎 1~2장 |
| Growing | **정체성 시작**. 작물 고유의 형태와 색상이 나타남 | 녹색 + 작물 고유색 시작 | 0.3~0.5m | 줄기 굵어짐, 잎 4~6장, 열매 미성숙 형태 |
| Harvestable | **완성과 보상**. 작물이 가장 화려한 상태. 수확 유도 | 작물 고유색 완전 발현 | 작물별 최종 높이 | 열매/뿌리 완성, 미세 반짝임 파티클 |

**전환 연출**:
- 단계 전환 시 **팝(Pop) 애니메이션**: 0.3초간 스케일 1.0 -> 1.15 -> 1.0 (바운스)
- Harvestable 진입 시 추가로 작은 빛 파티클 1회 발생
- 전환은 하루 시작(06:00) 배치 처리 후 일괄 발생 (-> see `docs/systems/time-season.md` 섹션 1.6)

### 6.2 품질 등급 시각 구분

품질 등급 정의: (-> see `docs/systems/crop-growth.md` 품질 공식, canonical)

수확물의 품질에 따라 시각적 구분을 제공한다. **Harvestable 상태의 밭 위 작물**과 **인벤토리 아이콘** 양쪽에 적용한다.

#### 밭 위 작물 (Harvestable 상태)

| 품질 | 밭 위 시각 효과 | 설명 |
|------|----------------|------|
| Normal | 효과 없음 | 기본 상태 |
| Good | 미세한 반짝임 파티클 (흰색, 3초 간격) | 살짝 윤기 있는 느낌 |
| Excellent | 은빛 반짝임 파티클 (`#C0C0C0`, 2초 간격) + 작물 크기 x1.05 | 약간 더 크고 빛나는 느낌 |
| Master | 금빛 반짝임 파티클 (`#FFD700`, 1.5초 간격) + 작물 크기 x1.1 + 미세한 글로우 | 확실히 특별해 보이는 느낌 |

[OPEN] Master 등급의 글로우 효과가 성능에 미치는 영향은 구현 단계에서 프로파일링 필요. 동시에 다수의 Master 작물이 있을 경우 파티클 수 제한 로직 검토.

#### 인벤토리 아이콘

| 품질 | 아이콘 표시 | 색상 |
|------|------------|------|
| Normal | 표시 없음 | - |
| Good | 아이콘 좌하단에 별 1개 | `#C0C0C0` (은색) |
| Excellent | 아이콘 좌하단에 별 2개 | `#FFD700` (금색) |
| Master | 아이콘 좌하단에 별 3개 + 아이콘 테두리 글로우 | `#9B59B6` (보라색) |

## 7. 비주얼 파라미터 수치 테이블

아키텍처 문서(`visual-architecture.md`)의 코드에서 참조하는 파라미터 기본값을 정의한다. 이 수치는 Unity Inspector에서 조정 가능한 튜닝 파라미터이며, 프로토타입 단계 검증 후 최종 확정한다.

### 7.1 조명 전환 파라미터

| 파라미터 | 기본값 | 설명 |
|----------|--------|------|
| `phaseTransitionDuration` | 16초 | 시간대(DayPhase) 전환 시 조명 보간 소요 시간 (게임 내 30분 기준 실시간 약 16초) |
| `seasonTransitionDuration` | 3초 | 계절 전환 시 Volume Profile Weight 보간 소요 시간 |

### 7.2 날씨 이펙트 파라미터

| 파라미터 | 기본값 | 설명 |
|----------|--------|------|
| `rainLightDimFactor` | 0.7 | 비(Rain) 날씨 시 Directional Light 강도 배율 (-30%) |
| `stormLightDimFactor` | 0.6 | 폭우(HeavyRain/Storm) 날씨 시 강도 배율 (-40%) |
| `snowLightDimFactor` | 0.8 | 눈(Snow) 날씨 시 강도 배율 (-20%) |
| `stormFogDensityAdd` | 0.02 | 폭우 시 추가 Fog Density |
| `blizzardFogDensityAdd` | 0.05 | 눈보라(Blizzard) 시 추가 Fog Density |
| `weatherTransitionDuration` | 2초 | 날씨 파티클 페이드 인/아웃 소요 시간 |

### 7.3 작물 비주얼 파라미터

| 파라미터 | 기본값 | 설명 |
|----------|--------|------|
| `harvestableSwaySpeed` | 1.2 | Harvestable 단계 작물 흔들림 속도 (초당 사이클) |

## Open Questions

1. **[OPEN]** 텍스처리스(Textureless) 방식과 최소 텍스처 방식 중 최종 선택은 Unity 프로토타입 시각 테스트 후 결정. 텍스처리스가 기본이나, 지형 타일의 자연스러운 느낌을 위해 노이즈 텍스처가 필요할 수 있음.

2. **[OPEN]** 플레이어 캐릭터 커스터마이징 범위 미정. 현재는 머리카락 스타일 3종만 정의했으나, 피부색/의상 색상 변경 등 추가 커스터마이징 여부는 스코프 검토 필요.

3. **[OPEN]** Master 등급 작물의 글로우 효과가 다수 동시 존재 시 성능 영향 확인 필요. 파티클 풀링 및 LOD 기반 파티클 제한 로직 검토.

4. **[OPEN]** 밤 시간 플레이어 주변 Point Light 반경(약 5타일)이 게임플레이에 적절한지는 플레이테스트 후 조정 필요.

5. **[OPEN]** 시설별 지붕 색상이 확정적이지 않음. NPC 악센트 색상과 겹치지 않도록 전체 색상 할당표를 Phase 2 초기에 정리해야 함.

## Risks

1. **[RISK]** 폴리곤 예산이 실제 구현 시 성능 병목을 일으킬 수 있음. 쿼터뷰에서 동시 렌더 오브젝트 수가 100개 이상일 때 프레임 드롭 가능성. LOD(Level of Detail) 시스템 도입 검토 필요.

2. **[RISK]** 로우폴리 텍스처리스 스타일에서 계절별 색상 변화를 머티리얼 교체로만 처리할 경우, 머티리얼 수가 급증하여 드로 콜(Draw Call) 증가. 머티리얼 프로퍼티 블록(MaterialPropertyBlock) 또는 셰이더 파라미터로 색상 변경하는 방식 권장.

3. **[RISK]** 색상 팔레트가 플레이테스트 없이 확정됨. 실제 화면에서의 조화와 가독성은 Unity 프로토타입에서 반드시 검증해야 함.

4. **[RISK]** 날씨 파티클(비, 눈, 폭풍)이 로우 스펙 PC에서 성능 부담이 될 수 있음. 파티클 품질 옵션 (낮음/중간/높음) 도입 권장.
