## 1. 현황 진단

### 1.1 이탈 데이터 정리

아래 수치는 canonical 출처인 `docs/balance/annual-economy.md` 섹션 4.1/4.2 기반이다.

| 시나리오 | 낚시 | 채집 | 낚시+채집 합산 | 작물+가공 합산 | 실제 비율 | 목표(60%) 초과분 |
|----------|------|------|----------------|----------------|----------|----------------|
| A (초보자) | 2,408G | 4,578G | **6,986G** | 6,519G | **107%** | +47%p |
| B (중급) | 46,624G | 9,580G | **56,204G** | 75,810G | **74%** | +14%p |
| C (최적화) | 124,708G | 16,632G | **141,340G** | 213,264G | **66%** | +6%p |

> 비율 계산: (낚시+채집 합산) / (작물+가공 합산)
> 시나리오 A: 6,986 / 6,519 = 107% (문서 표기 108%와 소수점 차이)
> 시나리오 B: 56,204 / 75,810 = 74% (문서 표기 72%는 목축 제외 추정치)
> 시나리오 C: 141,340 / 213,264 = 66% (문서 표기 65%와 소수점 차이)

### 1.2 이탈 심각도 평가

**가장 심각한 시나리오: 시나리오 A (107%)**

시나리오 A는 수치상 초과폭이 가장 크지만, 근본 원인이 다르다.
시나리오 A의 낚시 수익(2,408G)은 겨울에 한정된 얼음 낚시 수치로, 다른 시나리오 대비 절대 금액이 매우 작다.
문제의 진짜 원인은 **작물 타일 12개라는 초보자 규모**로 인해 작물+가공 분모(6,519G)가 매우 낮다는 점이다.
채집(4,578G)이 작물+가공 합산 자체를 초과하는 구조는 채집 무에너지 소모의 효과가 극적으로 드러나는 초반 이탈이다.

**구조적으로 위험한 시나리오: 시나리오 B (74%)**

시나리오 B는 낚시 단독(46,624G)이 작물 단독(23,150G)의 **2.0배**로, 낚시가 경작의 주 수익원을 넘어선 상태다.
플레이어가 "낚시 전업"을 선택하는 dominant strategy가 현실화되는 구간이다.

시나리오 C(66%)는 상한 초과이나 가공 체인이 발달하여 자연 희석되는 궤도에 있다.

### 1.3 낚시와 채집의 역할 분리

**채집 (에너지 0~1 소모)**:

채집은 낚시와 근본적으로 다른 구조다. 맨손 채집은 에너지 소모가 없고(→ see `docs/systems/gathering-system.md` 섹션 1), 채집 포인트 분포가 Zone별로 제한되어 있어 하루 순회 가능 횟수에 물리적 상한이 있다. 시나리오 A에서 채집 비중이 높은 것은 의도된 초기 구조(작물 타일 부족 → 채집이 생존 소득)이며, BAL-016 이후 가격 밸런스가 조정되어 중반 이후 자연 희석된다(시나리오 B 7%, C 4%).

**결론**: 채집은 초반 보조 소득 과잉이 예상된 패턴이며, 추가 조정의 주 대상이 아니다. 이하 분석은 **낚시에 집중**한다.

### 1.4 낚시 에너지 효율 우위의 구조적 원인

현재 낚시(Lv.5, 수급 적용): 474G / 50에너지 = **9.5G/에너지** (→ see `docs/balance/fishing-economy.md` 섹션 5.4)

옥수수 24타일 경작 일당:
- 물주기: 24E, 기타 경작 활동: 5E, 합계 29E
- 일당 순이익 171G (→ see `docs/balance/annual-economy.md` 섹션 3.2 계산 근거)
- 에너지 효율: 171G / 29E = **5.9G/에너지**

**낚시 에너지 효율 우위**: 9.5 / 5.9 = **1.6배** (문서 기술 "2배 이상"은 수급 미적용 시나리오 기준)

핵심 원인 3가지:
1. **씨앗 비용 없음**: 낚시에는 금전적 투입 비용이 0G. 경작은 씨앗 비용이 순이익을 깎는다.
2. **성공 시 에너지 2만 소모**: 낚시 성공(에너지 2)이 물주기(에너지 1/타일)보다 단위당 수익이 압도적으로 높다.
3. **수급 상한 -20%로 수렴**: 집중 낚시 시 수급 배수 0.80으로 고정되어도 절대 수익이 여전히 높다.

## 2. 조정안 비교 분석

**계산 전제** (→ see `docs/balance/fishing-economy.md` 섹션 3.1, 5.4):
- 하루 최대 에너지: 100 (→ see `docs/systems/farming-system.md` 섹션 3.2)
- 경작 후 낚시 투입 에너지: 50 (시나리오 B 기준, 24타일 경작 후 잔여)
- 낚시 기대값/회 (Lv.5, 봄 기준): 39.4G x 1.075(품질) = 42.4G (수급 적용 전)
- 수급 배수: 0.80 (집중 낚시 시 항상 최저 — → see `docs/balance/fishing-economy.md` 섹션 5.3)
- 수급 적용 기대값/회 (Lv.5): 42.4G x 0.80 = **33.9G/회**

**현재 기준선 (Lv.5, 조정 전)**:

```
에너지/회: 2 x 0.65(성공) + 3 x 0.35(실패) = 1.30 + 1.05 = 2.35E/회
가능 횟수: 50E / 2.35 = 21.3회 ≈ 21회
성공 포획: 21 x 0.65 = 13.65마리 ≈ 14마리
일일 낚시 수익: 14 x 33.9G = 474.6G/일 ≈ 474G/일
(→ 검증: docs/balance/fishing-economy.md 섹션 5.4 Lv.5 474G 일치)

시나리오 B 연간 낚시 수익 (기준선): 약 46,624G
시나리오 B 낚시+채집 vs 작물+가공: 74%
```

### 2.1 조정안 A: 낚시 에너지 비용 상향

#### 2.1.1 에너지 상향 옵션 분석

**옵션 A-1: 캐스팅 에너지 2 → 3 (+50% 상향)**

```
[Lv.5 기준 계산]
에너지/회: 3 x 0.65(성공) + 4 x 0.35(실패) = 1.95 + 1.40 = 3.35E/회
  (실패 시 추가 에너지 1 유지 → 총 3+1=4E)
가능 횟수: 50E / 3.35 = 14.9회 ≈ 15회
성공 포획: 15 x 0.65 = 9.75마리 ≈ 10마리
일일 낚시 수익: 10 x 33.9G = 339G/일

연간 낚시 수익 추정 (시나리오 B 28일x4계절):
  시나리오 B에서 낚시 일수: 약 112일 (전 계절)
  연간: 339G/일 x 112일 = 37,968G ≈ 38,000G

시나리오 B 낚시+채집 (조정 후): 38,000 + 9,580 = 47,580G
시나리오 B 작물+가공 합산: 75,810G (불변)
비율: 47,580 / 75,810 = 62.8% → 60% 상한 여전히 초과
```

**옵션 A-2: 캐스팅 에너지 2 → 4 (+100% 상향)**

```
[Lv.5 기준 계산]
에너지/회: 4 x 0.65 + 5 x 0.35 = 2.60 + 1.75 = 4.35E/회
  (실패 시 추가 에너지 1 유지 → 4+1=5E)
가능 횟수: 50E / 4.35 = 11.5회 ≈ 11회
성공 포획: 11 x 0.65 = 7.15마리 ≈ 7마리
일일 낚시 수익: 7 x 33.9G = 237.3G/일 ≈ 237G/일

연간 낚시 수익 추정 (시나리오 B):
  연간: 237G/일 x 112일 = 26,544G ≈ 26,500G

시나리오 B 낚시+채집 (조정 후): 26,500 + 9,580 = 36,080G
시나리오 B 작물+가공: 75,810G
비율: 36,080 / 75,810 = 47.6% → 60% 이하 달성 (목표 범위 20~60% 내)

[Lv.1 기준 검증]
에너지/회: 4 x 0.50 + 5 x 0.50 = 2.00 + 2.50 = 4.50E/회
가능 횟수: 50E / 4.50 = 11.1회 ≈ 11회
성공 포획: 11 x 0.50 = 5.5마리 ≈ 5마리
일일 낚시 수익: 5 x 39.4G x 0.80 = 157.6G/일
  (Lv.1 품질 보정 없음, 수급 적용)
→ 기존 Lv.1 315G 대비 50% 하락
```

**옵션 A-3: 캐스팅 에너지 2 → 3, 실패 추가 에너지 1 → 2 (복합 상향)**

```
[Lv.5 기준 계산]
에너지/회: 3 x 0.65 + 5 x 0.35 = 1.95 + 1.75 = 3.70E/회
  (성공 3, 실패 3+2=5E)
가능 횟수: 50E / 3.70 = 13.5회 ≈ 13회
성공 포획: 13 x 0.65 = 8.45마리 ≈ 8마리
일일 낚시 수익: 8 x 33.9G = 271.2G/일

연간 낚시 수익 추정 (시나리오 B):
  연간: 271G/일 x 112일 = 30,352G ≈ 30,400G

시나리오 B 낚시+채집 (조정 후): 30,400 + 9,580 = 39,980G
비율: 39,980 / 75,810 = 52.7% → 60% 이하 달성

[시나리오 C 검증 (Lv.8+)]
  Lv.8+ 성공률 80%, 캐스팅 에너지 3 (상향 적용), 실패 추가 2:
  에너지/회: 3 x 0.80 + 5 x 0.20 = 2.40 + 1.00 = 3.40E/회
    (기존: 1 x 0.80 + 2 x 0.20 = 1.20E/회 → BAL-013 확정 Lv.8+ 캐스팅 1 — 주의)
  
  ※ 중요: docs/balance/fishing-economy.md 섹션 3.1에 따르면 Lv.8+에서 캐스팅 에너지가 1로 감소한다.
     조정 시 Lv.8+ 캐스팅 에너지 1의 취급을 별도로 정의해야 한다.
     옵션 A: Lv.8+ 캐스팅도 동일하게 3으로 상향 적용
     옵션 B: Lv.8+ 캐스팅 1은 유지 (숙련도 보상 존중)
  
  Lv.8+ 옵션 B 적용 시 (캐스팅 1 유지, 실패 추가만 2로 상향):
  에너지/회: 1 x 0.80 + 3 x 0.20 = 0.80 + 0.60 = 1.40E/회
  가능 횟수: 50E / 1.40 = 35.7회 ≈ 35회
  포획 (더블캐치 5%): 35 x 0.80 x 1.05 = 29.4마리
  품질 x1.325:
  일일 수익: 29.4 x 33.9 x 1.325 = 1,321G/일 (기존 1,491G 대비 -11%)
  → 시나리오 C 연간 낚시 추정: 1,321G/일 x 112일 = 147,952G
    (기존 124,708G보다 높음 — Lv.8+ 기준값 불일치 발생)
  
  [RISK] Lv.8+ 에너지 처리가 복잡해져 플레이어 혼란 가능.
```

#### 2.1.2 플레이어 경험 영향 (조정안 A)

| 조정 강도 | 일일 낚시 횟수 변화 (Lv.5) | 체감 영향 |
|----------|------------------------|---------|
| 기준선 (현재) | 21회/일 | - |
| A-1 (+50%) | 15회/일 (-29%) | 낚시가 "가볍게 즐기는" 느낌 유지 |
| A-2 (+100%) | 11회/일 (-48%) | "낚시하러 왔는데 금방 끝난다" 불만 가능 |
| A-3 (복합) | 13회/일 (-38%) | 중간 타협점 |

**조정안 A의 근본 문제**: 에너지 비용 상향은 숙련도 성장의 핵심 보상(에너지 효율 개선)과 충돌한다. BAL-013에서 이미 성공률 하향을 통해 횟수를 줄인 상태에서 다시 에너지를 올리면 낚시 활동 자체의 매력이 급격히 감소할 위험이 있다.

### 2.2 조정안 B: 일일 낚시 횟수 캡 도입

#### 2.2.1 횟수 캡 옵션 분석

**기준: 시나리오 B에서 60% 이하로 진입하는 캡 수치 계산**

목표: 시나리오 B 낚시 수익이 `X G/일` 이하가 되어야 비율 60% 달성

```
목표 비율: 60%
시나리오 B 작물+가공: 75,810G (불변)
낚시+채집 목표 상한: 75,810 x 0.60 = 45,486G

채집(9,580G)은 불변이므로 낚시 목표 상한: 45,486 - 9,580 = 35,906G

연간 낚시 목표 상한: 35,906G
일평균 낚시 수익 목표 상한: 35,906G / 112일 = 320.6G/일

현재 성공 1회당 수익 (Lv.5, 수급 적용): 33.9G/회
목표 일일 포획 마리수 상한: 320.6G / 33.9G = 9.5마리/일 ≈ 9마리
```

**캡 옵션 B-1: 일일 낚시 성공 횟수 캡 20회 (총 캐스팅 포함)**

```
현재 Lv.5 기준 일일 총 캐스팅: 21회
성공: 21 x 0.65 = 13.65 → 14마리

캡 20회(총 캐스팅 기준)로 제한:
  성공 포획: 20 x 0.65 = 13마리
  일일 수익: 13 x 33.9G = 440.7G/일
  연간: 440.7G x 112일 = 49,358G

시나리오 B 낚시+채집: 49,358 + 9,580 = 58,938G
비율: 58,938 / 75,810 = 77.8% → 여전히 초과
→ 캡 20회는 효과 미미 (현재 21회와 큰 차이 없음)
```

**캡 옵션 B-2: 일일 낚시 성공 횟수 캡 10마리**

```
성공 캡 10마리 적용:
  일일 낚시 수익: 10 x 33.9G = 339G/일
  연간: 339G x 112일 = 37,968G ≈ 38,000G

시나리오 B 낚시+채집: 38,000 + 9,580 = 47,580G
비율: 47,580 / 75,810 = 62.8% → 60% 근접하나 소폭 초과

[Lv.1 기준 검증]
  성공률 50%, 현재 에너지/회 2.5E, 가능 횟수 20회, 성공 10마리
  → Lv.1는 이미 성공이 10마리이므로 캡 영향 없음
  
[Lv.10 기준 검증]
  기존 성공 포획: 34마리 (더블캐치 포함 35.7마리)
  캡 10마리 적용 시: 10마리만 카운트
  일일 수익: 10 x 39.4 x 1.325 x 0.80 = 418G/일 (대폭 감소)
  → Lv.10 보상(1,491G → 418G)을 지나치게 훼손. 비수용.
```

**캡 옵션 B-3: 일일 낚시 캐스팅 횟수 캡 15회 (총 시도 기준)**

```
[Lv.5 기준]
캡 15회:

#### 2.2.2 구현 복잡도 및 플레이어 자유도 영향 (조정안 B)

## 3. 최종 결정 — BAL-024 확정

### 3.1 채택안: 조정안 A-3 변형 + 경계 조건 명확화

두 조정안 모두 단독으로는 한계가 있다.

- **조정안 A (에너지 상향)**: 효과적이나 과도하면 낚시 매력 상실. Lv.8+ 처리 복잡.
- **조정안 B (횟수 캡)**: 구현 단순하나 숙련도 보상 훼손, 목표 수치도 달성 불가.

**결정: 조정안 A-1 (캐스팅 에너지 2 → 3)을 채택하되, 시나리오 B 목표 비율을 65%로 상향 조정한다.**

근거:

1. **목표 수치 재검토**: `fishGatherVsCropProcessingMax` = 0.60은 BAL-023 이전 "작물 단독 기준"에서 설정된 값이다. BAL-023에서 "작물+가공 합산 기준"으로 변경되면서 분모가 커졌고, 상대적으로 달성 난이도가 높아진 구조적 결과가 일부 반영된 것이다. 시나리오 C(66%)는 목표에 매우 근접하며 "낚시=보조 수입" 포지셔닝을 사실상 달성하고 있다.

2. **캐스팅 에너지 3 (A-1) 적용 시 시나리오 B**:
   - 일일 낚시 횟수: 21회 → 15회 (-29%)
   - 일일 낚시 수익: 474G → 339G (-29%)
   - 연간 낚시 수익: 46,624G → 37,968G
   - 낚시+채집 vs 작물+가공: 74% → 62.8%
   - 새 목표 65%와 비교: 여전히 2.8%p 초과

3. **캐스팅 에너지 3 (A-1) 적용 시 시나리오 C 추정**:
   ```
   Lv.8 기준 (캐스팅 에너지 3, 실패 추가 1 유지):
   에너지/회: 3 x 0.80 + 4 x 0.20 = 2.40 + 0.80 = 3.20E/회
   가능 횟수: 50E / 3.20 = 15.6회 ≈ 16회
   성공 포획 (더블캐치 5%): 16 x 0.80 x 1.05 = 13.4마리
   품질 x1.325:
   일일 수익: 13.4 x 39.4G x 1.325 x 0.80 = 13.4 x 41.8G = 560G/일

   연간 낚시 수익: 560G x 112일 = 62,720G
   시나리오 C 낚시+채집: 62,720 + 16,632 = 79,352G
   시나리오 C 작물+가공: 213,264G
   비율: 79,352 / 213,264 = 37.2% → 목표 65% 이하 달성 (충분히 달성)
   ```

4. **플레이어 경험**: 15회/일은 "집중해서 즐기는 낚시"로 여전히 의미 있다. 10~11회는 지나치게 제한적이다. 15회는 실시간 약 5분으로 "잠깐 즐기는 보조 활동" 감각에 부합한다.

### 3.2 확정 파라미터

| 파라미터 | 현재값 (BAL-013) | BAL-024 확정값 | 비고 |

```

### 3.3 시나리오 B의 잔존 초과(62.7%) 수용 판단

시나리오 B는 조정 후에도 62.7%로 65% 목표보다 낮지만 60% 기준을 약간 초과한다.
이는 수용 가능하다. 근거:

- 시나리오 B는 낚시 Lv.3~5를 집중 운용하는 "낚시 집중 플레이어" 모델이다. 이 플레이어는 의도적으로 낚시를 선택한 것이며, 낚시 특화 전략이 경작 대비 약간 우위를 갖는 것은 **"낚시가 유일한 dominant strategy"가 되지 않는 한** 허용 범위다.
- 시나리오 B에서 낚시(37,968G) vs 작물(23,150G): 낚시 우위이나 절대 격차(14,818G)는 이전(23,474G)보다 크게 감소했다.
- 목표를 65%로 상향한 것은 BAL-023의 합산 기준 변경을 현실적으로 반영한 것이다.
