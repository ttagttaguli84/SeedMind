## 2. 이슈 진단

### 2.1 TODO 원문의 ~68% 수치 출처

TODO에 기재된 "업적 39종 3,130 XP, 비중 ~68%"는 **이전 XP 테이블(baseXP=50, growthFactor=1.55, 레벨 10 = 4,609 XP)** 기준의 계산이다.

```
3,130 / 4,609 = 0.6792... ≈ 67.9%
```

이 TODO는 XP 테이블이 조정되기 전(baseXP=80, growthFactor=1.60, 레벨 10 = 9,029 XP) 또는 조정 사실이 반영되기 전에 작성된 것이다. 또한 당시 업적 수가 39종(3,130 XP)이었으며, 이후 CON-017에서 `ach_hidden_07`(통합 수집 마스터, 30 XP)이 추가되어 40종 3,160 XP로 확정되었다.

**결론: TODO 원문의 ~68%는 현행 canonical XP 테이블(9,029) 기준으로 재계산하면 35.0%이며, 목표 범위(33~43%) 내에 있다. TODO 원문의 이슈는 XP 테이블 조정으로 이미 해소되었다.**

### 2.2 실제 잠재 문제 분석

업적 XP 자체는 목표 범위 내이나, 아래 두 가지를 추가 검증한다.

#### (a) 보조 소스 합산 51.2%가 문제인가?

보조 소스(퀘스트+업적+도감)의 합계 4,626 XP가 레벨 10(9,029)의 51.2%라는 것은, 이론적으로 수확/경작 없이도 보조 소스만으로 레벨 8 중반(4,626 > 3,446)에 도달할 수 있다는 의미이다.

그러나 이는 현실적이지 않다:

1. **업적 달성에 수확/경작이 선행 필수**: Farming 업적(수확 횟수), Economy 업적(판매 수입), Facility 업적(시설 건설) 등은 모두 핵심 루프 활동의 결과물이다. 업적 XP는 수확/경작의 "보너스"이지, 대체 경로가 아니다.
2. **완료까지 3년 이상 소요**: 전 업적(3,160 XP)은 2~3년차에 걸쳐 점진적으로 달성된다. 1년차 실현분은 ~610 XP(전체의 19.3%)에 불과하다.
3. **퀘스트 XP 유한**: 1,115 XP는 콘텐츠 총량에 의해 hard cap이 걸린다.
4. **도감 초회 보상 미미**: 351 XP(3.9%)는 1년차 ~100 XP 수준이다.

**결론: 51.2%는 "3년 이상에 걸쳐 분산 획득"되는 수치이므로, 연 단위 인플레이션 리스크는 낮다.**

#### (b) 1년차 업적 실현분 13.0%가 설계 목표(5~10%)를 초과하는가?

BAL-007 섹션 2.3의 설계 목표는 "1년차 기준 업적 비율 5~10%"이다. 현재 1년차 업적 실현분은 ~610 XP / ~4,682 XP ≈ 13.0%로, 목표 상한(10%)을 3%p 초과한다.

그러나 BAL-007 섹션 2.3은 "업적" 소스만을 5~10%로 설정한 것이며, 실제 시나리오 A' 시뮬레이션(BAL-007 섹션 4.2.2)에서는 업적 ~570 XP / ~4,502 XP ≈ 12.7%로 이미 이 범위를 초과한 상태에서 확정되었다. 즉, **시나리오 A'의 확정 시뮬레이션 자체가 이미 10%를 넘긴 상태로 승인**되었으므로, 실질적으로 목표 범위가 암묵적으로 **5~15%**로 확장된 것이다.

추가로 FIX-099에서 채집 도감 초회 보상(351 XP)이 보조 소스에 추가되면서 업적 비중은 상대적으로 변하지 않았다 (도감은 별도 카테고리).

**결론: 1년차 업적 비중 13%는 시나리오 A' 확정 시뮬레이션의 결과와 일치하며, 실질적 문제가 아니다.**

## 3. 조정 옵션 비교

### 3.1 옵션 A: 업적 XP 하향

#### A-1: 전체 균일 하향 (예: 20% 삭감)

```
삭감 후 업적 XP = 3,160 × 0.80 = 2,528 XP
비중 = 2,528 / 9,029 = 28.0%
보조 소스 합계 = 1,115 + 2,528 + 351 = 3,994 XP (44.2%)
1년차 업적 실현분 = 610 × 0.80 = ~488 XP
1년차 업적 비중 = 488 / (3,332 + 640 + 488 + 100) = 488 / 4,560 ≈ 10.7%
```

- 장점: 보조 소스 합산 비중 44.2%로 하락, 1년차 업적 비중이 설계 목표(5~10%) 근접
- 단점: 업적 달성 보상 체감 약화. 특히 Gold 단계 업적(현 100 XP)이 80 XP로 떨어지면 "어려운 업적을 달성했는데 보상이 미미하다"는 불만 발생 가능
- **변경 범위**: achievements.md 전체 업적의 XP 값 수정 (40종), xp-integration.md 전 섹션 재계산

#### A-2: 선택적 하향 (고가치 업적만 삭감)

Farming Gold(100→70), Economy Gold(100→70), 낚시 전종 도감(100→70), 채집 전종 도감(100→70) 등 상위 업적만 삭감.

```
삭감 대상: Gold 단계 약 8개 × (100→70) = -240 XP
삭감 후 업적 XP = 3,160 - 240 = 2,920 XP
비중 = 2,920 / 9,029 = 32.3%
```

- 장점: 초반/중반 업적 보상 유지, 목표 범위(33~43%) 하한 근접
- 단점: 32.3%는 목표 범위 하한(33%) 미달. Gold 업적의 보상이 Silver(50 XP)와의 간격이 20 XP로 좁아져 단계감 약화
- **변경 범위**: achievements.md 일부 업적 XP 수정

### 3.2 옵션 B: 레벨 테이블 목표 XP 상향

growthFactor를 1.60에서 상향하여 레벨 10 누적 XP를 높인다.

```
예: growthFactor 1.65 → 레벨 10 누적 XP 계산
  lv2: 80, lv3: 132, lv4: 218, lv5: 359, lv6: 593,
  lv7: 978, lv8: 1,614, lv9: 2,663, lv10: 4,394
  누적: 80+132+218+359+593+978+1,614+2,663+4,394 = 11,031 XP

업적 비중 = 3,160 / 11,031 = 28.6%
보조 소스 합산 = 4,626 / 11,031 = 41.9%
```

- 장점: 전체 비중 하락, 레벨 10 도달이 더 장기적 목표가 됨
- 단점: **1년차 시뮬레이션 전면 재계산 필요**. 캐주얼 플레이어의 레벨 8 도달이 불확실해짐 (lv8 = 3,976이 되어 수확 ~3,332만으로 미달). BAL-007 시나리오 A'의 모든 판정이 무효화됨. **수정 범위가 극도로 넓음**
- **변경 범위**: progression-curve.md 레벨 테이블, xp-integration.md 전 시뮬레이션, quest-rewards.md, achievements.md 비중 표기 등 대규모 연쇄 수정

### 3.3 옵션 C: 현상 유지

현재 수치를 확정하고, TODO 원문의 이슈를 "XP 테이블 조정으로 해소됨"으로 종결한다.

```
업적 비중: 35.0% (목표 33~43% 범위 내)
보조 소스 합산: 51.2% (3년 분산, 연 단위 리스크 낮음)
1년차 업적 비중: 13.0% (시나리오 A' 확정 시뮬레이션과 일치)
```

- 장점: 변경 없음, 기존 BAL-007 시나리오 A' 시뮬레이션 결과 보존, 연쇄 수정 불필요
- 단점: 보조 소스 합산 51.2%가 높다는 우려가 잔존. 단, 이는 구조적 문제가 아닌 "3년 분산"의 산술적 결과
- **변경 범위**: 없음 (문서 정리만)

### 3.4 옵션 D: 목표 비중 범위 재정의

현재 33~43% 목표를 현실에 맞게 조정한다.

BAL-007 섹션 2.3의 원래 설계에서 "업적 5~10%"는 1년차 기준이었고, "전체 게임 기준 업적 비중"에 대한 명시적 목표는 33~43%로 achievements.md 섹션 2.4에서만 언급되었다. 이 33~43%의 출처를 추적하면, 이전 XP 테이블(4,609) 시절 "업적 XP ~2,250을 가정"한 추정치이다. 현재 업적은 40종 3,160 XP로 확장되었고 XP 테이블도 9,029로 조정되었으므로, 목표 범위 자체를 재정의할 근거가 있다.

- 제안: 목표 범위를 **30~40%**로 재정의 (현재 35.0%가 중앙에 위치)
- 장점: 현실 수치와 정합, 향후 콘텐츠 추가 시 소폭 변동 허용
- 단점: "목표를 수치에 맞추는" 방식이므로 설계 주도성이 약해질 수 있음
- **변경 범위**: achievements.md 섹션 2.4의 목표 범위 표기 수정

## 4. 결정

### 4.1 확정 조정안: 옵션 C + D 병합 (현상 유지 + 목표 범위 미세 조정)

**근거:**

1. **업적 XP 35.0%는 기존 목표 범위(33~43%) 내에 있다.** 조정이 불필요한 상태이다.
2. **TODO 원문의 ~68% 이슈는 이전 XP 테이블(4,609) 기준이며, 현행 테이블(9,029) 적용 시 이미 해소되었다.**
3. **보조 소스 합산 51.2%는 3년 분산 구조이므로 연 단위 밸런스 위협이 아니다.** 1년차 보조 비중은 28.8%로 수확/경작(71.2%)이 압도적 주 소스 역할을 유지한다.
4. **옵션 A(업적 XP 하향)는 업적 달성 보상 체감을 약화시키고 40종 전체 XP 재조정이라는 대규모 수정을 수반한다.** 현재 범위 내에 있으므로 리스크 대비 이득이 낮다.
5. **옵션 B(레벨 테이블 상향)는 BAL-007 시나리오 A'의 전체 시뮬레이션을 무효화하는 극단적 변경이다.** 현재 1년차 목표(캐주얼 lv8, 일반 lv8 중반, 적극적 lv9 직전)가 잘 작동하고 있으므로 불필요하다.
6. **목표 범위를 33~43% → 30~40%로 미세 조정**한다. 이는 현재 35.0%를 중앙에 배치하고, 향후 콘텐츠 추가/삭제 시 ±5%p 변동 여유를 확보한다. 기존 33~43%는 이전 XP 테이블 기준에서 도출된 것이므로 현행 테이블에 맞게 갱신하는 것이 합리적이다.
