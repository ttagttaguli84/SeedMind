## 1. Context

이 문서는 SeedMind의 세 가지 XP 소스(수확/경작, 퀘스트, 업적)를 통합 관점에서 분석하고, 단일 XP 예산 체계를 확립한다. 각 소스가 독립 설계되어 합산 시 필요 XP의 347%에 달하는 인플레이션 문제(-> see `docs/balance/quest-rewards.md` 섹션 3.4)를 해결하기 위한 재설계 문서이다.

### 1.1 본 문서가 canonical인 데이터

- XP 소스별 비율 목표 (연간 예산 배분)
- 통합 시나리오 시뮬레이션 결과 및 권장안
- 1년차 레벨 목표 확정 값
- XP 소스별 연간 상한(hard cap / soft target)
- progression-curve.md 수정 필요 사항 목록

### 1.2 본 문서가 canonical이 아닌 데이터 (참조만)

| 데이터 종류 | 참조처 |
|------------|--------|
| XP 테이블 (baseXP, growthFactor), 레벨별 누적 XP | `docs/balance/progression-curve.md` 섹션 1.3.2 (원본) 및 섹션 2.4.1 (조정) |
| 수확/경작 XP 시뮬레이션 (1년차 ~3,332 XP) | `docs/balance/progression-curve.md` 섹션 2.4.2 |

### 1.3 선행 분석의 불일치 식별

[RISK] **BAL-006에서 두 가지 XP 테이블을 혼용하고 있다.** BAL-006 섹션 3.1은 원본 테이블(baseXP=50, growthFactor=1.55, 레벨 10 = 4,609 XP)을 기준으로 퀘스트 XP 비율을 분석하나, 섹션 7.2의 수확/경작 XP(~3,332 XP)는 조정된 테이블(baseXP=80, growthFactor=1.60, 레벨 10 = 9,029 XP)의 시뮬레이션 결과이다. 이 혼용으로 인해 BAL-006의 "레벨 9 후반 도달" 판정이 부정확하다.

## 2. 설계 목표 재정의

### 2.1 1년차 레벨 목표

현행 설계(-> see `docs/balance/progression-curve.md` 섹션 2.4.3): 수확/경작 XP만으로 1년차 레벨 7 직전(~3,332 XP, 레벨 7 = 2,104 XP 초과, 레벨 8 = 3,446 XP 미달).

퀘스트와 업적 XP를 포함한 통합 레벨 목표:

| 플레이어 유형 | 1년차 목표 레벨 | 근거 |
|-------------|---------------|------|
| 캐주얼 (퀘스트 미수행, 업적 자연 달성만) | 7 | 수확/경작만으로 레벨 7 도달은 이미 검증됨 |
| 일반 (퀘스트 50% 수행, 업적 중간 진행) | **8** | 레벨 8 = 3,446 XP. 수확 3,332 + 보조 소스 ~500~800으로 도달 가능해야 함 |
| 적극적 (퀘스트 대부분 수행, 업적 적극 추적) | **9 초반** | 레벨 9 = 5,593 XP. 수확 3,332 + 보조 소스 ~1,500~2,000으로 근접 |

**확정 목표**: 1년차 종료 시 **일반 플레이어 레벨 8, 적극적 플레이어 레벨 9 초반**. 레벨 10(9,029 XP)은 2년차 중후반 달성 콘텐츠로 유지한다.

이전 설계의 "1년차 레벨 7~8"(-> see `docs/balance/progression-curve.md` 섹션 2.4.3)과 비교하면, 퀘스트/업적 보조 소스를 포함하여 **일반 플레이어 레벨 8**로 상향 조정한다. 이는 퀘스트를 수행하는 플레이어에게 의미 있는 진행 보상을 주면서도, 퀘스트를 안 하는 캐주얼 플레이어도 레벨 7에서 핵심 콘텐츠(수박, 가공소)를 모두 해금할 수 있게 한다.

### 2.2 XP 소스별 역할 정의

| XP 소스 | 역할 | 성격 |
|---------|------|------|
| **수확/경작** | Primary (주 소스) | 게임의 핵심 루프. 매일 반복하는 행위에서 꾸준히 획득. 레벨업의 기반 |
| **퀘스트** | Secondary (보조 가속) | 특정 목표 달성 시 보너스. 퀘스트를 수행하면 레벨업이 약간 빨라지되, 안 해도 치명적이지 않음 |
| **업적** | Tertiary (장기 보너스) | 마일스톤 달성 시 일회성 보상. 대부분 자연스럽게 달성되며, 의도적 추적은 소수 |

### 3.3 시나리오 B: 퀘스트 XP 삭감 + XP 테이블 소폭 하향

퀘스트 XP는 제안 A를 적용하되, growthFactor를 1.55로 되돌려 후반 레벨 문턱을 낮춘다. baseXP=80은 유지하여 초반 가속 억제는 보존.

**수정된 XP 테이블** (baseXP=80, growthFactor=1.55):

| 레벨 | 필요 XP | 누적 XP |
|------|---------|---------|
| 1 | - | 0 |
| 2 | 80 | 80 |
| 3 | 124 | 204 |
| 4 | 192 | 396 |
| 5 | 298 | 694 |
| 6 | 462 | 1,156 |
| 7 | 716 | 1,872 |
| 8 | 1,110 | 2,982 |
| 9 | 1,720 | 4,702 |
| 10 | 2,666 | 7,368 |

검산: `필요_XP(n) = round(80 * 1.55^(n-1))`
- 레벨 2: round(80 * 1) = 80
- 레벨 3: round(80 * 1.55) = 124
- 레벨 4: round(80 * 2.4025) = 192
- 레벨 5: round(80 * 3.724) = 298
- 레벨 6: round(80 * 5.772) = 462
- 레벨 7: round(80 * 8.946) = 716
- 레벨 8: round(80 * 13.867) = 1,110 (오차 수정: round = 1,109, 반올림 차이 허용)
- 레벨 9: round(80 * 21.494) = 1,720
- 레벨 10: round(80 * 33.316) = 2,665 (반올림 차이 허용)

**1년차 통합 시뮬레이션**:

| XP 소스 | 일반 플레이어 | 적극적 플레이어 |
|---------|-------------|---------------|
| 수확/경작 | ~3,332 XP | ~3,332 XP |
| 퀘스트 | ~400 XP | ~550 XP |
| 업적 | ~500 XP | ~660 XP |
| **합계** | **~4,232 XP** | **~4,542 XP** |

**레벨 도달 판정**:

| 플레이어 | 누적 XP | 도달 레벨 | 비고 |
|---------|---------|----------|------|
| 캐주얼 (수확만) | ~3,332 XP | 8 (2,982 초과, 4,702 미달) | 수확만으로 레벨 8 도달 -- **과도** |
| 일반 | ~4,232 XP | 8 (4,702 미달) | 레벨 9 근접 |

## 7. 보조 소스 XP 비중 관리 원칙 (DES-021)

> 작성 근거: DES-021 | BAL-019 Open Question RESOLVED

### 7.1 현재 상태 요약

(-> see 섹션 5.2) 현재 보조 소스(퀘스트+업적+도감) 합산은 총 ~4,626 XP로, 레벨 10 목표치(9,029 XP) 대비 **51.2%**이다. 이 수치는 1~3년차에 걸쳐 점진적으로 분산 획득되므로 연 단위 인플레이션 리스크는 낮다. 1년차 보조 소스 실현 비중은 ~28.8%로, 수확/경작(~71.2%)이 주 소스 역할을 유지한다.

### 7.2 비중 상한 기준 (Soft Cap)

보조 소스 합산 비중의 **soft cap은 60%**로 정한다. 이는 레벨 10 누적 XP(9,029 기준) 대비 5,417 XP에 해당한다.

- **현재 여유 마진**: 60% 상한(5,417 XP) - 현재 보조 소스 합산(4,626 XP) = **791 XP 여유**
- **경고 구간 (Yellow Zone)**: 55~60% (4,966~5,417 XP 범위). 이 구간에 진입하면 주의 상태로 모니터링을 강화하며, 신규 보조 소스 추가 시 반드시 재검토 절차를 거쳐야 한다.
- **재검토 트리거 (Red Zone)**: 보조 소스 합산이 60%(5,417 XP)를 초과할 경우 즉시 재검토를 트리거로 삼는다.

### 7.3 모니터링 체크포인트

보조 소스 합산 비중은 다음 시점에 반드시 재계산해야 한다.

1. **신규 보조 XP 소스 추가 시**: 신규 퀘스트 카테고리, 업적 종목, 도감 항목을 추가하는 DES/CON 작업이 완료되는 시점마다 섹션 5.2의 수치를 갱신하고 비중을 재계산한다.
2. **DES/CON 작업 완료 시 (보조 XP 관련)**: 퀘스트, 업적, 수집 시스템에 영향을 주는 모든 DES-* / CON-* 작업 완료 후 섹션 5.2를 갱신한다.
3. **XP 테이블 변경 시**: progression-curve.md의 레벨 테이블(baseXP, growthFactor)이 변경될 경우 분모가 바뀌므로 모든 비중 수치를 재계산해야 한다.

### 7.4 60% 초과 시 우선 조치 순서

보조 소스 합산이 60%를 초과하는 경우 아래 순서로 조치를 검토한다. 수정 범위가 좁은 옵션부터 적용하는 것을 원칙으로 삼는다.

**1순위: 신규 추가 보조 소스 XP 삭감**

초과를 유발한 신규 소스의 XP를 직접 삭감한다. 기존 확정 값을 건드리지 않으므로 수정 범위가 가장 좁다. 신규 콘텐츠 설계 단계에서 미리 여유 마진을 확인하고 예산 내로 설계하는 것이 최선이다.

**2순위: 기존 업적 XP 선택적 하향**

초과 폭이 크거나 1순위로 해소가 불가한 경우, 기존 업적 중 Gold 단계 등 고가치 업적의 XP를 선택적으로 하향한다. 업적 달성 보상 체감을 최대한 유지하기 위해 Silver 단계(50 XP) 이하는 건드리지 않는 것을 원칙으로 삼는다. 수정 범위: achievements.md 일부 업적 XP 수정.

**3순위: 레벨 테이블 목표 XP 상향 (최후 수단)**

1·2순위로 해소 불가한 경우에만 적용한다. growthFactor 상향으로 레벨 10 누적 XP를 높이면 분모가 커져 비중이 자동으로 낮아지나, BAL-007 시나리오 A' 시뮬레이션 전체가 무효화되고 레벨 진행 속도 전반이 영향을 받는다. 수정 범위가 가장 넓으므로 최후 수단으로만 사용해야 한다. 수정 범위: progression-curve.md 레벨 테이블 + xp-integration.md 전 시뮬레이션 + 연관 문서 대규모 연쇄 수정.

### 7.5 콘텐츠 확장 시 XP 예산 원칙

신규 보조 XP 소스(퀘스트/업적/도감)를 추가하는 모든 DES/CON 작업에서 다음 절차를 따라야 한다.

1. **작업 착수 전**: 섹션 5.2의 현재 보조 소스 합산 XP와 레벨 10 대비 비중을 확인한다.
2. **추가 예산 결정**: 추가하려는 신규 보조 XP 소스의 총량이 60% 상한(현재 기준 791 XP 이내) 안에 들어오는지 검증한다. Yellow Zone(55~60%)에 이미 진입한 상태라면 XP 규모를 줄이거나 재검토 절차를 선행한다.
3. **작업 완료 후**: 섹션 5.2를 갱신하고, 비중이 Yellow Zone(55%)를 초과했는지 확인한다. 초과 시 해당 DES/CON 작업 문서에 [RISK] 태그를 추가한다.
4. **60% 초과 시**: 섹션 7.4의 조치 순서에 따라 즉시 재검토를 진행한다.

**[RESOLVED: DES-021]** BAL-019 Open Question "보조 소스 합산 비중(51.2%)의 장기 모니터링 기준" — 본 섹션(7.2~7.5)으로 설계 원칙 확정. Soft cap 60%, Yellow Zone 55~60%, 조치 우선순위 3단계.

- [OPEN] **일일 목표 레벨 스케일링 제거 여부**: 제안 A에서 일일 목표의 레벨 스케일링 제거를 권장했는데, 이를 유지하되 상한만 낮추는(x1.0~x1.4) 방안이 플레이어 체감상 나을 수 있다. quest-system.md에서 결정 필요.

## Risks

- [RISK] **BAL-006 기준 XP 불일치**: BAL-006가 원본 테이블(4,609)과 조정 테이블(9,029)을 혼용하고 있어, BAL-006의 일부 판정이 부정확하다. BAL-006 수정이 선행되어야 본 문서의 제안이 정확히 구현된다.
- [RISK — BAL-019 검증 완료, 현상 유지 확정] **업적 XP 장기 누적**: 전 업적 완료 시 3,160 XP(낚시 4종 + 채집 5종 + 통합 수집 마스터 1종 추가 후, CON-017)는 레벨 10(9,029)의 35.0%. 목표 범위(30~40%, BAL-019 조정) 내이며, 3년 이상에 걸쳐 분산되므로 연 단위 인플레이션은 제한적. 단계형 업적의 Gold 단계(100 XP) 집중 해금 시 모니터링 유지. (-> see `docs/balance/bal-019-xp-balance.md`)
- [RISK] **캐주얼 플레이어의 레벨 8 진입**: 수확/경작만으로 ~3,532 XP에 도달하면 레벨 8(3,446)에 걸치는데, 이 수치는 "일반 플레이어" 시뮬레이션 기준이다. 비효율적 플레이어는 ~2,500~3,000 XP에 머물 수 있으며, 이 경우 레벨 7에 머문다. 이는 오히려 의도에 부합한다.
- [RISK] **퀘스트 XP가 ~1,010 XP로 BAL-007 확정값(900)을 초과한다**. 낚시 퀘스트 추가(~110 XP)로 전체 XP의 11.2%로 소폭 상승하나 허용 범위 내이다. 향후 DLC/시즌 이벤트 등 추가 콘텐츠에서 퀘스트가 더 늘어나면 이 예산이 다시 초과될 수 있으므로, 확장 콘텐츠의 XP 예산은 별도 관리해야 한다.
