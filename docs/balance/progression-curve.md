# 게임 진행 곡선 밸런스 시트 (Progression Curve Balance Sheet)

> 작성: Claude Code (Opus) | 2026-04-06  
> 문서 ID: BAL-002

---

## Context

이 문서는 SeedMind의 게임 진행 곡선을 정량적으로 설계하고 분석한다. 경험치(XP) 시스템, 레벨별 해금 타임라인, 계절별 자금 곡선, 핵심 마일스톤을 수치로 정의하여 플레이어가 체감하는 진행 속도와 성취감의 리듬을 검증한다.

**본 문서가 canonical인 데이터**: XP 획득량, 레벨별 필요 XP 테이블, XP 소스별 배분, 레벨 도달 예상 시점, 진행 마일스톤 정의, 자금 곡선 시뮬레이션(정밀 버전), 밸런스 조정 제안.

원시 수치(작물 가격, 성장일수, 시설 비용, 에너지 등)는 모두 canonical 문서를 참조한다.

## 1. 경험치(XP) 시스템 설계

### 1.1 설계 원칙

| 원칙 | 내용 |
|------|------|
| 초반 가속 | 레벨 1~3은 빠르게 달성하여 "해금의 즐거움"을 조기 체험 |
| 후반 둔화 | 레벨 7~10은 점진적으로 느려져 장기 목표로 기능 |
| 다중 소스 | 수확 외에도 도구 사용, 시설 건설, 가공 등에서 XP 획득 |
| 행동 보상 | 플레이어가 "게임을 플레이하는 행위" 자체에 경험치 부여 |

#### 1.2.1 작물 수확 XP (주요 소스, 전체 XP의 약 55%)

| 작물 | 기본 XP | 근거 |
|------|---------|------|
| 감자 | 5 XP | 기초 작물, 최저 XP |
| 당근 | 5 XP | 감자와 동일 tier |
| 토마토 | 8 XP | 중급 작물, 성장일수 5일 반영 |
| 옥수수 | 12 XP | 장기 투자 작물, 성장일수 7일 반영 |
| 딸기 | 6 XP | 재수확 작물이므로 회당 XP 억제 |
| 호박 | 18 XP | 고급 작물, 성장일수 10일 반영 |
| 해바라기 | 15 XP | 고급 작물, 성장일수 8일 반영 |
| 수박 | 25 XP | 최고급 작물, 성장일수 12일 반영 |

**XP 산출 공식**:

```
기본_XP = ceil(성장일수 * 1.5 + 해금_레벨 * 1.5)
```

검증:
- 감자: ceil(3 * 1.5 + 0 * 1.5) = 5 (시작 해금이므로 레벨 0 취급)
- 당근: ceil(3 * 1.5 + 0 * 1.5) = 5
- 토마토: ceil(5 * 1.5 + 2 * 1.5) = ceil(10.5) = 11 -> **수동 조정 8** (레벨 2 해금이 빠르므로 억제)
- 옥수수: ceil(7 * 1.5 + 3 * 1.5) = 15 -> **수동 조정 12** (여름 작물 과잉 XP 방지)
- 딸기: ceil(5 * 1.5 + 4 * 1.5) = 14 -> **수동 조정 6** (재수확 작물이므로 대폭 억제)
- 호박: ceil(10 * 1.5 + 5 * 1.5) = 23 -> **수동 조정 18** (가을 단기 재배이므로 적정 수준)
- 해바라기: ceil(8 * 1.5 + 5 * 1.5) = 20 -> **수동 조정 15**
- 수박: ceil(12 * 1.5 + 7 * 1.5) = 29 -> **수동 조정 25**

**참고**: 공식은 기준선을 제시하되, 최종 XP는 수동 조정한다. 재수확 작물(딸기)은 총 XP 수입이 과도해지지 않도록 회당 XP를 억제한다.

#### 1.2.2 품질 보너스 XP

품질 배수 (-> see `docs/systems/crop-growth.md` 섹션 4.3):

| 품질 | XP 배수 |
|------|---------|
| Normal | x1.0 |

##### 1.2.6.3 목축 XP 총합 시뮬레이션 (28일 = 1계절)

| 시나리오 | 일일 돌봄 XP | 일일 수확 XP | 일일 합계 | 28일 합계 |
|---------|------------|------------|----------|----------|
| 최소 (닭 x2) | 6 | 10 | 16 XP/일 | 448 XP |
| 중간 (닭 x4 + 소 x1) | 15 | 23 | 38 XP/일 | 1,064 XP |
| 적극 (닭 x4 + 소 x2 + 염소 x2) | 24 | 30 | 54 XP/일 | 1,512 XP |

**레벨업 기여도 분석**:
- 레벨 6→7 필요 XP: 839 XP (-> see 섹션 2.4.1). 최소 투자(닭 x2, 28일 448 XP)만으로는 1계절에 달성 불가. 작물 XP와 합산 시 레벨업 가속이 크다.
- 레벨 8→9 필요 XP: 2,147 XP (-> see 섹션 2.4.1). 중간 투자(닭 x4 + 소 x1) 시 목축 XP(28일 1,064 XP)만으로는 한 계절 내 달성 불가. 작물 XP 합산 필요.

[OPEN] 목축 XP가 레벨 6~10 구간의 레벨업 속도를 과도하게 가속시킬 수 있다. 기존 XP 소스(작물+경작+시설)와 합산 시 레벨 9~10 도달이 너무 빨라지면, 목축 XP에 일일 상한(예: 50 XP/일)을 도입하거나, 레벨 9~10 필요 XP를 상향 조정해야 할 수 있다.

#### 1.2.7 낚시 활동 XP (Zone F 해금 이후 추가 소스)

> **[FIX-064]** 낚시 XP는 Zone F 해금 이후에만 획득 가능하므로, 기존 레벨 1~5 진행 곡선에 영향을 주지 않는다. 목축 XP와 마찬가지로 기존 XP 소스 배분(~100%)에 **별도 가산**되는 구조이다.

**XPSource enum**: `FishingCatch` (-> see `docs/systems/fishing-architecture.md` 섹션 6.1, `docs/systems/progression-architecture.md` 섹션 2.2)

##### 1.2.7.1 희귀도별 기본 XP (FishingCatch)

| 희귀도 (FishRarity) | 기본 XP (expReward) | 근거 |
|---------------------|---------------------|------|
| Common | 10 XP | 작물 기초 수확 XP(5~8)의 약 1.5배. 낚시 미니게임 노력 보상 |
| Uncommon | 20 XP | Common의 2배. 중급 작물(옥수수 12 XP)보다 약간 높음 |
| Rare | 40 XP | Uncommon의 2배. 고급 작물(수박 25 XP)보다 높지만, 출현 빈도가 매우 낮음 |
| Legendary | 80 XP | Rare의 2배. 전설 어종은 특수 조건 필요하므로 최고 보상 |

**XP 산출 근거**: 희귀도 단계마다 2배 증가하는 등비 패턴. 낚시는 미니게임 스킬을 요구하므로 동일 시간 투자 대비 작물 수확보다 약간 높은 XP를 부여하되, 총 일일 XP가 작물 XP를 압도하지 않도록 설계한다.

##### 1.2.7.2 낚시 XP 계산 공식

```
낚시_XP = floor(fishData.expReward * qualityExpBonus[quality])
```

**품질 보정 배율**: 작물 수확과 **동일한 테이블을 공유**한다 (-> see 섹션 1.2.2 for canonical 테이블).

**예시**:
- Common 물고기 (Normal): floor(10 * 1.0) = 10 XP
- Common 물고기 (Gold): floor(10 * 1.5) = 15 XP
- Rare 물고기 (Silver): floor(40 * 1.2) = 48 XP
- Legendary 물고기 (Iridium): floor(80 * 2.0) = 160 XP

##### 1.2.7.3 낚시 XP 일일 시뮬레이션 (28일 = 1계절)

> 1일 낚시 가능 횟수를 에너지 제한 기반으로 약 8~12회로 가정 (-> see `docs/systems/farming-system.md` 섹션 3.2 for 에너지 시스템).

| 시나리오 | 일일 낚시 횟수 | 평균 희귀도 구성 | 일일 XP (추정) | 28일 합계 |
|---------|-------------|-----------------|---------------|----------|
| 캐주얼 (틈틈이) | 4회 | Common x3 + Uncommon x1 | 50 XP | 1,400 XP |
| 보통 (절반 시간) | 8회 | Common x5 + Uncommon x2 + Rare x1 | 130 XP | 3,640 XP |
| 집중 (전담) | 12회 | Common x7 + Uncommon x3 + Rare x1.5 + Legendary x0.5 | 210 XP | 5,880 XP |

**밸런스 판단**: "집중" 시나리오에서도 28일 5,880 XP는 레벨 6→7 필요 XP(839)를 크게 초과하므로, 낚시 전담 플레이 시 레벨업이 빠를 수 있다. 그러나 낚시 전담 시 작물 XP를 포기하는 기회비용이 있어 균형이 맞는다.

[OPEN] 낚시 XP가 목축 XP와 합산 시 레벨 7~10 구간 레벨업 속도를 과도하게 가속시킬 가능성이 있다. 필요 시 낚시 XP 일일 상한(예: 100 XP/일) 도입을 검토한다.

### 1.3 레벨별 필요 XP 테이블

#### 1.3.1 XP 곡선 설계 [DEPRECATED]

> **[DEPRECATED]** 이 섹션의 파라미터(baseXP=50, growthFactor=1.55)는 구 설계값이다. BAL-002 섹션 5(XP 조정)에서 재조정되었으며, 현재 canonical 파라미터는 **섹션 2.4.1**을 참조한다 (baseXP=80, growthFactor=1.60, 레벨 10 누적 XP=9,029). 아래 내용은 조정 과정의 맥락 이해용으로만 보존한다.

곡선 유형: **수정 지수 곡선** (초반 완만, 중반부터 가파르게)

```
필요_XP(레벨) = round(baseXP * growthFactor ^ (레벨 - 1))
```

### 2.3 문제 진단: XP 인플레이션

위 시뮬레이션의 핵심 문제는 **경작 활동 XP(물주기)가 과도하다**는 점이다:

```
물주기 XP 기여도 분석:
  봄: 420 XP / 1,085 XP = 39%
  여름: 560 XP / 1,562 XP = 36%
  
  15~20타일 x 28일 = 420~560 XP/계절
  이는 수확 XP(630~907)에 근접하는 양이다.
```

물주기는 매일 반복하는 기본 행동이므로, 이 자체가 거대한 XP 소스가 되면 "작물 선택"과 "재배 전략"의 XP 기여도가 희석된다. 섹션 5에서 XP 테이블과 소스 모두를 재조정한다.

| 레벨 10 누적 XP | 4,609 | **9,029** |

**효과**: 봄 1시즌에 레벨 4~5까지만 도달. 레벨 5(호박/온실)가 여름 초에 해금되어 계절과 정합.

#### 조정 2: 물주기 XP 제거 (P-02 해결)

| 항목 | 조정 전 | 조정 후 |
|------|--------|--------|
| 물주기 XP | 1 XP/회 | **0 XP** |
| 호미질 XP | 1 XP/회 | **2 XP/회** |

**효과**: 경작 활동 XP가 전체의 5% 이하로 감소. 수확 XP가 핵심 소스(85%+)가 되어 "무엇을 심느냐"가 진행 속도에 직접 영향.

**근거**: 물주기는 "해야 하는 일(chore)"이지 "선택하는 행위(choice)"가 아니다. XP는 플레이어의 의미 있는 선택에 부여해야 한다.

## Risks

- [RISK] XP 테이블은 수치 시뮬레이션에 기반하나, 실제 플레이 시 에너지 관리, 이동 시간, UI 조작 지연 등의 요인이 XP 수입을 20~30% 감소시킬 수 있다. 실제 플레이테스트에서 검증 필수
- [RISK] 가공소 해금(레벨 7, 가을) 이후 자금 곡선이 급격히 상승한다. BAL-001에서 지적한 가공 수익 과잉 문제가 해결되지 않으면, 본 문서의 자금 곡선 시뮬레이션이 과소 추정일 수 있다
- [RISK] 후반부(레벨 8~10) 해금 콘텐츠가 미정이므로, 현재 진행 곡선 분석은 레벨 7까지만 신뢰할 수 있다. 레벨 8~10 구간은 콘텐츠 확정 후 재검증 필요
- [RISK] 물주기 XP 제거(조정 2)로 인해, 물주기 자동화 시설(물탱크)의 XP 절감 메리트가 사라진다. 물탱크의 가치가 "에너지 절감"에만 의존하게 되며, 이는 충분히 강력한 동기인지 검토 필요
