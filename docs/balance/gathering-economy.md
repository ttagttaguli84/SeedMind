#### 2.1.1 일일 평균 활성 포인트 수

리스폰 주기와 확률을 반영하여 어느 임의의 날에 채집 가능한 포인트 수를 산출한다.

```
식물/버섯/열매 포인트 (12개소):
  2일 주기이므로, 매일 순회/수확한다면:
  Day N에 수확 -> Day N+2에 리스폰 확률 80%
  평균적으로 격일 수확 가능: 12 * (1/2) * 0.8 = 4.8개/일
  
  단, 매일 순회한다고 가정하면:
  Day 1: 12개소 모두 활성 (첫 수확)
  Day 2: 0개 (아직 리스폰 안 됨)
  Day 3: 12 * 0.8 = 9.6개
  Day 4: 0개
  ...
  평균: (12 + 0 + 9.6 + 0 + ...) / 2 = (12 + 9.6) / 2 * (1/2) ≈ 5.4개/일
  
  실질 보정: 계절 첫날은 100% 리스폰이므로 약간 상승.
  2일 주기 식물/열매 포인트 일평균 ≈ 5.4개/일 (격일 패턴)

광물 포인트 (3개소):
  3일 주기, 80% 리스폰:
  Day 1: 3개 활성
  Day 2: 0
  Day 3: 0
  Day 4: 3 * 0.8 = 2.4개
  Day 5: 0
  Day 6: 0
  ...
  일평균: (3 + 0 + 0 + 2.4 + 0 + 0) / 3 = 5.4 / 3 = 1.8 / 1 ≈ 0.8개/일
  보정: 3일 주기 광물 포인트 일평균 ≈ 0.8개/일
```

**Zone D 일평균 채집 아이템 수: 약 6.2개/일** (격일 변동 있음, 채집 가능일에는 ~12개, 비채집일에는 ~0.8개)

그러나 실제 플레이에서는 매일 순회하면:
- **짝수일 (리스폰일)**: 식물 ~9.6개 + 광물 ~0 or 2.4개 = 9.6~12개
- **홀수일**: 식물 0개 + 광물 0~2.4개 = 0~2.4개

**설계 의도에 부합하는 단순화 모델**: gathering-system.md 섹션 6.1에서는 "매일 80% 리스폰"으로 단순 계산하여 ~12개/일로 추정했다. 이는 "채집 포인트가 시간 차를 두고 리스폰되어 매일 일부가 활성"인 상황을 반영한 것으로 보인다.

**본 분석에서는 두 가지 모델을 병행**한다:

| 모델 | 일일 평균 아이템 수 | 적용 상황 |
|------|-------------------|----------|
| 모델 A: 격일 패턴 | 짝수일 ~10개, 홀수일 ~1개, 평균 5.5개 | 엄밀한 2일 주기 적용 |
| 모델 B: 단순 평균 | ~12개/일 (gathering-system.md 준용) | 포인트별 리스폰 시차 가정 |

[RISK] 모델 A와 모델 B 사이에 약 2배 차이가 있다. 이는 리스폰 시차(각 포인트가 서로 다른 날 리스폰)를 적용하는지 여부에 따른다. **리스폰 시차 적용이 설계 의도인지 확인이 필요**하다. 본 분석에서는 모델 B(단순 평균)를 기본으로 사용하되, 모델 A 결과도 병기한다.

## 7. 밸런스 이슈 식별 및 조정 제안

### 7.1 이슈 1: 초기~중기 채집 수입 비중 과도 (Critical)

**문제**: 채집 수입 비중이 초기 ~45%, 중기 ~38%로 설계 목표(15~20%)를 크게 초과한다. 채집이 경작/낚시보다 효율적인 수입원이 되어 "보조 활동" 포지셔닝이 무너진다.

**원인 분석**:
1. 채집물 판매가가 상대적으로 높음 (Common 평균 ~10G, 일일 12개 = 120G+)
2. 에너지 소모가 0(맨손)이어서 순수익 = 총수익
3. 일일 채집 가능 수량(12~17개)이 충분한 수입을 생성
4. 초기 작물 수입이 낮아 상대적으로 채집 비중이 높아짐

**조정안 A: 판매가 하향 (권장)**

전체 채집물 판매가를 50~60% 수준으로 하향 조정한다.

| 희귀도 | 현재 평균 판매가 | 조정 후 평균 판매가 | 비율 |
|--------|---------------|-------------------|------|
| Common | 8~15G | 4~8G | ~55% |
| Uncommon | 18~30G | 10~16G | ~55% |
| Rare | 45~100G | 25~55G | ~55% |
| Legendary | 150~300G | 80~160G | ~55% |

조정 후 Zone D Lv.1 가을 수입: 297G * 0.55 = ~163G
전 구역 Lv.1 가을 수입: 402G * 0.55 = ~221G

```
조정 후 초기 채집 비중:
  작물 (12칸): ~65G
  채집 (Zone D, 조정 후): ~121G (봄 기준 220*0.55)
  낚시 (Lv.1): ~200G
  합계: ~386G
  채집 비중: 121/386 = ~31% -- 여전히 높지만 개선
```

**조정안 B: 리스폰 주기 증가**

| 유형 | 현재 주기 | 조정 주기 |
|------|----------|----------|
| 식물/버섯/열매 | 2일 | 3일 |
| 광물 | 3일 | 4일 |

이 경우 일일 평균 채집 수량이 약 2/3로 감소:
- Zone D: 12개/일 -> 8개/일
- 전 구역: 17개/일 -> 11~12개/일

조정 후 Zone D Lv.1 가을: 297 * (8/12) = ~198G
조정 후 초기 채집 비중: 198 / (65+198+200) = ~43% -- 여전히 높음

**조정안 C: 판매가 하향 + 리스폰 주기 증가 (최강 조정)**

A + B 동시 적용:
- 판매가 55%
- 리스폰 3일/4일

Zone D Lv.1 가을: 297 * 0.55 * (8/12) = ~109G

```
조정 후 초기 채집 비중:
  작물: ~65G
  채집: ~109G (봄: 220*0.55*0.67 = ~81G)
  낚시: ~200G
  합계: ~374G (봄 ~346G)
  채집 비중: 81/346 = ~23% (봄), 109/374 = ~29% (가을)
```

여전히 20% 목표에는 미달하지만, "보조 수입원" 느낌에는 근접.

**조정안 D: 판매가 하향 40% 수준 (최적 추천)**

판매가를 현재의 40% 수준으로 하향한다.

| 희귀도 | 현재 평균가 | 조정 후 | 예시 |
|--------|-----------|---------|------|
| Common | 8~15G | 3~6G | 민들레 3G, 달래 5G, 도토리 3G, 능이 6G |
| Uncommon | 18~30G | 8~12G | 봄나물 8G, 표고 12G, 연잎 9G |
| Rare | 45~100G | 18~40G | 송이(봄) 18G, 송이버섯 32G, 동충하초 40G |
| Legendary | 150~300G | 60~120G | 산삼 80G, 천년 영지 120G |

조정 후 Zone D Lv.1 봄: 220 * 0.40 = ~88G

```
조정 후 초기 채집 비중:
  작물: ~65G
  채집: ~88G
  낚시: ~200G
  합계: ~353G
  채집 비중: 88/353 = ~25%
```

중기 (전 구역, Lv.5, 가을):
  채집: 630 * 0.40 * 0.85(수급) = ~214G
  작물 + 낚시 + 목축: ~860G
  합계: ~1,074G
  채집 비중: 214/1,074 = ~20% -- 목표 범위 도달

후기 (Lv.10):
  채집: 770 * 0.40 * 0.85 = ~262G
  기타: ~3,191G
  합계: ~3,453G
  채집 비중: 262/3,453 = ~7.6% -- 자연스럽게 하락

### 7.2 이슈 2: 에너지 무소모의 경제적 우위

**문제**: 맨손 채집은 에너지 0이므로 "공짜 수입"이 된다. 경작(에너지 30~55)이나 낚시(에너지 50)와 달리 기회비용이 극히 낮다.

**판정**: 채집의 기회비용은 에너지가 아니라 **시간**이다. 채집 순회에 1 게임시간이 필요하므로, 그 시간을 다른 활동에 쓸 수 없다. 다만 에너지 0은 여전히 강력한 이점이다.

**조정 불필요 (판매가 하향으로 해소)**: 조정안 D 적용 시 에너지 무소모의 이점이 낮은 판매가로 상쇄된다. 추가로 에너지 소모를 부과하면 "가볍게 순회하는" 채집 컨셉이 훼손될 수 있다.

### 7.3 이슈 3: 겨울 채집 수입이 과도하게 낮음

**문제**: 겨울 일일 채집 수입이 ~25G로 다른 계절(220~400G)의 6~16%에 불과하다.

**판정**: 이것은 **의도된 설계**이다 (-> see `docs/systems/gathering-system.md` 섹션 3.6). 겨울은 온실 경작과 가공에 집중하는 시즌이며, 야외 활동(채집, 낚시)은 제한적이어야 한다. **조정 불필요**.

### 7.4 이슈 4: 채집물 가공 레시피 미확정

**문제**: processing-system.md에 채집물 가공 레시피가 정식 등록되지 않아, 가공 수입의 정확한 분석이 불가능하다.

[OPEN] 채집물 가공 레시피 확정 시 본 분석의 섹션 4를 업데이트해야 한다. 가공 부가가치가 과도할 경우 채집 비중이 다시 상승할 수 있으므로 주의.

### 7.5 이슈 5: 모델 A vs 모델 B 불확실성

**문제**: 리스폰 시차 적용 여부에 따라 일일 수입이 약 2배 차이가 난다.

[OPEN] 리스폰 시차(각 포인트가 독립적 리스폰 타이머를 가져 매일 일부가 활성)를 적용하는 것이 설계 의도인지 확정이 필요하다. 모델 B(시차 적용, 매일 일정 수량 확보 가능)가 게임 경험상 바람직하지만, 아키텍처적으로는 모델 A(동기화 리스폰)가 구현이 단순하다.

## Risks

1. [RISK] **초기 채집 비중 과도 (Critical)**: 현행 판매가 기준 초기 채집 수입 비중이 ~45%로 설계 목표(15~20%)를 크게 초과. 조정안 D 적용 시에도 초기 36%(절대값 88G). 초기 작물 수입이 낮은 구조적 문제와 결합하여 발생. **gathering-system.md 판매가 하향이 필수적**.
2. [RISK] **에너지 0 + 높은 수입 = 지배 전략**: 맨손 채집(에너지 0)으로도 상당한 수입이 가능하면 "채집만 하는 것이 최적" 전략이 출현할 수 있다. 판매가 하향으로 완화되지만, 가공 레시피 추가 시 재검증 필요.
3. [RISK] **가공 추가 시 비중 재상승**: 채집물 가공 레시피가 추가되면 부가가치로 인해 채집 비중이 다시 상승할 수 있다. 가공 부가가치를 작물 가공보다 낮게 설정해야 한다.
4. [RISK] **수급 시스템이 형식적**: 전 구역 매일 순회 시 수급 배수가 즉시 최저(0.85)에 도달하여, 수급이 사실상 "고정 15% 할인"으로 작동한다.
