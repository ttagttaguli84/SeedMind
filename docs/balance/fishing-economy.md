## 1. basePrice 확정

### 1.1 확정 원칙

basePrice는 다음 기준에 따라 결정한다:

1. **희귀도 밴드**: 각 희귀도 등급 내에서 가격 범위를 설정하고, 등장 조건의 까다로움에 비례하여 밴드 내 위치를 결정
2. **보조 수입 포지셔닝**: Common 어종의 기대 수익이 작물 일일 수익률(-> see `docs/balance/crop-economy.md` 섹션 1.2)의 50~70% 수준
3. **가공 연계 적합성**: 가공 후 판매가가 작물 가공품 대비 경쟁력은 있되 압도하지 않는 수준
4. **에너지 대비 수익**: 에너지 1당 수익이 작물 활동(물주기+수확 평균)과 유사하거나 약간 높은 수준

### 1.2 희귀도별 가격 밴드

| 희귀도 | 가격 밴드 | 기준 |
|--------|----------|------|
| Common | 15~35G | 감자 판매가(30G) 이하~동등 수준 |
| Uncommon | 40~70G | 토마토(60G)~옥수수(100G) 판매가 사이 |
| Rare | 100~200G | 호박(200G) 판매가 수준까지 |
| Legendary | 400~800G | 수박(350G) 이상, 가공품 수준 |

### 2.3 낚시 1회 기대값 (희귀도 가중 평균)

**일반 낚시 시나리오** (Legendary 등장 조건 미충족 가정, 대부분의 시간):

등장 조건을 엄밀히 반영하면 각 시점의 어종 풀이 다르지만, 대표적 시나리오로 분석한다.

#### 시나리오 A: 여름 Afternoon Clear (최다 어종 풀)

어종 풀: 붕어(C), 미꾸라지(C), 잉어(C), 블루길(C), 가재(U), 황금 잉어(R)
- 연꽃 잉어(L)는 Dawn 전용이므로 제외

| 희귀도 | 가중치 | 어종수 | 개별 가중치 | 어종별 가격 |
|--------|--------|--------|-----------|-----------|
| Common | 60 | 4종 | 15 | 붕어 20, 미꾸라지 25, 잉어 30, 블루길 22 |
| Uncommon | 25 | 1종 | 25 | 가재 45 |
| Rare | 12 | 1종 | 12 | 황금 잉어 200 |
| Legendary | 0 | 0종 | - | - |
| **합계** | **97** | | | |

```
기대값 = (15*20 + 15*25 + 15*30 + 15*22 + 25*45 + 12*200) / 97
       = (300 + 375 + 450 + 330 + 1125 + 2400) / 97
       = 5,980 / 97
       = 39.4G/회 (Normal 품질)
```

#### 시나리오 B: 봄 Dawn Clear (희귀종 활성)

어종 풀: 붕어(C), 미꾸라지(C), 잉어(C), 송어(U), 산천어(R)

| 희귀도 | 가중치 | 어종수 | 개별 가중치 | 어종별 가격 |
|--------|--------|--------|-----------|-----------|
| Common | 60 | 3종 | 20 | 붕어 20, 미꾸라지 25, 잉어 30 |
| Uncommon | 25 | 1종 | 25 | 송어 50 |
| Rare | 12 | 1종 | 12 | 산천어 120 |
| **합계** | **97** | | | |

```
기대값 = (20*20 + 20*25 + 20*30 + 25*50 + 12*120) / 97
       = (400 + 500 + 600 + 1250 + 1440) / 97
       = 4,190 / 97
       = 43.2G/회
```

#### 시나리오 C: 가을 Night Storm (Legendary 활성)

어종 풀: 붕어(C), 잉어(C), 블루길(C), 메기(U), 뱀장어(U), 무지개송어(R), 전설의 메기왕(L)

| 희귀도 | 가중치 | 어종수 | 개별 가중치 | 어종별 가격 |
|--------|--------|--------|-----------|-----------|
| Common | 60 | 3종 | 20 | 붕어 20, 잉어 30, 블루길 22 |
| Uncommon | 25 | 2종 | 12.5 | 메기 55, 뱀장어 65 |
| Rare | 12 | 1종 | 12 | 무지개송어 150 |
| Legendary | 3 | 1종 | 3 | 전설의 메기왕 500 |

```
기대값 = (20*20 + 20*30 + 20*22 + 12.5*55 + 12.5*65 + 12*150 + 3*500) / 100

```

#### 시나리오 D: 겨울 Dawn Snow (얼음 낚시)

어종 풀: 빙어(C), 얼음 빙어왕(R)

| 희귀도 | 가중치 | 어종수 | 개별 가중치 | 어종별 가격 |
|--------|--------|--------|-----------|-----------|
| Common | 60 | 1종 | 60 | 빙어 18 |
| Rare | 12 | 1종 | 12 | 얼음 빙어왕 180 |
| **합계** | **72** | | | |

```
기대값 = (60*18 + 12*180) / 72
       = (1080 + 2160) / 72
       = 45.0G/회
```

**겨울 Dawn 외 시간(Clear)**은 빙어 1종만 등장하므로 기대값 = 18G/회.

### 2.4 Giant/품질 보정 포함 기대값

Giant 대상 어종: 잉어(C, 30G), 산천어(R, 120G), 황금 잉어(R, 200G), 무지개송어(R, 150G)

Giant 보정 기대값 상승분:
```
Giant 보정 = 해당 어종 basePrice * (giantChance * giantPriceMult - giantChance)
           = basePrice * 0.05 * (2.0 - 1.0)
           = basePrice * 0.05

잉어: 30 * 0.05 = +1.5G -> 실질 기대가 31.5G
산천어: 120 * 0.05 = +6G -> 실질 126G
황금 잉어: 200 * 0.05 = +10G -> 실질 210G
무지개송어: 150 * 0.05 = +7.5G -> 실질 157.5G
```

Giant 보정의 전체 기대값 영향은 미미 (시나리오 A 기준 +0.5G/회 미만).

**품질 보정 기대값** (숙련도별):

| 숙련도 | 품질 분포 추정 | 품질 가중 배수 |
|--------|--------------|---------------|
| Lv.1~2 | Normal 100% | x1.00 |
| Lv.3~5 | Normal 70%, Silver 30% | x1.075 |
| Lv.6~8 | Normal 50%, Silver 30%, Gold 20% | x1.175 |
| Lv.9~10 | Normal 30%, Silver 30%, Gold 30%, Iridium 10% | x1.325 |

품질 분포는 미니게임 잔여 시간 기반이므로 플레이어 실력에 의존한다. 위 수치는 중간 숙련 플레이어 추정치이다.

## 3. 시나리오 시뮬레이션

### 3.1 전제

- 하루 최대 에너지: 100 (-> see `docs/systems/farming-system.md` 섹션 3.2)
- 농작업 에너지 소모: 40~50 (물주기+수확+기타)
- 남은 에너지로 낚시: 50~60 에너지
- 1회 낚시 평균 에너지: 2.5 (Lv.1, 성공률 50%), 2.35 (Lv.5, 성공률 65%), 1.2 (Lv.8+, 성공률 80%) -- BAL-013 확정
- 1회 낚시 평균 실시간: ~20초
- 하루 게임 내 시간: 18시간(06:00~24:00), 실시간 ~10분

### 3.2 시나리오 1: 초보 낚시꾼 (Lv.1, 기본 낚싯대)

```
남은 에너지: 50
에너지/회: 2*0.50 + 3*0.50 = 2.5E (성공률 50% -- BAL-013 확정)
가능 낚시 횟수: 50 / 2.5 = 20회
성공률: 50% (-> see docs/systems/fishing-system.md 섹션 7.4)
실제 포획: 20 * 0.5 = 10마리
기대 수익: 10 * 39.4G = 394G/일

일일 낚시 소요 시간: 20 * 20초 = 400초 = ~7분 (실시간)
게임 내 시간: ~3.5시간 (대기 포함)
```

### 3.3 시나리오 2: 숙련 낚시꾼 (Lv.5, 강화 낚싯대)

```
남은 에너지: 50
에너지/회: 2*0.65 + 3*0.35 = 2.35E (성공률 65% -- BAL-013 확정)
가능 낚시 횟수: 50 / 2.35 ≈ 21회
성공률: 65% (-> see docs/systems/fishing-system.md 섹션 7.4)
실제 포획: 21 * 0.65 ≈ 14마리
품질 보정 배수: x1.075
기대 수익: 14 * 39.4 * 1.075 = 593G/일

바이트 대기 보정: x0.85 -> 평균 소요 ~17초/회
일일 낚시 소요 시간: 21 * 17초 = 357초 = ~6분
```

### 3.4 시나리오 3: 마스터 낚시꾼 (Lv.10, 전설 낚싯대)

```
남은 에너지: 50
에너지/회: 1*0.80 + 2*0.20 = 1.2E (Lv.8+ 캐스팅 에너지 1 + 실패20%*추가1, 성공률 80% -- BAL-013 확정)
가능 낚시 횟수: 50 / 1.2 = 42회
성공률: 80% (-> see docs/systems/fishing-system.md 섹션 7.4)
실제 포획: 42 * 0.80 = 34마리
더블 캐치(5%): +1.7마리 -> 35.7마리
품질 보정 배수: x1.325
기대 수익: 35.7 * 39.4 * 1.325 = 1,864G/일

바이트 대기 보정: x0.70 -> 평균 소요 ~14초/회
일일 낚시 소요 시간: 42 * 14초 = 588초 = ~10분
```

## 6. 밸런스 이슈 및 조정 제안

### 6.1 이슈 1: 낚시 수익이 "보조 수입" 수준을 초과

**문제 (BAL-012 분석 당시)**: 성공률 하향 전 Lv.10 낚시 일일 수익(수급 적용 1,490G)이 수박 24칸 직판(350G/일)의 4배 이상이었다. Lv.1에서도 473G로 경작 수익을 상회했다. BAL-013에서 성공률 하향을 확정 적용하여 해결.

**원인 분석**:
1. 낚시에 금전적 투입 비용(씨앗비)이 없어 순이익 = 총수익
2. 에너지 50으로 21~38회 낚시가 가능하여 회수가 많음
3. Common 기대값 23G가 감자 순이익(15G)보다 높음

**조정안 A: 성공률 하향 (BAL-013에서 확정 적용)**

성공률을 하향 조정하여 "보조 수입" 포지셔닝을 확보한다. canonical 수치는 `docs/systems/fishing-system.md` 섹션 7.4에 등록 완료.

| 숙련도 | 기존 추정 성공률 | 확정 성공률 (BAL-013) |
|--------|----------------|---------------------|
| Lv.1 | 70% | **50%** |
| Lv.5 | 80% | **65%** |
| Lv.10 | 90% | **80%** |

확정 후 수익 (수급 적용):

| 시나리오 | 확정 수익 (BAL-013) | 비교 대상 |
|----------|-------------------|----------|
| Lv.1 | 315G | 수박 24칸 350G/일 |
| Lv.5 | 474G | 수박 24칸 350G/일 |
| Lv.10 | 1,491G | 수박 24칸+가공 700G/일 |

Lv.1(315G)은 수박 24칸(350G) 이하로 "보조 수입" 포지셔닝에 부합한다. Lv.10 도달에 약 1~2계절이 필요한 점을 고려하면 후반 보상으로 수용 가능.

**조정안 B: 에너지 소모 상향**

| 행동 | 현재 | 조정안 |
|------|------|--------|
| 캐스팅 에너지 | 2 | 3 |
| 미니게임 실패 추가 | 1 | 2 |

이 경우 Lv.1 가능 횟수: 50/3.5 = ~14회, 포획 7마리, 수익 ~220G. "보조 수입"에 가까우나 낚시 체감이 나빠질 수 있다.

**조정안 C: 시간 기반 제약 강화**

낚시 중 게임 시간이 흐르는 설정(현재 [OPEN])을 확정하면, 1회 낚시에 게임 내 8~15분이 소요되어 하루 가용 시간이 제한된다. 18게임시간(06:00~24:00) 중 농작업 4시간을 빼면 14시간, 1회 ~10분이면 최대 84회이나 실제로는 이동/다른 활동으로 40~50회가 현실적.

**결론**: 조정안 A(성공률 하향)를 BAL-013에서 확정 적용 완료. 추가 조정이 필요하면 조정안 B를 병행 검토. 조정안 C는 시간 흐름 [OPEN] 결정 후 자동 반영.

### 6.2 이슈 2: Legendary 어종 반복 포획 시 수익 폭등

**문제**: 가을 Night Storm에서 전설의 메기왕(500G) 반복 포획 시 단시간 대량 수익 발생 가능.

**현행 제약**:
- Storm 발생 확률이 낮음 (-> see `docs/systems/time-season.md` 섹션 3.2)
- Night(20:00~24:00) = 4게임시간 = 실시간 ~2.2분
- Legendary 가중치 3/100 = 3% 확률

**시뮬레이션**: 4게임시간 동안 약 6~8회 낚시 가능. Legendary 기대 포획: 0.2~0.24마리. 500G * 0.22 = ~110G. 나머지 7.8회 * 일반 기대값 50G = 390G. 합계 ~500G.

**판정**: Storm Night 4시간 동안의 총 수익 ~500G는 과도하지 않다. Legendary 등장 확률 3%와 제한된 시간대가 충분한 제약을 제공한다. **조정 불필요**.

### 6.3 이슈 3: 겨울 낚시 수익 부족

**문제**: 겨울 Dawn Snow 외 시간(Clear)에는 빙어(18G) 1종만 등장. 시간당 수익이 극히 낮다.

**판정**: 이것은 의도된 설계다. 겨울 낚시는 "콘텐츠 다양성"이 목적이지 수익원이 아니다. 겨울의 주 수입은 온실 경작과 가공소 운영이어야 한다. **조정 불필요**.

### 6.4 이슈 4: 생선 초밥 ROI가 과도

**문제**: 생선 초밥(황금 잉어)의 시간당 부가가치가 110G/h로, 호박 잼(52.5G/h)의 2배 이상.

**제약 요인**:
- 황금 잉어(Rare): 여름 Morning~Afternoon Clear 전용, 출현 확률 12%
- 베이커리 해금: 레벨 9 (후반부)
- 쌀 획득 경로 미확정 [OPEN]

**판정**: 높은 해금 문턱과 재료 확보 난이도를 감안하면 후반부 보상으로 수용 가능. 단, 쌀의 가격이 30G 미만이면 ROI가 더 상승하므로, **쌀 가격을 40~50G 이상으로 설정하여 부가가치를 하향 조정할 것을 권장**한다.

[OPEN] 쌀 가격 확정 시 생선 초밥 ROI 재검증 필요.

### 6.5 이슈 5: fishing-system.md 섹션 9.1과의 시뮬레이션 차이

**문제**: fishing-system.md 섹션 9.1.1의 시간 기반 분석(~58G/게임시간)과 본 문서의 에너지 기반 분석(~591G/일)은 관점이 다르다.

- fishing-system.md: "1게임시간에 1.5회 낚시" 기준, 시간 효율 분석
- 본 문서: "남은 에너지 50 전부 투입" 기준, 일일 총수익 분석

**판정**: 두 분석은 모순이 아니라 보완적이다. fishing-system.md의 분석은 낚시 시간 중 게임 시간이 흐르는 것을 반영한 시간 효율이고, 본 문서의 분석은 에너지 예산 기반의 일일 총수익이다. BAL-013 적용 후 fishing-system.md 섹션 9.1.2에 본 문서 교차 참조를 추가 완료.

## Risks

1. [RISK] **"낚시만 하는 게 최적" 지배 전략 출현**: 현행 수치에서 Lv.10 낚시 수익이 경작 대비 과도하다. 조정안 A 미적용 시 경작 동기가 약화될 수 있다.
2. [RISK] **수급 시스템 무력화**: 집중 낚시 시 수급 배수가 즉시 최저(0.8)에 도달하여, 수급 시스템이 사실상 "고정 20% 할인"으로 작동한다. 수급 카테고리 파라미터(supplyDropRate, minSupplyMultiplier) 조정을 검토할 수 있다.
3. [RISK] **가공 체인 복잡도**: 생선 5종 가공 + 기존 작물 32종 가공 = 총 37종 레시피. 플레이어가 최적 가공 경로를 파악하기 어려울 수 있다. UI에서 시간당 수익 표시 등의 보조 정보 제공을 고려.
