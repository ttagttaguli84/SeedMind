## 3. XP 기여도 분석

### 3.1 기준 XP 데이터

전체 누적 XP (레벨 10 도달): **4,609 XP** (-> see `docs/balance/progression-curve.md` 섹션 1.3.2 레벨 테이블)

퀘스트 XP 설계 목표: 전체의 **10~15%** = 461~692 XP (-> see `docs/systems/quest-system.md` 섹션 7.3)

### 3.2 퀘스트 XP 비율 분석

| 카테고리 | XP | 전체 대비 비율 (4,609 기준) |
|----------|-----|---------------------------|
| 메인 퀘스트 | 1,300 XP | 28.2% |
| NPC 의뢰 | 590 XP | 12.8% |
| 일일 목표 (1년차) | 2,857 XP | 62.0% |
| 농장 도전 (전체) | 4,400 XP | 95.4% |
| **퀘스트 전체** | **9,147 XP** | **198.5%** |
| **1년차 실현** | **~6,247 XP** | **~135.5%** |

### 3.3 판정

**[RISK] 퀘스트 XP가 목표치를 극단적으로 초과한다.**

설계 목표는 퀘스트 XP가 전체의 **10~15%**(-> see `docs/systems/quest-system.md` 섹션 7.3)이나, 실제 집계 결과는:

- 메인 퀘스트 + NPC 의뢰만으로도 1,890 XP = 41.0% (목표의 2.7배)
- 일일 목표까지 합산하면 4,747 XP = 103% (퀘스트 XP만으로 레벨 10 도달)
- 농장 도전까지 포함하면 9,147 XP = 198.5% (필요 XP의 2배)
- 1년차 실현 기준 6,247 XP = 135.5% (역시 레벨 10 도달 가능)

이는 progression-curve.md의 XP 시뮬레이션이 퀘스트 XP를 포함하지 않은 상태에서 산출되었기 때문이다. 퀘스트 XP가 기존 수확/경작 XP에 **추가**로 들어오면, 레벨업 속도가 설계의 3~4배로 가속된다.

### 3.4 핵심 문제: XP 소스 간 관계 미정의

현재 설계는 다음 XP 소스들이 각각 독립적으로 설계되어, 합산했을 때 과잉이 발생한다:

| XP 소스 | 독자 설계 XP | 비고 |
|---------|-------------|------|
| 수확/경작 (progression-curve.md) | 4,609 XP (레벨 10) | 이것만으로 레벨 10 도달 설계 |
| 퀘스트 (quest-system.md) | 9,147 XP | 이것만으로도 레벨 10 2회 도달 가능 |
| 업적 (achievements.md) | 2,250 XP | 추가 보너스 (-> see [RISK] in `docs/content/achievements.md` 섹션 2.4) |
| **합산** | **~16,006 XP** | 필요 XP의 347% |

[OPEN] progression-curve.md의 XP 시뮬레이션은 "퀘스트 XP 없이" 1년차에 레벨 7~8 도달을 전제한다. 퀘스트 XP 추가 시, XP 테이블(baseXP, growthFactor) 재조정 또는 퀘스트 XP 대폭 삭감이 불가피하다. 어느 쪽을 선택할지 결정이 필요하다.

- **가을**: 25%로 적정.
- **[RISK] 겨울**: 75%로 심각하게 초과. 겨울은 작물 수익이 낮은 시기인데(온실 의존), 메인 퀘스트 보상이 1,500G나 되어 "퀘스트만 해도 충분한" 상황이 발생. 겨울의 경제적 긴장감(자원 관리)이 사라진다.

## 6. 밸런스 판정 및 조정 제안

### 6.1 종합 판정

| 항목 | 판정 | 상세 |
|------|------|------|
| 퀘스트 XP 비율 | **심각 초과** | 목표 10~15% vs 실제 100%+ |
| 메인 퀘스트 골드 (봄/여름) | 소폭 초과 | 38% vs 목표 20~30% |
| 메인 퀘스트 골드 (겨울) | **심각 초과** | 75% vs 목표 20~30% |
| 일일 목표 골드 (전체) | **심각 초과** | 작물 수익의 82% |
| 일일 목표 골드 (초반) | **심각 초과** | 레벨 1에서 작물 수익의 159% |
| 농장 도전 골드 | 적정 | 장기 도달 보상으로서 적절한 규모 |
| NPC 의뢰 골드 | 적정 | 작물 수익의 28%, 단계적 해금 |

### 6.2 조정 제안

#### 제안 A: XP 대폭 삭감 (권장)

퀘스트 XP를 전체 XP(4,609)의 10~15% 범위로 맞추려면, 목표 퀘스트 XP 총량 = 461~692 XP.

| 카테고리 | 현행 XP | 제안 XP | 삭감률 |
|----------|--------|---------|--------|
| 메인 퀘스트 | 1,300 XP | 200 XP | -85% |
| NPC 의뢰 | 590 XP | 100 XP | -83% |
| 일일 목표 (1년차) | 2,857 XP | 200 XP | -93% |
| 농장 도전 | 4,400 XP | 192 XP | -96% |
| **합계** | **9,147 XP** | **~692 XP** | **-92%** |

구체적 조정:
- 메인 퀘스트: 각 퀘스트 XP를 현행의 1/7로 (예: 30 XP -> 5 XP, 150 XP -> 20 XP)
- 일일 목표: XP를 2~5 범위로 대폭 축소, 레벨 스케일링 제거
- 농장 도전: 대형 도전만 XP 유지(20~30 XP), 소형 도전은 XP 제거

#### 제안 B: XP 테이블 상향 (대안)

퀘스트 XP를 줄이지 않고, XP 테이블의 growthFactor를 올려 총 필요 XP를 ~20,000으로 확장.

| 파라미터 | 현행 | 제안 |
|----------|------|------|
| baseXP | 50 | 50 |
| growthFactor | 1.55 | 1.90 |
| 총 필요 XP (레벨 10) | 4,609 | ~20,000 |

[RISK] XP 테이블 상향 시, 퀘스트를 하지 않는 플레이어의 레벨업이 극단적으로 느려진다. 수확/경작 XP만으로 레벨 10 도달에 5~6년차(실시간 ~100시간)가 필요해질 수 있어, 캐주얼 플레이어 이탈 우려.

#### 제안 C: 골드 보상 조정

| 카테고리 | 현행 | 제안 | 조정 근거 |
|----------|------|------|----------|
| 봄 메인 퀘스트 | 380G | 200G | 작물 수익의 20% 수준 |
| 여름 메인 퀘스트 | 1,150G | 600G | 작물 수익의 20% 수준 |
| 겨울 메인 퀘스트 | 1,500G | 500G | 작물 수익의 25% 수준 (겨울 경제 긴장감 유지) |
| 일일 목표 기본 골드 | 30~100G | 15~50G | 초반 과잉 방지, 50% 삭감 |
| 일일 목표 레벨 스케일링 | x1.0~x1.9 | x1.0~x1.4 | 상한 축소 (maxLevel 계수 0.1 -> 0.05) |

### 6.3 권장 조합

**제안 A + C를 병행 적용**하는 것을 권장한다.

- XP: 제안 A 적용으로 퀘스트 XP를 전체의 ~15%로 제한
- 골드: 제안 C 적용으로 퀘스트 골드가 작물 수익의 20~30% 이내 유지
- 농장 도전: 골드 보상은 현행 유지 (장기 목표 보상으로서 적절), XP만 삭감

조정 후 예상 수치:

| 카테고리 | 조정 후 골드 | 조정 후 XP |
|----------|------------|-----------|
| 메인 퀘스트 | ~2,200G | ~200 XP |
| NPC 의뢰 | 3,100G (유지) | ~100 XP |
| 일일 목표 (1년차) | ~4,500G | ~200 XP |
| 농장 도전 | ~11,800G (유지) | ~192 XP |
| **합계** | **~21,600G** | **~692 XP** |

조정 후 퀘스트 XP 비율: 692 / 4,609 = **15.0%** (목표 범위 상한)

조정 후 1년차 퀘스트 골드: ~12,750G (= 메인 2,200 + NPC 3,100 + 일일 4,500 + 도전 ~2,950)  
작물 수익 대비: 12,750 / 11,000 = 116% -> 여전히 높으나, 농장 도전의 대부분이 2~3년차에 달성되므로 1년차 실현은 ~9,800G(89%)로 수렴.

[OPEN] 1년차 퀘스트 골드가 작물 수익의 89%라면 아직도 높다. 추가로 NPC 의뢰 골드 삭감(현행의 60% 수준)이나 일일 목표 골드 추가 삭감을 고려해야 하는지, 또는 "퀘스트가 보조 수입원"이라는 전제 자체를 "퀘스트가 동등 수입원"으로 재정의할지 결정이 필요하다.

## 7. 조정 후 시뮬레이션

제안 A+C 적용 시, 1년차 일반 플레이어의 경제/XP 흐름:

### 7.1 계절별 통합 수입 (조정 후)

| 계절 | 작물 판매 | 메인 퀘스트 | NPC 의뢰 | 일일 목표 | 계절 합계 |
|------|----------|-----------|---------|---------|----------|
| 봄 | 1,000G | 200G | 200G | 1,000G | 2,400G |
| 여름 | 3,000G | 600G | 400G | 1,100G | 5,100G |
| 가을 | 5,000G | 700G | 500G | 1,200G | 7,400G |
| 겨울 | 2,000G | 500G | 300G | 1,200G | 4,000G |
| **연간** | **11,000G** | **2,000G** | **1,400G** | **4,500G** | **18,900G** |

작물 판매 비중: 11,000 / 18,900 = **58.2%** (작물이 주 수입원 지위 유지)

### 7.2 XP 통합 흐름 (조정 후)

| 소스 | 1년차 XP | 비율 (레벨 10 = 4,609 기준) |
|------|---------|------|
| 수확/경작 (기존, -> see `docs/balance/progression-curve.md`) | ~3,332 XP | 72.3% |
| 퀘스트 (조정 후 1년차) | ~592 XP | 12.8% |
| 업적 (1년차 실현분) | ~500 XP | 10.8% |
| **합계** | **~4,424 XP** | - |

1년차 종료 시 누적 XP ~4,424 -> 레벨 9 (2,941) 초과, 레벨 10 (4,609) 근접. progression-curve.md의 "1년차 레벨 7~8" 설계보다 빠르다.

[OPEN] 퀘스트 XP(제안 A) + 업적 XP 포함 시 1년차 레벨 9 후반 도달은 progression-curve.md의 "1년차 레벨 7~8 직전" 설계보다 상당히 빠르다. 업적 XP까지 합산하면 제안 A만으로도 추가 조정이 필요할 수 있다. 세 소스(수확/경작, 퀘스트, 업적)를 통합한 시뮬레이션 재설계가 필요하다.

## Risks

- [RISK] **XP 인플레이션**: 현행 수치 그대로 구현 시, 퀘스트 XP만으로 레벨 10 도달 가능. 진행 곡선 전체가 무의미해진다. 우선 해결 필요.
- [RISK] **골드 인플레이션**: 1년차 퀘스트 골드가 작물 수익을 능가하여, 농장 경영이 아닌 퀘스트 수행이 주 수입원이 되는 역전 현상.
- [RISK] **겨울 밸런스 붕괴**: 겨울 메인 퀘스트 1,500G는 작물 수익(2,000G)의 75%로, 경제적 긴장감이 사라진다.
- [RISK] **소형 농장 도전 수치 미확정**: 22개 중 15개의 정확한 보상이 quest-system.md에 상세 기재되어 있지 않아, 본 분석의 추정치(평균 200G + 100 XP)가 실제와 다를 수 있다.
